# 応用情報（AI向けメモ）

このディレクトリは **応用情報技術者試験（AP）** の学習ノート用 Obsidian ボルトです。人間が読むことも想定していますが、**この README は主に AI エージェント向け**の運用説明です。

## 目的

- 過去問・分野別の知識・用語・解法メモを蓄積する
- **Obsidian の wiki リンク**（`[[ページ名]]`）で知識をつなぐ

## ディレクトリ構成

| パス | 内容 |
|------|------|
| `問題/` | 過去問ノート（HTML クイズ形式） |
| `用語/` | 用語ページ（学習しながら追加） |
| `用語一覧.md` | シラバス用語の**索引のみ**（ページは先に作らない） |
| `テンプレート/` | ノートのひな形（`過去問ノート.md`・`用語ノート.md`。集計対象外） |

## 前提（用語一覧）

- 解説・用語ノートにリンクする語は **`用語一覧.md` に載っているものだけ**（載っていなければリンクにしない）
- **用語ページは一括作成しない**（ユーザーが学習のタイミングで追加。AI は明示依頼時のみ作成）
- 用語リンクの抽出・照合は **`scripts/check_question_terms.py`**（詳細は下記「スクリプト」）。`PATH` の `問題/…` `用語/…` は**作業中ノートの実パス**に置き換える（README の `…` をそのまま実行しない）

## 過去問ノートの作成・リンク（AI）

**依頼の例:** 「R3春期 問38 の過去問ノートを作って」「解説に用語リンクを付けて」

テンプレート: `テンプレート/過去問ノート.md` → `問題/午前/R3春期/問38.md` のように複製

### 手順

0. **テンプレートを複製** … `問題/午前/R3春期/問38.md` のように年度・問番号のパスに保存
1. **ノートを埋める** … 見出し（ap-siken リンク）・分野タグ・HTML クイズ（下記「HTML 形式」）
2. **解説を書く** … 問題文・各選択肢の `ap-choice-note`・`ap-explanation`（この時点では用語リンクは付けない）
3. **リンク候補の抽出**（ボルトルート）
   ```bash
   python3 scripts/check_question_terms.py --suggest --question 問題/…
   ```
4. **候補の取捨選択**（AI） … 手順3の各語について `用語一覧.md` の当該行（分野見出し・語名）を確認し、**問題文・解説の文脈と一致する試験用語だけ**残す
5. **用語をリンクする**（ボルトルート。手順4で残した語だけ）
   ```bash
   python3 scripts/check_question_terms.py --apply --question 問題/… --terms 語1,語2,語3
   ```
   - 未リンクの出現箇所を HTML internal-link に置換（用語ページ未作成でも可）
   - 確認だけ: 同コマンドに `--dry-run` を付ける
6. **照合**
   ```bash
   python3 scripts/check_question_terms.py --question 問題/…
   ```
   - 一覧に**載っていない語**へのリンクが出たら、**リンクを直す**（誤記なら一覧にある用語名に修正、不要ならリンク削除）。**一覧への追記は判断しない**（ユーザーが依頼したときだけ `build_glossary_index.py` 等で対応）
7. **`用語一覧.md` の `（N）` を更新**（ボルトルート。照合が通ったあと**毎回**実行）
   ```bash
   python3 scripts/count_term_question_links.py
   ```

### AI がやらないこと

- **`用語/○○.md` の新規作成**（用語ページ依頼がない限り）
- 一覧に無い語への **internal-link**（付けない。照合で出たら直す）
- HTML ブロック内の `[[...]]`（効かない）

### HTML 形式

- 種別タグ `#問題` を付ける（テンプレートに含まれる）。分野タグ（例: `#テクノロジ` `#セキュリティ`）を問題に合わせて調整
- 出典は見出し1行を ap-siken リンクにする。別途「出典:」行は付けない
- `<details class="ap-reveal">` 先頭に `<summary>`、**解説は `details` 内**
- **HTML ブロック内に空行を入れない**
- 各選択肢: `ap-choice-text` / `ap-choice-note` / `ap-mistake`（任意）
- テンプレートの形式を守る

## 用語ページの作成・リンク（AI）

**依頼の例:** 「`用語/脆弱性.md` に一覧の語をリンクして」「脆弱性の用語ページを作って」

形式は `テンプレート/用語ノート.md`（穴埋め。先頭説明は保存前に削除）。**リンク**は `## 出典` より前の本文のみ。出典の `[[問題/…]]` は照合対象外。

### 手順（リンクのみ・よくある依頼）

1. 本文・箇条書きがあることを確認（見出し1・`#用語`・`## 出典` はテンプレートどおり）
2. **リンク候補の抽出**（ボルトルート）
   ```bash
   python3 scripts/check_question_terms.py --suggest --term 用語/…
   ```
3. **候補の取捨選択**（AI） … `用語一覧.md` を見て、本文の文脈と一致する語だけ残す
4. **リンクを付与**（ボルトルート）
   ```bash
   python3 scripts/check_question_terms.py --apply --term 用語/… --terms 語1,語2
   ```
5. **照合**
   ```bash
   python3 scripts/check_question_terms.py --term 用語/…
   ```
   - 一覧に**載っていない語**へのリンクが出たら、**リンクを直す**（誤記なら一覧にある用語名に修正、不要ならリンク削除）。一覧への追記はユーザー依頼時のみ

### 手順（作成・明示依頼時のみ）

1. `テンプレート/用語ノート.md` を `用語/用語名.md` に複製し `{{…}}` を埋める（`用語一覧.md` に載る語のみ）
2. 上記「手順（リンクのみ）」の 2〜5 を実行

### AI がやらないこと

- 一覧に無い語への `[[...]]` 追加
- 見出し・出典・タグの不用意な改変（依頼がなければ）
- 依頼なしの用語ページ新規作成
- リンク先未作成を理由にリンクを付けない

用語ページの `[[問題/…]]` や用語同士の `[[リンク]]` は、過去問リンク数の集計（`用語一覧.md` の `（N）`）に**含めない**。

## AI が作業するときのルール（共通）

1. 既存ノートの `[[...]]` と HTML 内 `internal-link` を壊さない
2. 応用情報の正確性を優先。不確かな点は「要確認」と書く
3. `.obsidian/` は編集しない

## Obsidian の慣習

- **リンク**: `[[ページ名]]` または `[[用語/ページ名]]`
- **ファイル名**: リンク先と一致（例: `[[フットプリンティング]]` → `用語/フットプリンティング.md`）

## スクリプト（`check_question_terms.py`）

**ボルトルート**で実行。

| 対象 | `--suggest` | `--apply` | 照合 |
|------|-------------|-----------|------|
| 過去問 | `--question 問題/…` | `--question 問題/… --terms 語1,語2`（internal-link） | `--question 問題/…` |
| 用語 | `--term 用語/…` | `--term 用語/… --terms 語1,語2`（`[[用語名]]`） | `--term 用語/…` |

`--apply` で `--terms` を省略すると、`--suggest` と同じ全候補を付与する（誤リンクが増えやすいので、過去問は **AI が絞った語だけ渡す**運用を推奨）。

終了コード 1 = 一覧にない語へのリンクが残っている（リンクを直して再実行）。用語ページの有無は見ない。

### 過去問リンク後（毎回）

過去問ノートの作成・リンクが終わったら、**必ず** `count_term_question_links.py` を実行して `用語一覧.md` の `（N）` を更新する（用語ノートの `[[リンク]]` は数えない）。

## 用語一覧の再生成

[ap-siken 用語集](https://www.ap-siken.com/keyword/) から全23分野を取得して索引を更新するとき（スクショ不要）:

```bash
python3 scripts/build_glossary_index.py
# 特定分野のみ: python3 scripts/build_glossary_index.py --only 1,11
```

索引を作り直したあと、過去問との結びつき件数を付け直す:

```bash
python3 scripts/count_term_question_links.py
```

- **対象は `問題/**/*.md` のみ**（`#問題` タグあり。`internal-link` の `用語/○○` を数える）
- `用語一覧.md` の行を `- [[用語名]]（2）` のように更新（0 件の語には付けない）
- 確認だけ: `python3 scripts/count_term_question_links.py --dry-run`
