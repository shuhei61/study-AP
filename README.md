# 応用情報（AI向けメモ）

このディレクトリは **応用情報技術者試験（AP）** の学習ノート用 Obsidian ボルトです。人間が読むことも想定していますが、**この README は主に AI エージェント向け**の運用説明です。

## 目的

- 過去問・分野別の知識・用語・解法メモを蓄積する
- **Obsidian の wiki リンク**（`[[ページ名]]`）で知識をつなぐ

## ディレクトリ構成

| パス | 内容 |
|------|------|
| `問題/` | 過去問ノート（HTML クイズ形式） |
| `用語/` | 用語ページ（学習しながら追加） |
| `用語一覧.md` | シラバス用語の**索引のみ**（ページは先に作らない） |
| `テンプレート/` | ノートのひな形（`過去問ノート.md`・`用語ノート.md`。集計対象外） |

## 用語の運用（重要）

1. **`用語一覧.md`**（ボルトルート）に、分野ごとの用語名を `[[用語名]]` で列挙しておく（出典: [ap-siken 用語集](https://www.ap-siken.com/keyword/) など）
2. **用語ページ（`用語/○○.md`）は最初から一括作成しない**（ユーザーが学習のタイミングで追加）
3. 過去問の解説では、一覧に載る語を **HTML の internal-link** でリンクする（下記フロー）
4. 一覧にない語が問題に出た場合は、リンクにせずユーザーに確認するか、一覧への追記を提案する

## 用語ページの形式

テンプレート: `テンプレート/用語ノート.md` → 複製して `用語/用語名.md` に保存（**ユーザーが作成**。AI は明示依頼時のみ）

1. 見出し1: 用語名
2. 見出し直後に種別タグ `#用語`
3. 本文: 定義・要点（箇条書き可）。**他の用語・分野ハブとの関係は本文中の `[[リンク]]` でつなぐ**（末尾に「関連」一覧は付けない）
4. 末尾に **`## 出典` のみ**（「関連」セクションは作らない。過去問と外部URLを二重に書かない）
   - 過去問から作った場合: `[[問題/午前/R3春期/問38]]` のように**問題ノートへのパス**（見出しに ap-siken URL があるので外部URLは不要）
   - 過去問以外の根拠がある場合のみ: 外部リンク（例: [ap-siken 用語集](https://www.ap-siken.com/keyword/) の該当語）

用語ページの `[[過去問]]` や用語同士のリンクは、過去問リンク数の集計には**含めない**（集計は問題ページ側の HTML リンクのみ）。

## 用語ノートのリンク付け（AI）

**用語ページの新規作成はユーザーが行う。** AI は、ユーザーが書いた（または依頼で整える）既存の `用語/○○.md` に対し、**本文中の語**が `用語一覧.md` に載っていれば `[[用語名]]` の wiki リンクを付ける。

| ノート種別 | リンク形式 | 一覧の参照 |
|------------|------------|------------|
| 過去問（`問題/`） | HTML `internal-link`（`用語/○○`） | 用語一覧 |
| 用語（`用語/`） | wiki `[[用語名]]` | 用語一覧 |

### 手順

1. `用語一覧.md` から用語名の一覧を把握する（`- [[用語名]]` 行）
2. 対象の `用語/○○.md` の**本文・箇条書き**を読む（見出し1の用語名・`#用語`・`## 出典` は基本そのまま）
3. 文中に出てくる語のうち、**一覧に載る語**かつ**当該ノートの用語名自身ではない**ものを `[[用語名]]` に置き換える（未リンクのときのみ。既存リンクは壊さない）
4. **一覧に無い語**はリンクにしない。必要ならユーザーに一覧追記を提案
5. **`用語/○○.md` の新規作成はしない**（明示依頼時のみ）

### AI がやらないこと

- 用語一覧に無い語への `[[...]]` 追加
- 見出し・出典・タグの不用意な改変（依頼がなければ）
- リンク先ページが無いことを理由にリンクを付けない（未作成でも wiki リンクは可）

### 依頼の例

- 「この用語ノートに一覧の語をリンクして」
- 「`用語/脆弱性.md` の本文を用語一覧に合わせてリンク付け」

## Obsidian の慣習

- **リンク**: `[[ページ名]]` または `[[用語/ページ名]]`（サブフォルダ内ノート）
- **ファイル名**: リンク先と一致（例: `[[フットプリンティング]]` → `用語/フットプリンティング.md`）
- **`.obsidian/`**: エディタ設定。編集しない

## 過去問ノートの作成フロー（AI）

テンプレート: `テンプレート/過去問ノート.md`（**`#問題` は付けない**。コピー後に `問題/` へ置き、タグを足す）

配置例: `問題/午前/R3春期/問38.md`

### 手順

0. **テンプレートを複製** … `テンプレート/過去問ノート.md` をコピーし、`問題/午前/R3春期/問38.md` のように年度・問番号のパスに保存
1. **問題ノートを埋める** … 見出し（ap-siken リンク）・分野タグ・HTML クイズ（詳細は下記「HTML 形式」）。**最後に `#問題` を1行追加**
2. **解説を書く** … 問題文・各選択肢の `ap-choice-note`・`ap-explanation` に解説文を入れる
3. **用語をリンクする** … 解説文中の語で **`用語一覧.md` に載っているもの**は、必ず次の HTML 形式にする（`[[...]]` は HTML ブロック内では使わない）

```html
<a href="用語/用語名" class="internal-link" data-href="用語/用語名">用語名</a>
```

4. **用語ページの有無を確認** … リンクした各語について `用語/用語名.md` が存在するかチェック
   - **1つでも無ければ**: 不足している用語名をユーザーに伝え、**問題ノートの作成・更新はここで止める**（未作成のままリンクだけ貼って終わらない）
   - **すべてある場合**: 問題ノートの保存まで完了してよい
5. **（ユーザー作業）** 不足用語のページを作成したあと、AI が問題ノートの作業を再開するか、ユーザーが続きを依頼する
6. **（任意・ユーザー）** ボルトルートで `python3 scripts/count_term_question_links.py` を実行し、一覧の `（N）` を更新

### AI がやらないこと

- **用語ページ（`用語/○○.md`）の新規作成** … ユーザーが明示的に依頼した場合のみ
- 一覧に無い語への **internal-link の追加**（先に一覧追記の確認を取る）
- 用語ページが無い状態での問題ノート完成扱い

### 一覧・ページの対応

| 状態 | AI の扱い |
|------|-----------|
| 一覧にあり + `用語/○○.md` あり | HTML リンクを貼って問題ノートを完成してよい |
| 一覧にあり + `用語/○○.md` なし | ユーザーに通知して**終了** |
| 一覧にない | リンクにしない。一覧追記要否をユーザーに確認 |

## AI が作業するときのルール（共通）

1. **過去問**は `問題/` に置く（作成は「過去問ノートの作成フロー」）
2. **用語ページ**は原則ユーザーが作る。AI は依頼時のみ `用語/` に新規追加。既存ノートへの一覧語リンクは「用語ノートのリンク付け」
3. 既存ノートの `[[...]]` と HTML 内 `internal-link` を壊さない
4. 応用情報の正確性を優先。不確かな点は「要確認」と書く

## 過去問ノートの HTML 形式

- 種別タグ `#問題` を必ず付ける。分野タグ（例: `#テクノロジ` `#セキュリティ`）も併用可。「分類: テクノロジ系 » …」行は付けない
- 出典は見出し1行をリンクにする（例: `# [令和3年春期 午前 問38](https://www.ap-siken.com/kakomon/03_haru/q38.html)`）。別途「出典:」行は付けない
- 表示切替: `<details class="ap-reveal">` の先頭に `<summary>`、**解説は `details` 内**に書く。`.ap-quiz` の flex `order` で見た目は **ボタン → 問題文 → 選択肢 → 解説**
- 解説の開閉は `details` のネイティブ動作。○×・一行メモは `ap-quiz` スニペット + `:has(.ap-reveal[open])`
- **HTML ブロック内に空行を入れない**（入れると CSS が効かなくなる）
- 各選択肢: `ap-choice-text` / `ap-choice-note`（一行メモ）/ `ap-mistake`（任意・下線）
- HTML 内の用語リンク（`[[...]]` は効かない）:

```html
<a href="用語/用語名" class="internal-link" data-href="用語/用語名">用語名</a>
```

- 過去問ノート末尾に「関連用語」一覧は付けない

## 用語一覧の再生成

[ap-siken 用語集](https://www.ap-siken.com/keyword/) から全23分野を取得して索引を更新するとき（スクショ不要）:

```bash
python3 scripts/build_glossary_index.py
# 特定分野のみ: python3 scripts/build_glossary_index.py --only 1,11
```

索引を作り直したあと、過去問との結びつき件数を付け直す（**ボルトルート**で実行）:

```bash
cd "/Users/riki/obsidian/応用情報"
python3 scripts/count_term_question_links.py
```

- **対象は `問題/**/*.md` のみ**（`#問題` タグあり。`_` 始まりフォルダ／ファイルは除外）。`テンプレート/` は走査しない。`internal-link`（`用語/○○`）を数える
- **数えないもの**: 用語ページの `## 出典` の `[[過去問]]`、用語間の `[[リンク]]`、`用語一覧.md` 自体
- `用語一覧.md` の行を `- [[用語名]]（2）` のように更新（0 件の語には付けない）
- 確認だけ: `python3 scripts/count_term_question_links.py --dry-run`
