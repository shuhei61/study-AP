# 応用情報（AI向けメモ）

このディレクトリは **応用情報技術者試験（AP）** の学習ノート用 Obsidian ボルトです。人間が読むことも想定していますが、**この README は AI 向けの入口**です。

**運用:** 作業のたびに **この README（共通ルールまで）を読む**。依頼に応じて `docs/ai/` のプレイブックを**追加で1本**読む（過去問 or 用語 or スクリプト）。

## 目的

- 過去問・分野別の知識・用語・解法メモを蓄積する
- **Obsidian の wiki リンク**（`[[ページ名]]`）で知識をつなぐ

## ディレクトリ構成

| パス | 内容 |
|------|------|
| `問題/` | 過去問ノート（HTML クイズ形式）。ファイル名は **問番号のみ**（例: `問題/午前/R3春期/38.md`。`問38.md` にしない） |
| `assets/問題/` | 過去問の図（`fetch_question_figures.py` が `問題/` と同じ階層で保存） |
| 試験回・ap-siken URL | [`docs/ai/試験回とパス.md`](docs/ai/試験回とパス.md)（`R3春期`↔`03_haru`、`R3秋期`↔`03_aki` など） |
| `用語/` | 用語ページ（学習しながら追加） |
| `用語一覧.md` | シラバス用語の**索引のみ**（ページは先に作らない） |
| `タグ一覧.md` | 過去問の**分野タグ**のみ（`用語一覧` の見出しと ap-siken「分類」に対応。語の索引は見ない） |
| `テンプレート/` | ノートのひな形（集計対象外） |
| `docs/ai/` | **作業手順のプレイブック**（依頼ごとに追加で読む） |
| `docs/prompts/` | **再利用プロンプト**（過去問のサブエージェント並列作成など） |

## 共通ルール（毎回）

### 用語一覧・タグ一覧

- 解説・用語ノートにリンクする語は **`用語一覧.md` に載っているものだけ**（載っていなければリンクにしない）
- 過去問の分野タグは **`タグ一覧.md`** を参照（ap-siken の「分類」行 → Obsidian タグ。毎回 `用語一覧.md` 全体は読まない）
- **`build_tag_index.py` は実行しない**（`タグ一覧.md` の再生成はユーザーが**明示依頼したときだけ**）
- **用語ページは一括作成しない**（ユーザーが学習のタイミングで追加。AI は明示依頼時のみ作成）
- 用語リンクの流れ: `scripts/check_question_terms.py` の `--suggest` → AI が文脈で取捨 → `--apply --terms` → 照合
- **`build_glossary_index.py` は実行しない**（ユーザーが一覧の再生成を**明示依頼したときだけ**。照合で一覧外リンクが出ても、AI が勝手に一覧を更新しない）
- コマンドの `問題/…` `用語/…` は**作業中ノートの実パス**に置換する（`…` をそのまま実行しない）

### スクリプト実行

- 形式: `python3 scripts/<名前>.py …`（**1コマンド1行**。`cd`・`&&` でつなげない）
- 例: `python3 scripts/fetch_question_figures.py --apply --question 問題/午前/R3春期/16.md`
- ap-siken の本文取得は **WebFetch 等のみ**。`fetch_question_figures.fetch_html` をワーカーが import して HTML をパースしない
- allowlist: [`docs/ai/スクリプト.md`](docs/ai/スクリプト.md) / [`.cursor/rules/scripts-cli.mdc`](.cursor/rules/scripts-cli.mdc)

### 作業中

1. 既存ノートの `[[...]]` と HTML 内 `internal-link` を壊さない
2. 応用情報の正確性を優先。不確かな点は「要確認」と書く
3. `.obsidian/` は編集しない

### Obsidian の慣習

- **リンク**: `[[ページ名]]` または `[[用語/ページ名]]` / `[[問題/…/番号]]`
- **ファイル名**: リンク先と一致（例: `[[フットプリンティング]]` → `用語/フットプリンティング.md`）
- **過去問**: 見出し・HTML の「問38」表記は試験どおり。ファイル名・出典の wiki は番号のみ（`38.md` / `[[38]]` または `[[問題/午前/R3春期/38]]`）

## プレイブック（依頼時に追加で読む）

| 依頼・作業 | 読むファイル |
|------------|----------------|
| 過去問ノートの作成・解説・用語リンク | [`docs/ai/過去問ノート.md`](docs/ai/過去問ノート.md) |
| 試験回・ap-siken URL・assets パス | [`docs/ai/試験回とパス.md`](docs/ai/試験回とパス.md) |
| 用語ページの作成・本文リンク | [`docs/ai/用語ページ.md`](docs/ai/用語ページ.md) |
| スクリプトの詳細（図取得・用語リンク・集計。一覧再生成はユーザー依頼時のみ） | [`docs/ai/スクリプト.md`](docs/ai/スクリプト.md) |

例: 「R3春期 問38 を作って」→ README + `過去問ノート.md`（保存先は `問題/午前/R3春期/38.md`）。「R3秋期 問1」→ 同手順で `問題/午前/R3秋期/1.md`、URL は `03_aki`（[`試験回とパス.md`](docs/ai/試験回とパス.md)）。「脆弱性にリンクして」→ README + `用語ページ.md`。
