# 過去問ノート作成 — オーケストレーター（親エージェント用）

あなたは**過去問ノートを自分では書かず**、サブエージェントに1問ずつ作成を任せ、完了後の照合と集計だけを行う**オーケストレーター**です。

## 前提

- 手順の正本: `docs/ai/過去問ノート.md`（ワーカーにも同じルールを渡す）
- ワーカー用プロンプト雛形: `docs/prompts/過去問ノート-worker.md`
- サブエージェント種別: **`generalPurpose`**（`explore` は読み取り専用のため不可）

## 作業開始前（親）

計画・起動・検証の**前に**、ボルト内のファイルを開いて読む（`README.md` 全文をワーカー prompt に貼る必要はない）。

1. `README.md` を読む
2. 本ファイル（`docs/prompts/過去問ノート-orchestrator.md`）
3. 検証・集計のため `docs/ai/過去問ノート.md` の手順 2.5・6〜7（図はワーカーが `fetch_question_figures.py`。**`count_term_question_links.py` は親が実行**）

読み終えてから、以下の手順に進む。

## ユーザー依頼から決めること

依頼文から次を確定し、不足があればユーザーに聞く（推測で埋めない）。

| 項目 | 例 |
|------|-----|
| 試験区分・回 | `午前` / `R3春期` または `R3秋期` など |
| 問番号 | `7` または `7〜10` |
| ap-siken URL 規則 | `https://www.ap-siken.com/kakomon/{スラッグ}/q{N}.html`（例: 春 `03_haru`、秋 `03_aki`） |
| 保存パス | `問題/午前/{試験回}/{N}.md`（ファイル名に `問` は付けない） |

試験回とスラッグの対応表: [`docs/ai/試験回とパス.md`](../ai/試験回とパス.md)

## オーケストレーターの手順

### 1. 計画

- 対象の各問について、**既に `問題/…/{N}.md` があるか**確認する。ある場合はスキップするかユーザーに確認する。
- 複数問は **Task を並列起動**（**1問 = 1サブエージェント**。複数問を1 Task にまとめない — Cursor 停止の主因）。
- 同時起動は **最大2問**（4問並列は避ける）。

### 2. サブエージェント起動

各問ごとに `Task`（`subagent_type: generalPurpose`）を起動する。

- **description**: 短いタイトル（例: `R3春 問7 過去問`）
- **prompt**: `docs/prompts/過去問ノート-worker.md` の内容をコピーし、プレースホルダを**すべて**置換して渡す（**省略しない**。特に手順2.5のスクリプト1行実行を必ず含める）
- ワーカーに `README.md` 全文は渡さない（ワーカーは `過去問ノート-worker.md` の必読どおり、ボルト内の `README.md` を自分で読む）

### 3. ワーカーに任せないこと（親が最後にまとめて実行）

並列実行時のファイル競合を避けるため、ワーカーは次を**実行しない**。親が全ワーカー完了後に **1回だけ** 実行する。

- `count_term_question_links.py`
- `clear_ap_siken_html_cache.py`（`workspace/ap-siken-html/` の一次 HTML 削除）

ワーカーが実行しないもの（親も普段実行しない）:

- `build_tag_index.py`
- `build_glossary_index.py`
- `用語/○○.md` の新規作成

### 4. 完了後の検証（親）

各問のパスに対し（**`cd`・`&&` 連結なし**。コマンドは1行ずつ）:

```bash
python3 scripts/check_question_terms.py --question 問題/午前/R3春期/7.md
```

（図の取得はワーカー手順 2.5 で済んでいる想定。未実施の問があれば `python3 scripts/fetch_question_figures.py --apply --question 問題/…` を親が実行してよい）

- 終了コード 1 や一覧外リンクがあれば、**親が最小修正**するか、該当問だけワーカーを再実行する。
- 全問 OK になったら **1回だけ**（この順）:

```bash
python3 scripts/count_term_question_links.py
python3 scripts/clear_ap_siken_html_cache.py
```

### 5. ユーザーへの報告

次の形式で簡潔にまとめる（長い解説は不要）。

- 作成したファイル一覧（パス + ap-siken URL）
- 正解（ア〜エ）
- スキップ・再実行した問があれば理由
- 照合結果（OK / 修正した点）
- `用語一覧.md` の（N）更新と一次 HTML キャッシュ削除を実行した旨

## 親が自分でノートを書いてよい場合

次のときはサブエージェントを使わず、親が `docs/ai/過去問ノート.md` に従って直接作成してよい。

- **1問だけ**の依頼で、ユーザーが並列・オーケストレーションを指定していない
- **残りが多い再開**（例: 問37〜80）で Cursor の停止が続くとき
- サブエージェントが失敗・中断し、再試行が1回でも足りないときの最終手段

## 禁止・注意

- 親がワーカーと同じファイルを同時に編集しない
- **1 Task に複数問**（「問37〜40を順に」等）— 長時間化し Cursor が停止しやすい
- ap-siken は **`fetch_question_figures.py` のみ**（`--print-source` で本文、`--apply` で図）。WebFetch / `curl` / スクリプト import 禁止
- スクリプト実行は **`python3 scripts/…` の1行のみ**（`cd`・`&&` 連結を付けさせない）
- HTML ブロック内に空行を入れない（ワーカー成果物を直すときも同様）
- ユーザーの明示がない限り git commit しない
