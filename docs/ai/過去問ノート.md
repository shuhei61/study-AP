# 過去問ノートの作成・リンク（AI）

**依頼の例:** 「R3春期 問38 の過去問ノートを作って」「解説に用語リンクを付けて」

テンプレート: `テンプレート/過去問ノート.md` → `問題/午前/R3春期/38.md` のように複製（ファイル名に `問` は付けない）。

共通ルール: ボルトルート `README.md`。スクリプト詳細: `docs/ai/スクリプト.md`

## ap-siken ページの読み方（作成時）

- 依頼 URL（または見出し用の ap-siken 問題ページ）を **WebFetch 等で自分で開いて読む**（**1回で足りる。再取得のためシェル HTTP は使わない**）
- **禁止（ap-siken 取得）:** `curl` / `wget` / `urllib.request` / `requests` 等、HTML パース用の `python3 -c`、ネットワーク許可を伴うスクレイピング（実質 `curl` と同じ扱い）
- 図・表が画像だけで WebFetch に数値が無いとき: **解説テキストにある範囲だけ**使う。それ以外は「元ページの図を参照」「要確認」（**推測で表の数値を埋めない**）
- そこから拾うもの: **問題文**・**分類**・**解説**・各選択肢の説明（ページにあれば）
- 選択肢が画像のみなどでテキストが取れない部分は、注記を足すか要確認と明記（推測で補うときはその旨を書く）

## 手順

0. **テンプレートを複製** … `問題/午前/R3春期/38.md` のように `問題/{試験区分}/{年度・回}/{問番号}.md` に保存
1. **ノートを埋める** … 上記で読んだ内容から、見出し（ap-siken リンク）・分野タグ・HTML クイズ（下記「HTML 形式」）
   - **分野タグ**: ap-siken の **分類** 行（`テクノロジ系 » 基礎理論 » 離散数学` など）を読み、**`タグ一覧.md`** で Obsidian タグに変換して `#問題` の直後に付ける（`用語一覧.md` は語の索引用。タグ決定に全文は読まない）
2. **解説を書く** … 問題文・各選択肢の `ap-choice-note`・`ap-explanation`（この時点では用語リンクは付けない）
   - **`ap-explanation`**: [ap-siken](https://www.ap-siken.com/) の解説に沿う（独自の言い換え・真理値表の数値列の追加などはしない。図は手順2.5で埋め込むまで「元ページの図を参照」等の注記でよい）
   - **`ap-choice-note`**: 元 HTML に各選択肢の説明があればそれを参考にする。なければ下記「`ap-choice-note` の書き分け」で一行を書く（この時点では用語リンクは付けない）
2.5. **図の取得・埋め込み**（ボルトルート。**毎回**実行。ワーカーが curl 要否を判断する必要はない）
   ```bash
   python3 scripts/fetch_question_figures.py --apply --question 問題/…
   ```
   - スクリプトが ap-siken HTML を解析し、`assets/問題/…/{問番号}/` に PNG を保存し、`ap-question` / `ap-choice-text` / `ap-explanation` に `class="ap-fig"` の `<img>` を挿入する
   - **図なし**と出たらスキップ（テキストのみの問題）
   - 確認だけ: `--apply` を付けずに実行（検出一覧のみ）。書き込み前確認: `--apply --dry-run`
3. **リンク候補の抽出**（ボルトルート）
   ```bash
   python3 scripts/check_question_terms.py --suggest --question 問題/…
   ```
4. **候補の取捨選択**（AI） … 手順3の各語について `用語一覧.md` の当該行（分野見出し・語名）を確認し、**問題文・解説の文脈と一致する試験用語だけ**残す
5. **用語をリンクする**（ボルトルート。手順4で残した語だけ）
   ```bash
   python3 scripts/check_question_terms.py --apply --question 問題/… --terms 語1,語2,語3
   ```
   - 未リンクの出現箇所を HTML internal-link に置換（**`ap-choice-note`・`ap-explanation` 内**。問題文・選択肢本文は触らない）
   - 確認だけ: 同コマンドに `--dry-run` を付ける
6. **照合**
   ```bash
   python3 scripts/check_question_terms.py --question 問題/…
   ```
   - 一覧に**載っていない語**へのリンク、または**問題文・選択肢本文**の internal-link が出たら、**リンクを直す**（誤記なら一覧にある用語名に修正、不要ならリンク削除）。**`build_glossary_index.py` は実行しない**（一覧の再生成はユーザーが明示依頼したときだけ）
7. **`用語一覧.md` の `（N）` を更新**（ボルトルート。照合が通ったあと**毎回**実行）
   ```bash
   python3 scripts/count_term_question_links.py
   ```

## AI がやらないこと

- **手動のシェル HTTP で ap-siken を取得**（`curl` / `wget` / HTML スクレイピング用 `python3 -c` 等。本文作成時の取得は WebFetch 等のみ。**図は `fetch_question_figures.py` に任せる**）
- **`用語/○○.md` の新規作成**（用語ページ依頼がない限り）
- **`build_tag_index.py` の実行**（`タグ一覧.md` の再生成はユーザー明示依頼時のみ。タグ付けは既存の `タグ一覧.md` を参照）
- 一覧に無い語への **internal-link**（付けない。照合で出たら直す）
- HTML ブロック内の `[[...]]`（効かない）

## `ap-choice-note` の書き分け

その肢の note で「何を伝えるか」で形式を決める（特定の過去問番号ではなく、毎回この基準で判断する）。

### A. `これは<a>用語名</a>の説明です`（正解肢は `正しい。詳細：<a>用語名</a>`）

**使う条件:** その選択肢は、**別の用語・概念の定義そのもの**を問っている（肢の内容＝「それは何か」）。

- 誤肢: 用語一覧にある語で、その語の意味を知れば「なぜ正解でないか」が分かる
- 正解肢: 正解の用語（一覧にある場合）を同形式で示す

**例（イメージ）:** 正解がフットプリンティングなのに、アの肢が「Web改ざんで主張を発信」→ note は **これはハクティビズムの説明です**

### B. `詳細：<a>用語名</a>`

**使う条件:** 肢のラベルは用語名だが、**その用語の定義を一行で述べる形にならない**。note で補うべきは「その用語を調べる入口」。

- 肢は演算名・専門語の羅列などで、誤りは「本問の条件を満たさない」ことが主因
- 「本問の正解条件ではない」だけ書いても、**肢に書かれた用語自体**は学べない → `詳細：` で用語ページへ誘導する

**例（イメージ）:** 「排他的論理和の相補演算はどれか」でイが「否定論理和」→「相補演算ではない」だけでは NAND を学べない → **詳細：否定論理和**

### C. 平文の一行（リンクは文脈上必要な語だけ）

**使う条件:** 上記 A・B に当てはまらない。

- 誤りの理由・本問との関係が主で、特定用語の定義紹介が主目的ではない
- 複数要素の比較（「○○と同様の事前準備だが該当しない」など）

## HTML 形式

- 種別タグ `#問題` を付ける（テンプレートに含まれる）。分野タグは ap-siken の分類に合わせ `タグ一覧.md` から**実名**を付与（テンプレの `#大分類` `#中分類` `#小分類` はプレースホルダ。例: `#テクノロジ` `#基礎理論` `#離散数学`）
- 出典は見出し1行を ap-siken リンクにする。別途「出典:」行は付けない
- `<details class="ap-reveal">` 先頭に `<summary>`、**解説は `details` 内**
- **HTML ブロック内に空行を入れない**
- 各選択肢: `ap-choice-text` / `ap-choice-note` / `ap-mistake`（任意）
- **用語リンクの位置**: `ap-choice-note` と `ap-explanation`（`--apply` / 集計も同様）。**付けない**: `ap-question`・`ap-choice-text`
- テンプレートの形式を守る
