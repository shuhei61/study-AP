# 試験回とパス（過去問ノート）

ボルト内の保存先と [ap-siken](https://www.ap-siken.com/) の URL・画像パスの対応。**試験回ごとにスラッグが違う**ので、依頼・ノート作成前にここで確認する。

共通の作成手順は [`過去問ノート.md`](過去問ノート.md)。並列作成は [`docs/prompts/過去問ノート-orchestrator.md`](../prompts/過去問ノート-orchestrator.md)。

## 命名の二系統

| 使う場所 | 春期の例 | 秋期の例 | 備考 |
|----------|----------|----------|------|
| **ボルト（ディレクトリ・見出し）** | `R3春期` / 令和3年春期 | `R3秋期` / 令和3年秋期 | 日本語。URL スラッグは使わない |
| **ap-siken（URL パス）** | `03_haru` | `03_aki` | `haru`＝春、`aki`＝秋。年号＋季節（下表参照） |

**混同しないこと:** フォルダ名を `03_aki` にしない。ap-siken の URL に `R3秋期` は出てこない。

**スラッグの読み方**

- 令和 `Rn` … 下2桁＋`_haru` / `_aki`（例: R3春 → `03_haru`）
- 平成 `Hnn` … 下2桁＋`_haru` / `_aki`（例: H30秋 → `30_aki`）
- 一覧ページ: `https://www.ap-siken.com/kakomon/{スラッグ}/`

**ap-siken にない回（本表に載せない）**

- 令和元年春期・令和2年春期 … サイト上は `H31春期`（`31_haru`）までが平成31年春期として掲載
- 新しい回を追加するときは [ap-siken 年度別過去問](https://www.ap-siken.com/) でスラッグを確認してから行を足す

## パスの規則（全試験回共通）

### 問題ページ URL

```
https://www.ap-siken.com/kakomon/{スラッグ}/q{問番号}.html
```

### ノート

```
問題/午前/{TERM_DIR}/{問番号}.md
```

見出し1行目（`N`＝問番号、`{EXAM_LABEL}`・`{スラッグ}`は上表）:

```markdown
# [{EXAM_LABEL} 午前 問{N}](https://www.ap-siken.com/kakomon/{スラッグ}/q{N}.html)
```

例（R3秋期 問1）:

```markdown
# [令和3年秋期 午前 問1](https://www.ap-siken.com/kakomon/03_aki/q1.html)
```

### 図（assets）

```
assets/問題/午前/{TERM_DIR}/{問番号}/{画像ファイル名}
```

`fetch_question_figures.py` は見出し URL から取得。画像は `…/kakomon/{スラッグ}/img/` 配下（例: R3秋 問1 → `01.png`、R3春 問71 → `71.png`）。

ノートからの相対パス例:

```html
<img class="ap-fig" src="../../../assets/問題/午前/R3秋期/1/01.png" …>
```

問番号 `1` に対し ap-siken 側の `1.png` と `01.png` の**両方**を同一問の図として扱う（スクリプト変更不要）。

```bash
python3 scripts/fetch_question_figures.py --apply --question 問題/午前/R3秋期/1.md
```

### ap-siken HTML のブロック（参照用）

| 領域 | HTML の目印 | 用途 |
|------|-------------|------|
| 問題文 | `<div id="mondai">` | 問題・問題図 |
| 選択肢 | `<div class="ansbg">` 内 `selectList` | 肢テキスト or 選択肢表画像 |
| 解説 | `<div id="kaisetsu">`（または同等） | 解説文・解説図 |

数式だけの問題は PNG ではなく `<sub>` / `<span class="frac">` の HTML のことがある（図なしで正しい）。
