# スクリプト（AI）

コマンドは次の形式（**1行ずつ**。`cd`・`&&` 連結は使わない）:

```bash
python3 scripts/<スクリプト名>.py [オプション]
```

例:

```bash
python3 scripts/fetch_question_figures.py --apply --question 問題/午前/R3秋期/48.md
```

## Cursor でボルト用スクリプトを自動許可する

以下は自動承認されるので、実行時は **`python3 scripts/...` の形**を守ること。

```json
{
  "terminalAllowlist": [
    "python3 scripts/check_question_terms.py",
    "python3 scripts/count_term_question_links.py",
    "python3 scripts/fetch_question_figures.py",
    "python3 scripts/build_tag_index.py",
    "python3 scripts/build_glossary_index.py"
  ]
}
```

## `fetch_question_figures.py`

過去問ノートの見出し URL から ap-siken の図を取得し、`assets/問題/{試験パス}/{問番号}/` に保存して HTML に埋め込む。

```bash
python3 scripts/fetch_question_figures.py --question 問題/…
python3 scripts/fetch_question_figures.py --apply --question 問題/…
python3 scripts/fetch_question_figures.py --apply --dry-run --question 問題/…
```

- **図なし** … 終了コード 0（テキストのみの問題）。ワーカーが「curl するか」を判断しない
- **選択肢表（4肢まとめ）** … ap-siken の `selectList` 内 `{問番号}a.png` など。`ap-question` 末尾（問題図の直後）に表を埋め込み。`ap-choice-text` はア〜エのラベルのみ（組合せの転記は不要）
- **`--apply` なし** … 検出だけ。埋め込みは `--apply`
- **AI は ap-siken を `curl` 等で取得・検証しない**（図の有無・枚数もこのスクリプトの出力だけで確認。取得失敗時はスクリプトを直す）
- 画像は `assets/問題/午前/R3春期/14/14.png` のように `問題/` と同じ階層。ノートからは相対パス `../../../assets/問題/…`
- 過去問作成の手順 2 のあと（用語リンクの前）に **毎回** `--apply` を実行（`docs/ai/過去問ノート.md` 手順 2.5）

## `check_question_terms.py`

| 対象 | `--suggest` | `--apply` | 照合 |
|------|-------------|-----------|------|
| 過去問 | `--question 問題/…` | `--question 問題/… --terms 語1,語2`（`ap-choice-note`・`ap-explanation` 内） | `--question 問題/…` |
| 用語 | `--term 用語/…` | `--term 用語/… --terms 語1,語2`（`[[用語名]]`） | `--term 用語/…` |

- `--apply` で `--terms` を省略すると、`--suggest` と同じ全候補を付与する（過去問は **AI が絞った語だけ渡す**運用を推奨）
- 過去問の `--suggest` / `--apply` は **`ap-choice-note`・`ap-explanation` 内だけ**（問題文 `ap-question`・選択肢本文 `ap-choice-text` には付けない）。既存 `<a>` 内の部分一致置換もしない
- 照合で **問題文・選択肢本文に internal-link** があると終了コード 1（削除するか note / 解説へ移す）
- 終了コード 1 = 一覧にない語へのリンク、または解説外リンクが残っている（リンクを直して再実行）。用語ページの有無は見ない

### 過去問リンク後（毎回）

過去問ノートの作成・リンクが終わったら、**必ず** `count_term_question_links.py` を実行して `用語一覧.md` の `（N）` を更新する（用語ノートの `[[リンク]]` は数えない）。

## `count_term_question_links.py`

- **対象は `問題/**/*.md` のみ**（`#問題` タグあり。`internal-link` の `用語/○○` を数える）
- `用語一覧.md` の行を `- [[用語名]]（2）` のように更新（0 件の語には付けない）
- 確認だけ: `python3 scripts/count_term_question_links.py --dry-run`

## `build_tag_index.py`（普段は使わない）

**AI はユーザーから明示的に依頼があったとき以外、実行しない。** 過去問作成時は既存の `タグ一覧.md` を読んでタグを付けるだけ。

ユーザーが「タグ一覧を再生成して」等と依頼したときだけ、`用語一覧.md` の見出しから `タグ一覧.md` を作り直す:

```bash
python3 scripts/build_tag_index.py
```

`用語一覧.md` を `build_glossary_index.py` で作り直したあとは、ユーザーが依頼した場合に続けて実行する。

## `build_glossary_index.py`（普段は使わない）

**AI はユーザーから明示的に依頼があったとき以外、実行しない。** 過去問・用語のリンク作業や照合の過程で一覧に無い語が出ても、このスクリプトで一覧を更新しない（リンクの修正・削除で対応する）。

ユーザーが「用語一覧を再生成して」等と依頼したときだけ、[ap-siken 用語集](https://www.ap-siken.com/keyword/) から索引を取得して更新する:

```bash
python3 scripts/build_glossary_index.py
# 特定分野のみ: python3 scripts/build_glossary_index.py --only 1,11
```

索引を作り直したあと:

```bash
python3 scripts/count_term_question_links.py
```
