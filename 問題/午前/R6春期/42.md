#問題 #テクノロジ #ネットワーク #通信プロトコル

<p class="ap-question"><strong>問42</strong>　PCからサーバに対し，IPv6を利用した通信を行う場合，ネットワーク層で暗号化を行うのに利用するものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　IPsec</div>
<div class="ap-choice-note">正しい。<a href="用語/IPsec" class="internal-link" data-href="用語/IPsec">IPsec</a>はIPを拡張してセキュリティを高め、<a href="用語/改ざん" class="internal-link" data-href="用語/改ざん">改ざん</a>の検知、通信データの暗号化、送信元の<a href="用語/認証" class="internal-link" data-href="用語/認証">認証</a>などを<a href="用語/OSI基本参照モデル" class="internal-link" data-href="用語/OSI基本参照モデル">OSI基本参照モデル</a>の<a href="用語/ネットワーク層" class="internal-link" data-href="用語/ネットワーク層">ネットワーク層</a>レベル（<a href="用語/TCP/IP" class="internal-link" data-href="用語/TCP/IP">TCP/IP</a>モデルではインターネット層）で提供するプロトコルである。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　PPP</div>
<div class="ap-choice-note">これはPoint-to-Point Protocolの略で、電話回線のダイヤルアップ接続など2点間を接続してデータ通信を行うためのプロトコルである。本問の<a href="用語/ネットワーク層" class="internal-link" data-href="用語/ネットワーク層">ネットワーク層</a>での暗号化には該当しない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　SSH</div>
<div class="ap-choice-note">これはSecure Shellの略で、遠隔地のコンピュータとの間で安全にリモート通信するためのプロトコルである。<a href="用語/ネットワーク層" class="internal-link" data-href="用語/ネットワーク層">ネットワーク層</a>ではなく、概ね<a href="用語/アプリケーション層" class="internal-link" data-href="用語/アプリケーション層">アプリケーション層</a>相当で利用される。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　TLS</div>
<div class="ap-choice-note">これはTransport Layer Securityの略で、通信の暗号化や<a href="用語/デジタル証明書" class="internal-link" data-href="用語/デジタル証明書">デジタル証明書</a>による<a href="用語/改ざん" class="internal-link" data-href="用語/改ざん">改ざん</a>検知などを提供する。名称のとおり<a href="用語/トランスポート層" class="internal-link" data-href="用語/トランスポート層">トランスポート層</a>のプロトコルである。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/IPv6" class="internal-link" data-href="用語/IPv6">IPv6</a>は、<a href="用語/IPv4" class="internal-link" data-href="用語/IPv4">IPv4</a>に代わるものとして策定された次世代版となる通信プロトコルである。<a href="用語/IPv4" class="internal-link" data-href="用語/IPv4">IPv4</a>では2<sup>32</sup>（約42億）個であった<a href="用語/IPアドレス" class="internal-link" data-href="用語/IPアドレス">IPアドレス</a>が、2<sup>128</sup>（約340澗）個まで使えるようになったのが大きな特徴の一つである。それ以外にも、これまでは上位の層で補完しなければならなかったユーザー<a href="用語/認証" class="internal-link" data-href="用語/認証">認証</a>および<a href="用語/パケット" class="internal-link" data-href="用語/パケット">パケット</a>の暗号化をIP層で行う<a href="用語/IPsec" class="internal-link" data-href="用語/IPsec">IPsec</a>（Security Architecture for IP）の機能がサポートされているなど機能強化が図られている。</p>
</div>
