<p class="ap-question"><strong>問49</strong>　JIS X 33002:2017(プロセスアセスメント実施に対する要求事項)の説明として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　組織のプロセスを継続的に改善して品質を高めるための要求事項を規定している。</div>
<div class="ap-choice-note"><a href="用語/JIS Q 9001" class="internal-link" data-href="用語/JIS Q 9001">JIS Q 9001</a>(品質マネジメントシステム－要求事項)の説明です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　組織プロセスの品質を客観的に診断するための要求事項を規定している。</div>
<div class="ap-choice-note">正しい。JIS X 33002の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　プロジェクトの実施に重要で，かつ，影響を及ぼすプロジェクトマネジメントの概念及びプロセスに関する包括的な手引きを規定している。</div>
<div class="ap-choice-note"><a href="用語/JIS Q 21500" class="internal-link" data-href="用語/JIS Q 21500">JIS Q 21500</a>(プロジェクトマネジメントの手引)の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　明確に定義された用語を使用し，ソフトウェアライフサイクルプロセスの共通枠組みを規定している。</div>
<div class="ap-choice-note"><a href="用語/JIS X 0160" class="internal-link" data-href="用語/JIS X 0160">JIS X 0160</a>(ソフトウェアライフサイクルプロセス)の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>JIS X 33002は、組織内のソフトウェア開発プロセスを理解し、診断することを目的とするアセスメントについての要求事項を規定したJIS規格です。<a href="用語/プロセスアセスメント" class="internal-link" data-href="用語/プロセスアセスメント">プロセスアセスメント</a>を実施するための<a href="用語/標準化" class="internal-link" data-href="用語/標準化">標準化</a>された枠組みを提供します。<a href="用語/プロセスアセスメント" class="internal-link" data-href="用語/プロセスアセスメント">プロセスアセスメント</a>は、JIS Q 33001～33019の一式の規格により体系化されていて、その一部を構成する規格という位置付けです。</p>
<p>入力、役割及び責任、<a href="用語/アクティビティ" class="internal-link" data-href="用語/アクティビティ">アクティビティ</a>、出力の4つの要素を中心として、全ての<a href="用語/プロセスアセスメント" class="internal-link" data-href="用語/プロセスアセスメント">プロセスアセスメント</a>に要求される一般要求事項、高度なアセスメントを実施する際に要求される個別要求事項などが定義されています。</p>
<p>「組織プロセスの品質を診断」という記述から「イ」が適切な説明と判断できます。</p>
