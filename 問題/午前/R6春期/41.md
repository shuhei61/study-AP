<p class="ap-question"><strong>問41</strong>　WAFによる防御が有効な攻撃として，最も適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　DNSサーバに対するDNSキャッシュポイズニング</div>
<div class="ap-choice-note"><a href="用語/DNS" class="internal-link" data-href="用語/DNS">DNS</a>サーバに対する攻撃なので対象外です。<a href="用語/DNSSEC" class="internal-link" data-href="用語/DNSSEC">DNSSEC</a>の導入などによって防御します。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　REST APIサービスに対するAPIの脆弱性を狙った攻撃</div>
<div class="ap-choice-note">正しい。<a href="用語/Webサーバ" class="internal-link" data-href="用語/Webサーバ">Webサーバ</a>に対する攻撃なので、<a href="用語/WAF" class="internal-link" data-href="用語/WAF">WAF</a>での防御が有効です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　SMTPサーバの第三者不正中継の脆弱性を悪用したフィッシングメールの配信</div>
<div class="ap-choice-note">メールサーバに対する攻撃なので対象外です。第三者不正中継は、メールサーバの設定で禁止することで防御します。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　電子メールサービスに対する大量，かつ，サイズの大きな電子メールの配信</div>
<div class="ap-choice-note">メールサーバに対する攻撃なので対象外です。本肢の攻撃はメールボム(爆弾)と呼ばれ、スパム<a href="用語/フィルタリング" class="internal-link" data-href="用語/フィルタリング">フィルタリング</a>によって防御します。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/WAF" class="internal-link" data-href="用語/WAF">WAF</a>(Web Application Firewall)は、<a href="用語/Webサーバ" class="internal-link" data-href="用語/Webサーバ">Webサーバ</a>の前段に配置し、通過する<a href="用語/パケット" class="internal-link" data-href="用語/パケット">パケット</a>の<a href="用語/IPアドレス" class="internal-link" data-href="用語/IPアドレス">IPアドレス</a>や<a href="用語/ポート番号" class="internal-link" data-href="用語/ポート番号">ポート番号</a>だけでなくペイロード部(データ部分)をチェックすることで、Webアプリケーションに対する攻撃を検知し、遮断することが可能な<a href="用語/ファイアウォール" class="internal-link" data-href="用語/ファイアウォール">ファイアウォール</a>です。例えば、<a href="用語/Webブラウザ" class="internal-link" data-href="用語/Webブラウザ">Webブラウザ</a>からのリクエスト<a href="用語/パケット" class="internal-link" data-href="用語/パケット">パケット</a>にSQL文の断片が含まれていたら、<a href="用語/SQLインジェクション" class="internal-link" data-href="用語/SQLインジェクション">SQLインジェクション</a>の可能性があるとして遮断するなどです。<a href="用語/WAF" class="internal-link" data-href="用語/WAF">WAF</a>は、<a href="用語/多層防御" class="internal-link" data-href="用語/多層防御">多層防御</a>の一環として、データの「入口」でのセキュリティを高めることを目的として設置されます。</p>
<p><a href="用語/WAF" class="internal-link" data-href="用語/WAF">WAF</a>により防御することができるのは、Webアプリケーションの<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>を狙った攻撃だけです。Webアプリケーションとは、<a href="用語/HTTP" class="internal-link" data-href="用語/HTTP">HTTP</a>(S)通信の仕組みを使い<a href="用語/Webブラウザ" class="internal-link" data-href="用語/Webブラウザ">Webブラウザ</a>と<a href="用語/Webサーバ" class="internal-link" data-href="用語/Webサーバ">Webサーバ</a>のやり取りによって機能を提供するアプリケーションなので、選択肢の中では<a href="用語/REST" class="internal-link" data-href="用語/REST">REST</a> APIが正解となります。<a href="用語/REST" class="internal-link" data-href="用語/REST">REST</a> APIとは、<a href="用語/HTTP" class="internal-link" data-href="用語/HTTP">HTTP</a>(S)を使った<a href="用語/Web API" class="internal-link" data-href="用語/Web API">Web API</a>システムの設計モデルです。</p>
</div>
