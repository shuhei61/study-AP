<p class="ap-question"><strong>問48</strong>　作業成果物の作成者以外の参加者がモデレーターとして主導する役割を受け持つこと，並びに公式な記録及び分析を行うことが特徴のレビュー技法はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　インスペクション</div>
<div class="ap-choice-note">正しい。<a href="用語/インスペクション" class="internal-link" data-href="用語/インスペクション">インスペクション</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　ウォークスルー</div>
<div class="ap-choice-note"><a href="用語/ウォークスルー" class="internal-link" data-href="用語/ウォークスルー">ウォークスルー</a>は、開発者が主体となりエラーの早期発見を目的としてプログラムのステップごとにシミュレーションを行いながら確認をしていく<a href="用語/レビュー" class="internal-link" data-href="用語/レビュー">レビュー</a>技法です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　パスアラウンド</div>
<div class="ap-choice-note">パスアラウンドは、電子メールなどを使って、対象の成果物を複数のレビュアに配布・回覧し、フィードバックを求める<a href="用語/レビュー" class="internal-link" data-href="用語/レビュー">レビュー</a>技法です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　ペアプログラミング</div>
<div class="ap-choice-note"><a href="用語/ペアプログラミング" class="internal-link" data-href="用語/ペアプログラミング">ペアプログラミング</a>は、二人一組で実装を行い、1人が実際のコードをコンピュータに打ち込み、もう1人はそれをチェックしながら補佐するという役割を随時交代しながら開発作業を進める手法です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/インスペクション" class="internal-link" data-href="用語/インスペクション">インスペクション</a>(Inspection)は、事前に役割を決められた参加者が責任のある第三者(<a href="用語/モデレーター" class="internal-link" data-href="用語/モデレーター">モデレーター</a>)の下で成果物を確認する、最も公式な<a href="用語/レビュー" class="internal-link" data-href="用語/レビュー">レビュー</a>技法です。通常3人から6人が参加して実施します。他の<a href="用語/レビュー" class="internal-link" data-href="用語/レビュー">レビュー</a>法との違いは、参加者の役割が明確になっていること、<a href="用語/チェックリスト" class="internal-link" data-href="用語/チェックリスト">チェックリスト</a>など形式的な文書に基づいて実施すること、正式な記録を強調することです。</p>
<p><a href="用語/インスペクション" class="internal-link" data-href="用語/インスペクション">インスペクション</a>の参加者に割り振られる役割には以下の5種類があります。<br><a href="用語/モデレーター" class="internal-link" data-href="用語/モデレーター">モデレーター</a>(Moderator) … 議長・司会として<a href="用語/インスペクション" class="internal-link" data-href="用語/インスペクション">インスペクション</a>全体を運営する<br>オーナー(Owner) … <a href="用語/レビュー" class="internal-link" data-href="用語/レビュー">レビュー</a>対象となる成果物の作成者で、発見された問題に応じて成果物の修正を行う<br>インスペクター(Inspector) … 評価者として<a href="用語/レビュー" class="internal-link" data-href="用語/レビュー">レビュー</a>対象となる成果物の問題発見を行う<br>プレゼンタ(Presenter) … ミーティングにて参加者に資料の説明を行う<br>スクライブ(Scribe) … 書記として<a href="用語/レビュー" class="internal-link" data-href="用語/レビュー">レビュー</a>で発見された問題などを記録する</p>
<p>本問の「作成者以外が<a href="用語/モデレーター" class="internal-link" data-href="用語/モデレーター">モデレーター</a>として主導する」「公式な記録及び分析を行う」という特徴に該当するのは<a href="用語/インスペクション" class="internal-link" data-href="用語/インスペクション">インスペクション</a>です。したがって正解は「ア」です。</p>
</div>
