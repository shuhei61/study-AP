<p class="ap-question"><strong>問57</strong>　ITサービスマネジメントの活動のうち，インシデント管理及びサービス要求管理として行うものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　サービスデスクに対する顧客満足度が合意したサービス目標を満たしているかどうかを評価し，改善の機会を特定するためにレビューする。</div>
<div class="ap-choice-note">これは<a href="用語/サービスレベル管理" class="internal-link" data-href="用語/サービスレベル管理">サービスレベル管理</a>の役割です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　ディスクの空き容量がしきい値に近づいたので，対策を検討する。</div>
<div class="ap-choice-note">これは<a href="用語/キャパシティ管理" class="internal-link" data-href="用語/キャパシティ管理">キャパシティ管理</a>の役割です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　プログラム変更を行った場合の影響度を調査する。</div>
<div class="ap-choice-note">これは<a href="用語/変更管理" class="internal-link" data-href="用語/変更管理">変更管理</a>の役割です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　利用者からの障害報告を受けて，既知の誤りに該当するかどうかを照合する。</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/インシデント管理" class="internal-link" data-href="用語/インシデント管理">インシデント管理</a></div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/インシデント管理" class="internal-link" data-href="用語/インシデント管理">インシデント管理</a>は、サービスに対する計画外の中断、サービスの品質の低下、あるいは顧客へのサービスにまだ影響していない事象が報告された時に、サービスの中断時間を最小限に抑えて速やかに回復することを目指すプロセスです。</p>
<p><a href="用語/ITIL" class="internal-link" data-href="用語/ITIL">ITIL</a>では、<a href="用語/インシデント管理" class="internal-link" data-href="用語/インシデント管理">インシデント管理</a>が独立したプロセスとして定義されていますが、JIS Q 20000-1「第1部 <a href="用語/サービスマネジメントシステム" class="internal-link" data-href="用語/サービスマネジメントシステム">サービスマネジメントシステム</a>要求事項」では、インシデントと<a href="用語/サービス要求" class="internal-link" data-href="用語/サービス要求">サービス要求</a>の両方を合わせた「インシデント及び<a href="用語/サービス要求管理" class="internal-link" data-href="用語/サービス要求管理">サービス要求管理</a>」として定義されています。<a href="用語/サービス要求" class="internal-link" data-href="用語/サービス要求">サービス要求</a>とは、情報，助言，サービスへのアクセス，又は事前に承認されている変更に対する要求です。</p>
<p>アは<a href="用語/サービスレベル管理" class="internal-link" data-href="用語/サービスレベル管理">サービスレベル管理</a>、イは<a href="用語/キャパシティ管理" class="internal-link" data-href="用語/キャパシティ管理">キャパシティ管理</a>、ウは<a href="用語/変更管理" class="internal-link" data-href="用語/変更管理">変更管理</a>の役割です。エは、既知のエラーであれば前回発生時の対応策を採ることで迅速な障害回復が可能となるため、<a href="用語/インシデント管理" class="internal-link" data-href="用語/インシデント管理">インシデント管理</a>及び<a href="用語/サービス要求管理" class="internal-link" data-href="用語/サービス要求管理">サービス要求管理</a>として行う活動です。</p>
