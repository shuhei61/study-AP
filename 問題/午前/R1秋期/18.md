<p class="ap-question"><strong>問18</strong>　ページング方式の仮想記憶において，主記憶に存在しないページをアクセスした場合の処理や状態の順番として，適切なものはどれか。ここで，現在主記憶には，空きページはないものとする。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　置換え対象ページの決定→ページイン→ページフォールト→ページアウト</div>
<div class="ap-choice-note"><a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>が最初に発生しない順序であり、<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>と<a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a>の順も逆です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　置換え対象ページの決定→ページフォールト→ページアウト→ページイン</div>
<div class="ap-choice-note">置換え対象の決定は<a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>の後に行うため、先頭の順序が誤っています。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　ページフォールト→置換え対象ページの決定→ページアウト→ページイン</div>
<div class="ap-choice-note">正しい。空きページがない前提でのページング処理の順序です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　ページフォールト→置換え対象ページの決定→ページイン→ページアウト</div>
<div class="ap-choice-note">置換え対象を退避（<a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a>）してから必要なページを移す（<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>）必要があります。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>ページング方式とは、<a href="用語/仮想記憶管理" class="internal-link" data-href="用語/仮想記憶管理">仮想記憶管理</a>方式の一つで仮想アドレス空間を「ページ」と呼ばれる固定長の区画に分割するとともに<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>のアドレス空間（実アドレス空間）も同じように固定長に分割して、このページ単位で<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>と<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>（<a href="用語/補助記憶" class="internal-link" data-href="用語/補助記憶">補助記憶</a>装置）のアドレス変換を行う方式です。</p>
<p>ページング方式では、ページの置換えに伴う3つの処理を次のように呼びます。</p>
<ul>
<li><a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a> … アクセス要求のあったページが<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>上に存在しない状態</li>
<li><a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a> … ページを<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>から<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>に移すこと</li>
<li><a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a> … ページを<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>から<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>に移すこと</li>
</ul>
<p>ページング処理の順番としては、次のような流れになります。</p>
<ol>
<li><a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>上に必要なデータが存在しない状態が発生する（<a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>）</li>
<li><a href="用語/FIFO" class="internal-link" data-href="用語/FIFO">FIFO</a>や<a href="用語/LRU" class="internal-link" data-href="用語/LRU">LRU</a>アルゴリズムを用いて置換え対象のページを決定する</li>
<li>置換え対象のページを<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>から<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>に退避させる（<a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a>）</li>
<li>実行に必要なページを<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>から<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>に移す（<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>）</li>
</ol>
<p>したがって「ウ」の順番が適切です。</p>
</div>
