# [令和元年秋期 午前 問55](https://www.ap-siken.com/kakomon/01_aki/q55.html)

#問題 #マネジメント #サービスマネジメント

<div class="ap-quiz">
<details class="ap-reveal">

<p class="ap-question"><strong>問55</strong>　ITIL 2011 editionによれば，サービス・パッケージの説明として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　コアサービス，実現サービス及び強化サービスの組合せで構成された，特定の種類の顧客ニーズへのソリューションを提供する複数のサービスの集まりである。</div>
<div class="ap-choice-note">正しい。サービス・パッケージの説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　サービス・パイプライン，サービス・カタログ及び廃止済みサービスで構成された，サービス・プロバイダによって管理されている全てのサービスである。</div>
<div class="ap-choice-note">これは<a href="用語/サービスポートフォリオ" class="internal-link" data-href="用語/サービスポートフォリオ">サービスポートフォリオ</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　成果物，価格，連絡先などが内容として含まれた，稼働中の全てのITサービスに関する情報を格納するデータベース又は構造化された文書である。</div>
<div class="ap-choice-note">これは<a href="用語/サービスカタログ" class="internal-link" data-href="用語/サービスカタログ">サービスカタログ</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　ハードウェア，ソフトウェア，ライセンス，文書などで構成された，稼働中のITサービスに対して承認された変更を実施するためのコンポーネントの集合である。</div>
<div class="ap-choice-note">これは<a href="用語/リリース" class="internal-link" data-href="用語/リリース">リリース</a>の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>サービス・パッケージは、特定のタイプの顧客ニーズにソリューションを提供したり、特定のビジネス成果を支えるために組み合わされた2つ以上のサービスのことです。サービスパッケージは、コアサービス、有効化サービス、および拡張サービスの組み合わせで構成されます。サービスパッケージは、特定のレベルのユーティリティと保証を提供します。</p>
</div>
