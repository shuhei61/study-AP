<p class="ap-question"><strong>問13</strong>　システムの性能を向上させるための方法として，スケールアウトが適しているシステムはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　一連の大きな処理を一括して実行しなければならないので，並列処理が困難な処理が中心のシステム</div>
<div class="ap-choice-note">並列処理が困難なので<a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>は適さない。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　参照系のトランザクションが多いので，複数のサーバで分散処理を行っているシステム</div>
<div class="ap-choice-note">正しい。参照系が中心で並列処理済みのため<a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>が適する。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　データを追加するトランザクションが多いので，データの整合性を取るためのオーバーヘッドを小さくしなければならないシステム</div>
<div class="ap-choice-note">整合性維持のオーバーヘッドを小さくしたいので<a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>は適さない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　同一のマスタデータベースがシステム内に複数配置されているので，マスタを更新する際にはデータベース間で整合性を保持しなければならないシステム</div>
<div class="ap-choice-note">整合性の保持が必要なので<a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>は適さない。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>システムの処理能力を向上させる施策には、「<a href="用語/スケールアップ" class="internal-link" data-href="用語/スケールアップ">スケールアップ</a>」と「<a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>」という2つのアプローチがあります。</p>
<p><a href="用語/スケールアップ" class="internal-link" data-href="用語/スケールアップ">スケールアップ</a>は、サーバを構成する各処理装置をより性能の高いものに交換したり、プロセッサの数などを増やすことでサーバ当たりの処理能力を向上させる。追加・削除・更新処理が頻繁に発生するなど、並列実行による負荷分散が困難なシステムに適している。</p>
<p><a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>は、接続するサーバの台数を増やすことでサーバシステム全体としての処理能力や<a href="用語/可用性" class="internal-link" data-href="用語/可用性">可用性</a>を向上させる。同等の性能アップであれば<a href="用語/スケールアップ" class="internal-link" data-href="用語/スケールアップ">スケールアップ</a>よりも低コストであることが多い。</p>
<p>更新処理（追加や削除も含む）を伴う機能を複数のサーバに負荷分散すると、データの整合性を維持するための処理や<a href="用語/排他制御" class="internal-link" data-href="用語/排他制御">排他制御</a>による遅延が生じてしまい、効果的な性能向上につながりません。このため、更新処理が主要なシステムに対しては<a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>は不向きです。<a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>が適しているのは、更新処理の整合性要件が緩く、個々に独立して並列実行できる処理を扱うシステムです。</p>
</div>
