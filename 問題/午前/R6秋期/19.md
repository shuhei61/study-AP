<p class="ap-question"><strong>問19</strong>　分散開発環境において，各開発者のローカル環境に全履歴を含んだ中央リポジトリの完全な複製をもつことによって，中央リポジトリにアクセスできないときでも履歴の調査や変更の記録を可能にする，バージョン管理ツールはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　Apache Subversion</div>
<div class="ap-choice-note">単一の中央<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>を全員が編集する集中型の<a href="用語/バージョン管理ツール" class="internal-link" data-href="用語/バージョン管理ツール">バージョン管理ツール</a>であり、分散<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>のサポートはありません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　CVS</div>
<div class="ap-choice-note">サーバ上の共有ファイルを全員が編集する集中型であり、分散<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>のサポートはありません。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　Git</div>
<div class="ap-choice-note">正しい。各自が完全な<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>を保有できる分散型の<a href="用語/バージョン管理ツール" class="internal-link" data-href="用語/バージョン管理ツール">バージョン管理ツール</a>です。<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>の完全なコピーをローカル環境に保持できるので、中央<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>にアクセスできないときでも変更履歴を調査したり変更を記録したりできます。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　RCS</div>
<div class="ap-choice-note">ファイル単位で変更を管理する初期の<a href="用語/バージョン管理" class="internal-link" data-href="用語/バージョン管理">バージョン管理</a>システムです。複数人での作業やプロジェクトのサポートがありません。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>Apache <a href="用語/Subversion" class="internal-link" data-href="用語/Subversion">Subversion</a>は、単一の中央<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>を全員が編集する集中型の<a href="用語/バージョン管理ツール" class="internal-link" data-href="用語/バージョン管理ツール">バージョン管理ツール</a>です。分散<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>のサポートはありません。</p>
<p>CVS(Concurrent Versions System)は、ソースコードなどのテキストファイルを共有し、その変更を記録し管理するツールです。サーバ上の共有ファイルを全員が編集する集中型で、分散<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>のサポートはありません。</p>
<p><a href="用語/Git" class="internal-link" data-href="用語/Git">Git</a>(ギット)は、各自が完全な<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>を保有できる分散型の<a href="用語/バージョン管理ツール" class="internal-link" data-href="用語/バージョン管理ツール">バージョン管理ツール</a>です。元々はLinuxのソースコード管理に用いるために開発されたツールですが、現在では多くのプロジェクトのソースコード管理に利用されています。<a href="用語/Git" class="internal-link" data-href="用語/Git">Git</a>では、<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>の完全なコピーをローカル環境に保持することができるので、中央<a href="用語/リポジトリ" class="internal-link" data-href="用語/リポジトリ">リポジトリ</a>にアクセスできないときでも、変更履歴を調査したり変更を記録したりすることなどが可能です。</p>
<p>RCS(Revision Control System)は、ファイル単位で変更を管理する初期の<a href="用語/バージョン管理" class="internal-link" data-href="用語/バージョン管理">バージョン管理</a>システムです。複数人での作業やプロジェクトのサポートがありません。</p>
</div>
