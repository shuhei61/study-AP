<p class="ap-question"><strong>問63</strong>　業務改善の4原則としてのEliminate，Combine，Rearrange，Simplifyは，業務改善を実現する上での視点を示すものである。次の業務改善例のうち，Rearrangeを適用したものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　経費精算申請業務において，決裁に不要な人を申請フローから外し，決裁までに要する時間を短縮した。</div>
<div class="ap-choice-note">Eliminate（排除）の適用例です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　商品配送ルートを可視化した結果，移動距離の無駄が発見された。そこで，配送順を入れ替えることで，一日に配送できる件数を向上させた。</div>
<div class="ap-choice-note">正しい。Rearrange（入替え）の適用例です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　表計算ソフト上で各項目を手入力していたが，一つの項目を入力すれば他の関連項目が自動計算できるように関数を設定し，作業効率を向上させた。</div>
<div class="ap-choice-note">Simplify（簡素化）の適用例です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　複数部門が集まる会議の議事録を，各部門のスタッフが，それぞれ独自に作成していた。各部門での作成をやめ，代表者1名が作成し，部門間で共有した。</div>
<div class="ap-choice-note">Combine（結合）の適用例です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/ECRSの原則" class="internal-link" data-href="用語/ECRSの原則">ECRSの原則</a>（イクルス）は，Eliminate：排除，Combine：結合，Rearrange：入替え，Simplify：簡素化の頭文字を取ったもので，業務改善を進めるために必要な視点と改善順序を示した<a href="用語/フレームワーク" class="internal-link" data-href="用語/フレームワーク">フレームワーク</a>です。文字の並びどおり，排除⇒結合⇒入替え⇒簡素化の順で業務改善を進めるのが良いとされています。</p>
<p>①Eliminate：排除 … 業務<a href="用語/プロセス" class="internal-link" data-href="用語/プロセス">プロセス</a>や作業の中で不要な手順や重複した作業を取り除く。例）不要な承認<a href="用語/プロセス" class="internal-link" data-href="用語/プロセス">プロセス</a>や会議，報告書を削減する。</p>
<p>②Combine：結合 … 類似する作業や<a href="用語/プロセス" class="internal-link" data-href="用語/プロセス">プロセス</a>を組み合わせることで，業務全体を効率化する。例）部署間の類似タスクを統合する，システムを統合する。</p>
<p>③Rearrange：入替え … 業務の順序や<a href="用語/プロセス" class="internal-link" data-href="用語/プロセス">プロセス</a>の流れを見直し，最適な配置や順序に変更する。例）作業工程の順番を入れ替える，配置場所を交換する。</p>
<p>④Simplify：簡素化 … 複雑な業務や<a href="用語/プロセス" class="internal-link" data-href="用語/プロセス">プロセス</a>を可能な限り簡素化し，わかりやすくする。例）システムで自動化する，業務を<a href="用語/標準化" class="internal-link" data-href="用語/標準化">標準化</a>する，書類をテンプレート化する。</p>
</div>
