<p class="ap-question"><strong>問13</strong>　データアクセス方式の一つであるオブジェクトストレージに関する記述のうち，最も適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　ストレージにファイルシステムを構成し，ファイル単位でデータにアクセスする。</div>
<div class="ap-choice-note">ファイルストレージの説明です。<a href="用語/オブジェクトストレージ" class="internal-link" data-href="用語/オブジェクトストレージ">オブジェクトストレージ</a>は階層構造を持たず、ファイル単位ではなくオブジェクト単位でデータを管理します。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　ストレージの論理ボリュームを決まったサイズに分割し，その単位でデータにアクセスする。</div>
<div class="ap-choice-note">これは<a href="用語/ブロックストレージ" class="internal-link" data-href="用語/ブロックストレージ">ブロックストレージ</a>の説明です。<a href="用語/オブジェクトストレージ" class="internal-link" data-href="用語/オブジェクトストレージ">オブジェクトストレージ</a>はブロックに分割せず、オブジェクト単位でデータにアクセスします。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　データとメタデータをセットにして保存し，階層構造にすることによってデータにアクセスする。</div>
<div class="ap-choice-note">データと<a href="用語/メタデータ" class="internal-link" data-href="用語/メタデータ">メタデータ</a>をセットにする点は近いですが、<a href="用語/オブジェクトストレージ" class="internal-link" data-href="用語/オブジェクトストレージ">オブジェクトストレージ</a>は階層構造を持たずフラットな構造で管理されます。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　データとメタデータをセットにして保存し，固有の識別子でデータにアクセスする。</div>
<div class="ap-choice-note">正しい。各オブジェクトにデータ本体・<a href="用語/メタデータ" class="internal-link" data-href="用語/メタデータ">メタデータ</a>・固有の識別子が付与され、その識別子でアクセスします。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/オブジェクトストレージ" class="internal-link" data-href="用語/オブジェクトストレージ">オブジェクトストレージ</a>は、ファイルやディレクトリのような階層構造ではなく、オブジェクトという単位でデータを格納・管理する記憶方式です。フラットな構造で<a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>が容易なので、非構造型データを大量に保存する<a href="用語/データレイク" class="internal-link" data-href="用語/データレイク">データレイク</a>に適しています。Amazon S3が<a href="用語/オブジェクトストレージ" class="internal-link" data-href="用語/オブジェクトストレージ">オブジェクトストレージ</a>の代表格です。</p>
<p>各オブジェクトは、データ本体とともに<a href="用語/メタデータ" class="internal-link" data-href="用語/メタデータ">メタデータ</a>と固有の識別子が付与された状態で格納されています。階層構造がないので、①データの複製、移動、分散がしやすい、②効率的にアクセスできる、③ディレクトリ数やデータ数の制限を受けないといった特徴があります。また、<a href="用語/オブジェクトストレージ" class="internal-link" data-href="用語/オブジェクトストレージ">オブジェクトストレージ</a>には識別子を指定して<a href="用語/REST" class="internal-link" data-href="用語/REST">REST</a> <a href="用語/API" class="internal-link" data-href="用語/API">API</a>(<a href="用語/HTTP" class="internal-link" data-href="用語/HTTP">HTTP</a>)でアクセスします。このため、利用者は格納場所を覚えておく必要がなく、様々なプラットフォームから共通の手順でデータ操作を行うことができます。</p>
</div>
