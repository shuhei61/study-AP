<p class="ap-question"><strong>問50</strong>　レスポンシブWebデザインを実現するに当たって，単一のHTML文書を用いて，Webコンテンツを各種端末のディスプレイの大きさに合わせた形式で表示するために使用する機能はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　User-Agent</div>
<div class="ap-choice-note"><a href="用語/User-Agent" class="internal-link" data-href="用語/User-Agent">User-Agent</a>は、<a href="用語/Webブラウザ" class="internal-link" data-href="用語/Webブラウザ">Webブラウザ</a>が<a href="用語/Webサーバ" class="internal-link" data-href="用語/Webサーバ">Webサーバ</a>に伝える情報で、端末・ブラウザ・OSの種類やバージョンを識別するためのものです。<a href="用語/User-Agent" class="internal-link" data-href="用語/User-Agent">User-Agent</a>でデスクトップとスマートフォンを判別し、それに応じた<a href="用語/HTML" class="internal-link" data-href="用語/HTML">HTML</a>文書を出し分けるスマホ対応もありますが、これは<a href="用語/レスポンシブWebデザイン" class="internal-link" data-href="用語/レスポンシブWebデザイン">レスポンシブWebデザイン</a>ではありません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　WebSocket</div>
<div class="ap-choice-note">WebSocketは、Webアプリケーションにおいて<a href="用語/クライアント" class="internal-link" data-href="用語/クライアント">クライアント</a>（<a href="用語/Webブラウザ" class="internal-link" data-href="用語/Webブラウザ">Webブラウザ</a>）と<a href="用語/Webサーバ" class="internal-link" data-href="用語/Webサーバ">Webサーバ</a>との間で効率的な双方向通信を実現するプロトコルです。主にリアルタイム通信が必要なアプリケーション（チャットやゲームなど）で使われます。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　マッシュアップ</div>
<div class="ap-choice-note">マッシュアップは、Web上に公開されている様々なWebサービスを組み合わせて、新しいWebサービスを作り出す手法です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　メディアクエリ</div>
<div class="ap-choice-note">正しい。<a href="用語/メディアクエリ" class="internal-link" data-href="用語/メディアクエリ">メディアクエリ</a>は、<a href="用語/CSS" class="internal-link" data-href="用語/CSS">CSS</a>の機能のひとつで、デバイスの特性（画面の幅や高さ、解像度、向き）に応じて異なるスタイルを適用することができる仕組みです。同じ<a href="用語/HTML" class="internal-link" data-href="用語/HTML">HTML</a>文書で異なる画面サイズやデバイスに最適化されたレイアウトを提供する<a href="用語/レスポンシブWebデザイン" class="internal-link" data-href="用語/レスポンシブWebデザイン">レスポンシブWebデザイン</a>の中心的な技術です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/レスポンシブWebデザイン" class="internal-link" data-href="用語/レスポンシブWebデザイン">レスポンシブWebデザイン</a>は、単一の<a href="用語/HTML" class="internal-link" data-href="用語/HTML">HTML</a>文書を用いて、各種端末のディスプレイの大きさに合わせた形式で<a href="用語/Webコンテンツ" class="internal-link" data-href="用語/Webコンテンツ">Webコンテンツ</a>を表示する手法です。その中心的な技術が<a href="用語/メディアクエリ" class="internal-link" data-href="用語/メディアクエリ">メディアクエリ</a>です。<a href="用語/メディアクエリ" class="internal-link" data-href="用語/メディアクエリ">メディアクエリ</a>は<a href="用語/CSS" class="internal-link" data-href="用語/CSS">CSS</a>の機能のひとつで、デバイスの特性（画面の幅や高さ、解像度、向き）に応じて異なるスタイルを適用することができます。</p>
<p>例えば、次のように画面幅に応じてフォントサイズを変更できます。</p>
<p>@media (条件) {<br>/* 条件が満たされた場合に適用する<a href="用語/CSS" class="internal-link" data-href="用語/CSS">CSS</a> */<br>}<br>body { font-size: 16px; }<br>/* 画面幅768px以下（主にタブレットやスマートフォン向け）に適用するスタイル */<br>@media (max-width: 768px) {<br>body { font-size: 14px; }<br>}<br>/* 画面幅480px以下（主にスマートフォン向け）に適用するスタイル */<br>@media (max-width: 480px) {<br>body { font-size: 12px; }<br>}</p>
<p>各選択肢については次のとおりです。<br>ア … <a href="用語/User-Agent" class="internal-link" data-href="用語/User-Agent">User-Agent</a>は端末やブラウザを識別する情報です。<a href="用語/HTML" class="internal-link" data-href="用語/HTML">HTML</a>の出し分けによるスマホ対応はありますが、<a href="用語/レスポンシブWebデザイン" class="internal-link" data-href="用語/レスポンシブWebデザイン">レスポンシブWebデザイン</a>ではありません。<br>イ … WebSocketは<a href="用語/クライアント" class="internal-link" data-href="用語/クライアント">クライアント</a>とサーバ間の双方向通信を実現するプロトコルです。ディスプレイサイズに合わせた表示とは関係ありません。<br>ウ … マッシュアップは複数のWebサービスを組み合わせて新しいWebサービスを作る手法です。ディスプレイサイズに合わせた表示とは関係ありません。<br>エ … 正解。<a href="用語/メディアクエリ" class="internal-link" data-href="用語/メディアクエリ">メディアクエリ</a>は<a href="用語/レスポンシブWebデザイン" class="internal-link" data-href="用語/レスポンシブWebデザイン">レスポンシブWebデザイン</a>の中心的な技術です。</p>
</div>
