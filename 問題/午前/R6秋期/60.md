#問題 #マネジメント #システム監査 #システム監査

<p class="ap-question"><strong>問60</strong>　システム監査基準(令和5年)におけるフォローアップの説明として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　監査対象先が，監査報告書の指摘事項及び改善提案を基に改善計画の策定を行うこと</div>
<div class="ap-choice-note"><a href="用語/改善計画" class="internal-link" data-href="用語/改善計画">改善計画</a>の策定は<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>対象部門が主体で行う活動であり，<a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>人が行う<a href="用語/フォローアップ" class="internal-link" data-href="用語/フォローアップ">フォローアップ</a>の説明ではありません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　監査部門の責任者が，監査報告書を基に監査の実施状況と指摘事項の妥当性を確認すること</div>
<div class="ap-choice-note"><a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>部門内での<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>品質管理の話であり，<a href="用語/改善提案" class="internal-link" data-href="用語/改善提案">改善提案</a>の実施状況を<a href="用語/モニタリング" class="internal-link" data-href="用語/モニタリング">モニタリング</a>する<a href="用語/フォローアップ" class="internal-link" data-href="用語/フォローアップ">フォローアップ</a>の説明ではありません。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　システム監査人が，監査報告書に記載した改善提案の実施状況に関する情報を収集し，改善状況をモニタリングすること</div>
<div class="ap-choice-note">正しい。<a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>人が<a href="用語/改善提案" class="internal-link" data-href="用語/改善提案">改善提案</a>の実施状況を収集・<a href="用語/モニタリング" class="internal-link" data-href="用語/モニタリング">モニタリング</a>して改善の実現を支援することを<a href="用語/フォローアップ" class="internal-link" data-href="用語/フォローアップ">フォローアップ</a>といいます。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　システム監査人が，時間の関係で調査が終了しなかった監査項目を追跡調査して報告すること</div>
<div class="ap-choice-note">時間不足で未完了の<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>項目を追跡調査することは，<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>の継続・完了の話であり，<a href="用語/改善提案" class="internal-link" data-href="用語/改善提案">改善提案</a>の実施状況を確認する<a href="用語/フォローアップ" class="internal-link" data-href="用語/フォローアップ">フォローアップ</a>ではありません。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>は、<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>報告書の作成と提出をもって終了しますが、<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>報告書に<a href="用語/改善提案" class="internal-link" data-href="用語/改善提案">改善提案</a>を記載した場合、<a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>人は、当該改善事項が適切かつ適時に実施されているかどうかを確かめておく必要があります。<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>対象部門が策定した<a href="用語/改善計画" class="internal-link" data-href="用語/改善計画">改善計画</a>およびその実施状況に関する情報を収集し、改善状況を<a href="用語/モニタリング" class="internal-link" data-href="用語/モニタリング">モニタリング</a>することで改善の実現を支援することを<a href="用語/フォローアップ" class="internal-link" data-href="用語/フォローアップ">フォローアップ</a>といい、<a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>人の役割のひとつとなっています。</p>
<p>注意点としては、あくまでも支援なので、改善活動は<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>対象部門が主体となって行うということです。<a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>人は改善の実施そのものに責任を負うことはありません。したがって「ウ」が適切な説明です。</p>
</div>
