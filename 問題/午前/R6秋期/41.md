<p class="ap-question"><strong>問41</strong>　JVNなどの脆弱性情報サイトで採用されているCVE(Common Vulnerabilities and Exposures)識別子の説明はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　コンピュータで必要なセキュリティ設定項目を識別するための識別子</div>
<div class="ap-choice-note">これはCCE(Common Configuration Enumeration)の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　脆弱性が悪用されて改ざんされたWebサイトのスクリーンショットを識別するための識別子</div>
<div class="ap-choice-note">該当する識別子は存在しません。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　製品に含まれる脆弱性を識別するための識別子</div>
<div class="ap-choice-note">正しい。<a href="用語/CVE" class="internal-link" data-href="用語/CVE">CVE</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　セキュリティ製品の種別を識別するための識別子</div>
<div class="ap-choice-note">これはCPE(Common Platform Enumeration)の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/CVE" class="internal-link" data-href="用語/CVE">CVE</a>(共通<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>識別子)は、<a href="用語/情報セキュリティ" class="internal-link" data-href="用語/情報セキュリティ">情報セキュリティ</a>に関して一般公表されている<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>ごとに割り振られる識別子です。各<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>に一意の識別番号を付けることにより、ベンダーやセキュリティ機関で共通の<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>情報を参照することを可能としています。</p>
<p><a href="用語/CVE" class="internal-link" data-href="用語/CVE">CVE</a>は、<a href="用語/JVN" class="internal-link" data-href="用語/JVN">JVN</a>(Japan Vulnerability Notes)などの<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>情報を取り扱うサイトで公開されており、ベンダーやセキュリティ機関で共通の<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>情報を参照することが可能です。識別子の形式は<a href="用語/CVE" class="internal-link" data-href="用語/CVE">CVE</a>-&lt;西暦年号&gt;-&lt;4桁以上の数字&gt;です。例えば <a href="用語/CVE" class="internal-link" data-href="用語/CVE">CVE</a>-2021-44228 は、Log4jというJavaのロギングライブラリに関する<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>を指します。</p>
<p>したがって正しい記述は「ウ」になります。</p>
<p><a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>対策関連情報には似たような英語略字の用語があります。次の5つは押さえておきましょう。<br><a href="用語/CVE" class="internal-link" data-href="用語/CVE">CVE</a> … 共通<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>識別子<br>CCE … 共通セキュリティ設定一覧<br><a href="用語/CWE" class="internal-link" data-href="用語/CWE">CWE</a> … 共通<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>タイプ一覧<br>CPE … 共通プラットフォーム一覧<br><a href="用語/CVSS" class="internal-link" data-href="用語/CVSS">CVSS</a> … 共通<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>評価システム</p>
