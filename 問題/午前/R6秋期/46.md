<p class="ap-question"><strong>問46</strong>　エクスプロイトコードの説明はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　攻撃コードとも呼ばれ，ソフトウェアの脆弱性を悪用するコードのことであり，使い方によっては脆弱性の検証に役立つこともある。</div>
<div class="ap-choice-note">正しい。<a href="用語/エクスプロイトコード" class="internal-link" data-href="用語/エクスプロイトコード">エクスプロイトコード</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　マルウェア定義ファイルとも呼ばれ，マルウェアを特定するための特徴的なコードのことであり，マルウェア対策ソフトによるマルウェアの検知に用いられる。</div>
<div class="ap-choice-note">これはシグネチャコードの説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　メッセージとシークレットデータから計算されるハッシュコードのことであり，メッセージの改ざん検知に用いられる。</div>
<div class="ap-choice-note">これは<a href="用語/メッセージ認証" class="internal-link" data-href="用語/メッセージ認証">メッセージ認証</a>符号(<a href="用語/MAC" class="internal-link" data-href="用語/MAC">MAC</a>：Message Authentication Code)の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　ログインのたびに変化する認証コードのことであり，不正に取得しても再利用できないので不正アクセスを防ぐ。</div>
<div class="ap-choice-note">これは<a href="用語/ワンタイムパスワード" class="internal-link" data-href="用語/ワンタイムパスワード">ワンタイムパスワード</a>やログイン認証コードの説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/エクスプロイトコード" class="internal-link" data-href="用語/エクスプロイトコード">エクスプロイトコード</a>(Exploit Code)とは、ソフトウェアの<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>を悪用した不正な動作を再現するために作成されたスクリプトやプログラムを指す言葉です。</p>
<p>このようなプログラムは、善意の第三者や製品開発元の間で安全に流通している状況であれば問題ありませんが、世の中に出回ると攻撃方法の詳細が広く知られることになり、攻撃者によって個人や組織への攻撃に悪用されてしまう可能性があります。実際に様々な<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>に対して攻撃が成功するように、複数の<a href="用語/エクスプロイトコード" class="internal-link" data-href="用語/エクスプロイトコード">エクスプロイトコード</a>をまとめた<a href="用語/エクスプロイトキット" class="internal-link" data-href="用語/エクスプロイトキット">エクスプロイトキット</a>というツールも存在し、攻撃に利用されているのが現状です。</p>
<p><a href="用語/エクスプロイトコード" class="internal-link" data-href="用語/エクスプロイトコード">エクスプロイトコード</a>は、多くの場合、悪意を持ったマルウェアのことを指しますが、セキュリティ向上目的での研究材料や<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>検査のために作成された実証用コードという意味でも用いられます。</p>
<p>したがって「ア」が正解です。</p>
