<p class="ap-question"><strong>問78</strong>　著作権法で保護されるものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　アルゴリズム</div>
<div class="ap-choice-note">解法・<a href="用語/アルゴリズム" class="internal-link" data-href="用語/アルゴリズム">アルゴリズム</a>は<a href="用語/著作権法" class="internal-link" data-href="用語/著作権法">著作権法</a>の例外として保護対象外です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　コンパイラのプログラム</div>
<div class="ap-choice-note">正しい。<a href="用語/プログラム言語" class="internal-link" data-href="用語/プログラム言語">プログラム言語</a>によって作成された<a href="用語/プログラム" class="internal-link" data-href="用語/プログラム">プログラム</a>（ソースコード）は著作物として保護されます。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　プログラム言語</div>
<div class="ap-choice-note"><a href="用語/プログラム言語" class="internal-link" data-href="用語/プログラム言語">プログラム言語</a>そのものは<a href="用語/著作権法" class="internal-link" data-href="用語/著作権法">著作権法</a>の例外として保護対象外です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　プロトコル</div>
<div class="ap-choice-note">規約やプロトコルは<a href="用語/著作権法" class="internal-link" data-href="用語/著作権法">著作権法</a>の例外として保護対象外です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/著作権" class="internal-link" data-href="用語/著作権">著作権</a>は、<a href="用語/知的財産権" class="internal-link" data-href="用語/知的財産権">知的財産権</a>の中でも文化・芸術に関するものにおいて、創作物および創作者に認められる権利です。そして<a href="用語/著作権法" class="internal-link" data-href="用語/著作権法">著作権法</a>はこれらを保護するための法律です。情報産業の分野では、<a href="用語/プログラム" class="internal-link" data-href="用語/プログラム">プログラム</a>や<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>などが著作物に該当し、これらも保護対象になっています。</p>
<p><a href="用語/著作権法" class="internal-link" data-href="用語/著作権法">著作権法</a>には、著作物に対する例外として、著作物を作成するために用いる次の3つには及ばないとしています。</p>
<ul>
<li><a href="用語/プログラム言語" class="internal-link" data-href="用語/プログラム言語">プログラム言語</a> … <a href="用語/プログラム" class="internal-link" data-href="用語/プログラム">プログラム</a>を表現する手段としての文字その他の記号及びその体系</li>
<li>規約 … 特定の<a href="用語/プログラム" class="internal-link" data-href="用語/プログラム">プログラム</a>における前号の<a href="用語/プログラム言語" class="internal-link" data-href="用語/プログラム言語">プログラム言語</a>の用法についての特別の約束</li>
<li>解法 … <a href="用語/プログラム" class="internal-link" data-href="用語/プログラム">プログラム</a>における電子計算機に対する指令の組合せの方法</li>
</ul>
<p>この規定より、<a href="用語/プログラム言語" class="internal-link" data-href="用語/プログラム言語">プログラム言語</a>そのもの、規約やプロトコル、解法・<a href="用語/アルゴリズム" class="internal-link" data-href="用語/アルゴリズム">アルゴリズム</a>は<a href="用語/著作権" class="internal-link" data-href="用語/著作権">著作権</a>保護の対象外となります。<a href="用語/プログラム言語" class="internal-link" data-href="用語/プログラム言語">プログラム言語</a>は対象外ですが、<a href="用語/プログラム言語" class="internal-link" data-href="用語/プログラム言語">プログラム言語</a>によってつくられた<a href="用語/プログラム" class="internal-link" data-href="用語/プログラム">プログラム</a>（ソースコード）は保護対象になります。したがって、選択肢の中で<a href="用語/著作権法" class="internal-link" data-href="用語/著作権法">著作権法</a>で保護されるのは「<a href="用語/コンパイラ" class="internal-link" data-href="用語/コンパイラ">コンパイラ</a>の<a href="用語/プログラム" class="internal-link" data-href="用語/プログラム">プログラム</a>」のみとなります。</p>
</div>
