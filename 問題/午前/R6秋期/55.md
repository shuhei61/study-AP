<p class="ap-question"><strong>問55</strong>　サービスマネジメントにおいて，中断したサービスを復旧させるときの目標を定めた指標に，RTO(目標復旧時間)，RPO(目標復旧時点)及びRLO(目標復旧レベル)がある。RTOとRLOとを定めた例として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　サービスが中断する3時間前の時点の状態にデータを復旧し，利用者の50%以上にサービスを提供できるようにする。</div>
<div class="ap-choice-note"><a href="用語/RPO" class="internal-link" data-href="用語/RPO">RPO</a>と<a href="用語/RLO" class="internal-link" data-href="用語/RLO">RLO</a>を定めた例です。「3時間前の時点の状態」が<a href="用語/RPO" class="internal-link" data-href="用語/RPO">RPO</a>、「利用者の50%以上」が<a href="用語/RLO" class="internal-link" data-href="用語/RLO">RLO</a>に該当します。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　サービスが中断する直前の状態にデータを復旧し，当日のサービス終了時刻を，サービスが中断していた時間だけ延長する。</div>
<div class="ap-choice-note"><a href="用語/RPO" class="internal-link" data-href="用語/RPO">RPO</a>を定めた例です。「中断する直前の状態」が<a href="用語/RPO" class="internal-link" data-href="用語/RPO">RPO</a>に該当します。後半の施策はどれにも該当しません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　サービスの中断から1時間以内に，中断する1時間前の時点の状態にデータを復旧する。</div>
<div class="ap-choice-note"><a href="用語/RTO" class="internal-link" data-href="用語/RTO">RTO</a>と<a href="用語/RPO" class="internal-link" data-href="用語/RPO">RPO</a>を定めた例です。「中断から1時間以内」は<a href="用語/RTO" class="internal-link" data-href="用語/RTO">RTO</a>、「1時間前の時点の状態」は<a href="用語/RPO" class="internal-link" data-href="用語/RPO">RPO</a>に該当します。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　サービスの中断から1日以内に，中断したサービスのうちの重要なサービスに限定してサービスを復旧する。</div>
<div class="ap-choice-note">正しい。<a href="用語/RTO" class="internal-link" data-href="用語/RTO">RTO</a>と<a href="用語/RLO" class="internal-link" data-href="用語/RLO">RLO</a>とを定めた例です。「中断から1日以内」が<a href="用語/RTO" class="internal-link" data-href="用語/RTO">RTO</a>、「重要なサービスに限定してサービスを<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>」が<a href="用語/RLO" class="internal-link" data-href="用語/RLO">RLO</a>に該当します。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/RTO" class="internal-link" data-href="用語/RTO">RTO</a>、<a href="用語/RPO" class="internal-link" data-href="用語/RPO">RPO</a>、<a href="用語/RLO" class="internal-link" data-href="用語/RLO">RLO</a>は、いずれも<a href="用語/災害" class="internal-link" data-href="用語/災害">災害</a>が起きた際の<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>目標を表す指標です。<a href="用語/RTO" class="internal-link" data-href="用語/RTO">RTO</a>（Recovery Time Objective：目標<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>時間）『いつまでに<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>するのか』という時間的な目標のこと。例）サーバダウンからから2時間以内に<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>する</p>
<p><a href="用語/RPO" class="internal-link" data-href="用語/RPO">RPO</a>（Recovery Point Objective：目標<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>地点）『いつの時点のデータまで復元するのか』というデータの復元度合い関する目標のこと。例）サービス中断した日の0時時点のデータまで復元する</p>
<p><a href="用語/RLO" class="internal-link" data-href="用語/RLO">RLO</a>（Recovery Level Objective：目標<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>レベル）『<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>時にどの程度の機能を回復させるのか』という機能レベルに関する目標のこと。基本的には<a href="用語/RTO" class="internal-link" data-href="用語/RTO">RTO</a>と組み合わせて設定する。例）製造ラインが稼働できるよう、生産管理システムを<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>させる</p>
</div>
