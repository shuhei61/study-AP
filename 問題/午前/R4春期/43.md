<p class="ap-question"><strong>問43</strong>　JIS Q 27000:2019(情報セキュリティマネジメントシステム－用語)における"リスクレベル"の定義はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　脅威によって付け込まれる可能性のある，資産又は管理策の弱点</div>
<div class="ap-choice-note">これは<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>の定義です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　結果とその起こりやすさの組合せとして表現される，リスクの大きさ</div>
<div class="ap-choice-note">正しい。<a href="用語/リスクレベル" class="internal-link" data-href="用語/リスクレベル">リスクレベル</a>の定義です。<a href="用語/リスク" class="internal-link" data-href="用語/リスク">リスク</a>は影響度と発生確率の積で大きさを見積もります。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　対応すべきリスクに付与する優先順位</div>
<div class="ap-choice-note">この記述に対応する用語はありません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　リスクの重大性を評価するために目安とする条件</div>
<div class="ap-choice-note">これは<a href="用語/リスク基準" class="internal-link" data-href="用語/リスク基準">リスク基準</a>の定義です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>JIS Q 27000:2019では、<a href="用語/リスクレベル" class="internal-link" data-href="用語/リスクレベル">リスクレベル</a>は「結果とその起こりやすさの組合せとして表現される、<a href="用語/リスク" class="internal-link" data-href="用語/リスク">リスク</a>の大きさ」と定義されています。<a href="用語/リスク" class="internal-link" data-href="用語/リスク">リスク</a>の大きさは、一般に影響度と発生確率（起こりやすさ）の組合せで見積もります。</p>
<p>アは<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>、エは<a href="用語/リスク基準" class="internal-link" data-href="用語/リスク基準">リスク基準</a>の定義です。ウの記述は、同規格の用語としては定義されていません。</p>
</div>
