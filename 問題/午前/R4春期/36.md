<p class="ap-question"><strong>問36</strong>　複数のシステムやサービスの間で利用されるSAML(Security Assertion Markup Language)はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　システムの負荷や動作状況に関する情報を送信するための仕様</div>
<div class="ap-choice-note">SNMPやWMIなどが該当します。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　脆弱性に関する情報や脅威情報を交換するための仕様</div>
<div class="ap-choice-note">TAXII(Trusted Automated eXchange of Indicator Information)が該当します。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　通信を暗号化し，VPNを実装するための仕様</div>
<div class="ap-choice-note">IPsecやTLSなどのプロトコルが該当します。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　認証や認可に関する情報を交換するための仕様</div>
<div class="ap-choice-note">正しい。<a href="用語/SAML" class="internal-link" data-href="用語/SAML">SAML</a>の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/SAML" class="internal-link" data-href="用語/SAML">SAML</a>(Security Assertion Markup Language：サムエル)は、XMLベースのメッセージのやり取りによって、複数のシステムやサービスの間で認証情報、属性情報、権限の認可情報を交換するための仕様です。標準化団体OASISによって策定されていて、2024年現在の最新バージョンは2005年にリリースのSAML2.0です。主に複数のドメインにまたがったセキュアな<a href="用語/シングルサインオン" class="internal-link" data-href="用語/シングルサインオン">シングルサインオン</a>を実現するための仕組みとして使用されています。したがって「エ」が正解です。<a href="用語/SAML" class="internal-link" data-href="用語/SAML">SAML</a>では様々な方法による<a href="用語/シングルサインオン" class="internal-link" data-href="用語/シングルサインオン">シングルサインオン</a>が規定されていますが、その一つの基本的な流れは以下のとおりです。利用者がSP(Service Provider)にアクセスする→SPは利用者をIdP(Identity Provider)にリダイレクトする→IdPは利用者を認証し、<a href="用語/デジタル署名" class="internal-link" data-href="用語/デジタル署名">デジタル署名</a>が付いた<a href="用語/SAML" class="internal-link" data-href="用語/SAML">SAML</a>アサーションを生成して、<a href="用語/SAML" class="internal-link" data-href="用語/SAML">SAML</a>アサーションとともに利用者をSPにリダイレクトする→SPは<a href="用語/SAML" class="internal-link" data-href="用語/SAML">SAML</a>アサーションの<a href="用語/デジタル署名" class="internal-link" data-href="用語/デジタル署名">デジタル署名</a>を検証し、<a href="用語/SAML" class="internal-link" data-href="用語/SAML">SAML</a>アサーションの内容を確認する→SPは利用者をログインさせ、最初にアクセスがあったページにリダイレクトする。</p>
