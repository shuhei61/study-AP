<p class="ap-question"><strong>問10</strong>　キャッシュメモリのフルアソシエイティブ方式に関する記述として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　キャッシュメモリの各ブロックに主記憶のセットが固定されている。</div>
<div class="ap-choice-note"><a href="用語/セットアソシエイティブ方式" class="internal-link" data-href="用語/セットアソシエイティブ方式">セットアソシエイティブ方式</a>がセット化するのはキャッシュ側のブロックであり、<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>のブロックはセット化しません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　キャッシュメモリの各ブロックに主記憶のブロックが固定されている。</div>
<div class="ap-choice-note">ダイレクトマップ方式の説明です。フルアソシエイティブでは任意のキャッシュブロックに格納できます。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　主記憶の特定の1ブロックに専用のキャッシュメモリが割り当てられる。</div>
<div class="ap-choice-note"><a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>と同サイズのキャッシュが必要となり現実的ではありません。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　任意のキャッシュメモリのブロックを主記憶のどの部分にも割り当てられる。</div>
<div class="ap-choice-note">正しい。<a href="用語/フルアソシエイティブ方式" class="internal-link" data-href="用語/フルアソシエイティブ方式">フルアソシエイティブ方式</a>の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>上の格納位置と<a href="用語/キャッシュメモリ" class="internal-link" data-href="用語/キャッシュメモリ">キャッシュメモリ</a>上の格納位置とをどのように対応させるかを決める方式として、ダイレクトマップ、フルアソシエイティブ、セットアソシエイティブがあります。</p>
<p><strong>ダイレクトマップ</strong> … ハッシュ演算などにより、<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>のアドレスから<a href="用語/キャッシュメモリ" class="internal-link" data-href="用語/キャッシュメモリ">キャッシュメモリ</a>のブロック番号が一意に定まる方式。<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>のブロックと<a href="用語/キャッシュメモリ" class="internal-link" data-href="用語/キャッシュメモリ">キャッシュメモリ</a>のブロックが1対1で対応する</p>
<p><strong>フルアソシエイティブ</strong> … <a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>のブロックを<a href="用語/キャッシュメモリ" class="internal-link" data-href="用語/キャッシュメモリ">キャッシュメモリ</a>上のどのブロックにも格納することができる方式</p>
<p><strong>セットアソシエイティブ</strong> … <a href="用語/キャッシュメモリ" class="internal-link" data-href="用語/キャッシュメモリ">キャッシュメモリ</a>のブロックを連続した一定数ごとにまとめた"セット"を用意し、<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>のアドレスから対応する"セット"が一意に定まり、そのセット内のブロックならどこでも格納できる方式。ダイレクトマップとフルアソシエイティブの中間的な存在で、現在ほぼすべての<a href="用語/CPU" class="internal-link" data-href="用語/CPU">CPU</a>アーキテクチャで採用されている</p>
</div>
