#問題 #マネジメント #システム監査 #システム監査

<p class="ap-question"><strong>問59</strong>　監査調書に関する記述のうち，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　監査調書には，監査対象部門以外においても役立つ情報があるので，全て企業内で公開すべきである。</div>
<div class="ap-choice-note"><a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>調書は容易に参照できるよう保管されますが，重要情報・機密情報を含むことが多いため，受け渡し・持ち出しのルールや未承認アクセス・散逸・<a href="用語/改ざん" class="internal-link" data-href="用語/改ざん">改ざん</a>への対策が必要です。全社公開すべきという記述は誤りです。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　監査調書の役割として，監査実施内容の客観性を確保し，監査の結論を支える合理的な根拠とすることなどが挙げられる。</div>
<div class="ap-choice-note">正しい。<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>調書は実施内容の客観性を確保し，結論に至った過程を明らかにし，結論を支える合理的な根拠とするために作成されます。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　監査調書は，通常，電子媒体で保管されるが，機密保持を徹底するためバックアップは作成すべきではない。</div>
<div class="ap-choice-note"><a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>調書は<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>のプロセスを記録し結論の基礎となる重要書類なので適切に保管する必要があります。電子保管では適切な<a href="用語/バックアップ" class="internal-link" data-href="用語/バックアップ">バックアップ</a>対策が必要であり，<a href="用語/バックアップ" class="internal-link" data-href="用語/バックアップ">バックアップ</a>を作らないという記述は誤りです。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　監査調書は監査の過程で入手した客観的な事実の記録なので，監査担当者の所見は記述しない。</div>
<div class="ap-choice-note"><a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>基準(平成30年)では<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>調書に記載される事項の例として，発見事実に関する<a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>人の所見などが挙げられます。所見が記載されることもあるため誤りです。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>調書は，後から容易に参照・活用できるよう保管されますが，組織の重要情報や機密情報を含むことが一般的です。そのため<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>調書の受け渡しや持ち出し等のルールを定めるとともに，未承認アクセスへの防止，散逸，<a href="用語/改ざん" class="internal-link" data-href="用語/改ざん">改ざん</a>等に十分留意する必要があります。「ア」は誤りです。</p>
<p><a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>調書は，実施内容の客観性等を確保するとともに，<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>の結論に至った過程を明らかにし，<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>の結論を支える合理的な根拠とするために作成されます。「イ」が適切です。</p>
<p><a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>調書は<a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>人が実施した<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>のプロセスを記録したものであり，<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>の結論の基礎となる重要な書類です。通常は電子媒体で保管されるため，適切な<a href="用語/バックアップ" class="internal-link" data-href="用語/バックアップ">バックアップ</a>対策を講じる必要があります。「ウ」は誤りです。</p>
<p><a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>基準(平成30年)では，<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>調書に通常記載される事項として，<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>実施者及び実施日時，<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>の目的，実施した<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>手続，入手した<a href="用語/監査証拠" class="internal-link" data-href="用語/監査証拠">監査証拠</a>，発見事実（事象，原因，影響範囲等）及び発見事実に関する<a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>人の所見，レビュー実施時のレビューアの氏名及びレビュー日などが挙げられます。<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>担当者の所見が記載されることもあるため「エ」は誤りです。</p>
<p>正解は「イ」です。</p>
</div>
