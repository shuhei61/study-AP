<p class="ap-question"><strong>問57</strong>　基幹業務システムの構築及び運用において，データ管理者(DA)とデータベース管理者(DBA)を別々に任命した場合のDAの役割として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　業務データ量の増加傾向を把握し，ディスク装置の増設などを計画して実施する。</div>
<div class="ap-choice-note">ディスク装置の増設計画は<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>管理者（DBA）の役割です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　システム開発の設計工程では，主に論理データベース設計を行い，データ項目を管理して標準化する。</div>
<div class="ap-choice-note">正しい。論理<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>設計とデータ項目の<a href="用語/標準化" class="internal-link" data-href="用語/標準化">標準化</a>は<a href="用語/データ管理" class="internal-link" data-href="用語/データ管理">データ管理</a>者（DA）の役割です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　システム開発のテスト工程では，主にパフォーマンスチューニングを担当する。</div>
<div class="ap-choice-note"><a href="用語/パフォーマンス" class="internal-link" data-href="用語/パフォーマンス">パフォーマンス</a>チューニングは<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>管理者（DBA）の役割です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　システム障害が発生した場合には，データの復旧や整合性のチェックなどを行う。</div>
<div class="ap-choice-note">データの<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>や整合性のチェックは<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>管理者（DBA）の役割です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/データ管理" class="internal-link" data-href="用語/データ管理">データ管理</a>者（DA）と<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>管理者（DBA）は、<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>の管理を専門的に扱う職種です。<a href="用語/データ管理" class="internal-link" data-href="用語/データ管理">データ管理</a>者（DA:Data Administrator）業務の実世界から概念設計、システム化の範囲で<a href="用語/論理設計" class="internal-link" data-href="用語/論理設計">論理設計</a>などデータそのものの管理を行う。<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>管理者（DBA:DataBase Administrator）DAが設計した<a href="用語/論理データモデル" class="internal-link" data-href="用語/論理データモデル">論理データモデル</a>から物理設計を行い、<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>を構築したり、構築後の<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>の運用設計および運用保守など<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>の管理を行う。選択肢の中では、「主に論理<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>設計を行い」という部分から「イ」が<a href="用語/データ管理" class="internal-link" data-href="用語/データ管理">データ管理</a>者(DA)の役割であることがわかります。その他の選択肢は<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>管理者(DBA)の役割です。</p>
</div>
