<p class="ap-question"><strong>問34</strong>　IPの上位階層のプロトコルとして，コネクションレスのデータグラム通信を実現し，信頼性のための確認応答や順序制御などの機能をもたないプロトコルはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　ICMP</div>
<div class="ap-choice-note">Internet Control Message Protocolの略。インターネットプロトコル(IP)の通信制御を補完するプロトコルで、データ配送中のエラー通知や送達エラーを通知する機能を持ちます。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　PPP</div>
<div class="ap-choice-note">Point-to-Point Protocolの略。電話回線を通じてコンピュータをネットワークに接続するダイヤルアップ接続で使われる、2点間を接続してデータ通信を行うための通信プロトコルです。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　TCP</div>
<div class="ap-choice-note">Transmission Control Protocolの略。<a href="用語/TCP/IP" class="internal-link" data-href="用語/TCP/IP">TCP/IP</a>のネットワークにおいて送達管理や、伝送管理などの機能を持つコネクション型のプロトコルです。SYN<a href="用語/パケット" class="internal-link" data-href="用語/パケット">パケット</a>とACK<a href="用語/パケット" class="internal-link" data-href="用語/パケット">パケット</a>のやり取りによってコネクション確立を行う3ウェイハンドシェイクを行います。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　UDP</div>
<div class="ap-choice-note">正しい。<a href="用語/UDP" class="internal-link" data-href="用語/UDP">UDP</a>はコネクションレス型の伝送制御プロトコルです。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/UDP" class="internal-link" data-href="用語/UDP">UDP</a>(User Datagram Protocol)は、IPの上位層として使われる伝送制御プロトコルの一つで、コネクションレス型の通信を提供するプロトコルです。コネクションレスとは通信する相手との接続を確認せずに、一方的にデータを送り付けるという意味です。</p>
<p>同じく伝送制御プロトコルである<a href="用語/TCP" class="internal-link" data-href="用語/TCP">TCP</a>(Transmission Control Protocol)が、通信の<a href="用語/信頼性" class="internal-link" data-href="用語/信頼性">信頼性</a>を保証するために3ウェイハンドシェイク、誤り制御、再送制御、順序制御、輻輳制御などの機能を有しているのに対して、<a href="用語/UDP" class="internal-link" data-href="用語/UDP">UDP</a>はこれらを一切持ちません。しかし、その分<a href="用語/TCP" class="internal-link" data-href="用語/TCP">TCP</a>と比べて処理が軽くデータグラム中のデータ比率も高まるため、応答性を重視する<a href="用語/DNS" class="internal-link" data-href="用語/DNS">DNS</a>や<a href="用語/DHCP" class="internal-link" data-href="用語/DHCP">DHCP</a>、<a href="用語/NTP" class="internal-link" data-href="用語/NTP">NTP</a>などのプロトコルや、途中でデータが抜け落ちても問題が少ない音声や画像の<a href="用語/ストリーミング" class="internal-link" data-href="用語/ストリーミング">ストリーミング</a>配信などで用いられています。</p>
</div>
