<p class="ap-question"><strong>問40</strong>　JPCERTコーディネーションセンターとIPAとが共同で運営するJVNの目的として，最も適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　ソフトウェアに内在する脆弱性を検出し，情報セキュリティ対策に資する。</div>
<div class="ap-choice-note">MyJVNバージョンチェッカなどの説明です。<a href="用語/JVN" class="internal-link" data-href="用語/JVN">JVN</a>自体が<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>を検出するわけではありません。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　ソフトウェアの脆弱性関連情報とその対策情報とを提供し，情報セキュリティ対策に資する。</div>
<div class="ap-choice-note">正しい。<a href="用語/JVN" class="internal-link" data-href="用語/JVN">JVN</a>は<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>関連情報と対策情報を提供するポータルサイトです。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　ソフトウェアの脆弱性に対する汎用的な評価手法を確立し，情報セキュリティ対策に資する。</div>
<div class="ap-choice-note"><a href="用語/CVSS" class="internal-link" data-href="用語/CVSS">CVSS</a>(Common Vulnerability Scoring System)の説明です。評価手法の確立は<a href="用語/JVN" class="internal-link" data-href="用語/JVN">JVN</a>の目的ではありません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　ソフトウェアの脆弱性のタイプを識別するための基準を提供し，情報セキュリティ対策に資する。</div>
<div class="ap-choice-note"><a href="用語/CWE" class="internal-link" data-href="用語/CWE">CWE</a>(Common Weakness Enumeration)の説明です。<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>タイプの識別基準提供は<a href="用語/JVN" class="internal-link" data-href="用語/JVN">JVN</a>の目的ではありません。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/JVN" class="internal-link" data-href="用語/JVN">JVN</a>は(Japan Vulnerability Notes)は、日本で使用されているソフトウェアなどの<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>関連情報とその対策情報を提供し、<a href="用語/情報セキュリティ" class="internal-link" data-href="用語/情報セキュリティ">情報セキュリティ</a>対策に資することを目的とする<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>対策情報ポータルサイトです。<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>関連情報の受付と安全な流通を目的とした「<a href="用語/情報セキュリティ早期警戒パートナーシップ" class="internal-link" data-href="用語/情報セキュリティ早期警戒パートナーシップ">情報セキュリティ早期警戒パートナーシップ</a>」に基いて、2004年7月よりJPCERT コーディネーションセンターと独立行政法人情報処理推進機構(IPA)が共同で運営しています。</p>
<p><a href="用語/JVN" class="internal-link" data-href="用語/JVN">JVN</a>に掲載されている情報は、<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>が確認された製品とバージョン、<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>の詳細や分析結果、製品開発者によって提供された対策や関連情報へのリンクなどで、対策にはパッチだけではなく<a href="用語/回避策" class="internal-link" data-href="用語/回避策">回避策</a>（ワークアラウンド）が掲載される事もあります。</p>
