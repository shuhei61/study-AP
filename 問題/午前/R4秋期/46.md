<p class="ap-question"><strong>問46</strong>　仕様書やソースコードといった成果物について，作成者を含めた複数人で，記述されたシステムやソフトウェアの振る舞いを机上でシミュレートして，問題点を発見する手法はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　ウォークスルー</div>
<div class="ap-choice-note">正しい。<a href="用語/ウォークスルー" class="internal-link" data-href="用語/ウォークスルー">ウォークスルー</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　サンドイッチテスト</div>
<div class="ap-choice-note">サンドイッチテスト(折衷テスト)は、<a href="用語/トップダウンテスト" class="internal-link" data-href="用語/トップダウンテスト">トップダウンテスト</a>と<a href="用語/ボトムアップテスト" class="internal-link" data-href="用語/ボトムアップテスト">ボトムアップテスト</a>を組み合わせて行う結合テストの手法です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　トップダウンテスト</div>
<div class="ap-choice-note"><a href="用語/トップダウンテスト" class="internal-link" data-href="用語/トップダウンテスト">トップダウンテスト</a>は、上位モジュールから下位モジュールへ順に結合しながら行う結合テストの手法です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　並行シミュレーション</div>
<div class="ap-choice-note">並行<a href="用語/シミュレーション" class="internal-link" data-href="用語/シミュレーション">シミュレーション</a>は、テスト用プログラムと検証対象プログラムに同じデータを入力し出力を比較する手法です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/ウォークスルー" class="internal-link" data-href="用語/ウォークスルー">ウォークスルー</a>は、設計上の誤りを早期に発見することを目的として、各工程の終了時点で、作成者を含めた複数の関係者が設計書やソースコードを検査する<a href="用語/デザインレビュー" class="internal-link" data-href="用語/デザインレビュー">デザインレビュー</a>の手法です。①原則として管理者には参加させない、②誤りの発見を目的としその場で解決策を議論しない、③事前に資料を配布しておくことで短時間で効率よく行うなどの特徴があります。</p>
</div>
