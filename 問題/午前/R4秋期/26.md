<p class="ap-question"><strong>問26</strong>　データ項目の命名規約を設ける場合，次の命名規約だけでは回避できない事象はどれか。<br>〔命名規約〕データ項目名の末尾には必ず"名"，"コード"，"数"，"金額"，"年月日"などの区分語を付与し，区分語ごとに定めたデータ型にする。データ項目名と意味を登録した辞書を作成し，異音同義語や同音異義語が発生しないようにする。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　データ項目"受信年月日"のデータ型として，日付型と文字列型が混在する。</div>
<div class="ap-choice-note">「受信年月日」で<a href="用語/日付型" class="internal-link" data-href="用語/日付型">日付型</a>と<a href="用語/文字列型" class="internal-link" data-href="用語/文字列型">文字列型</a>が混在する事象は，末尾区分「年月日」ごとに定めた型に統一されるため，命名規約で回避できる。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　データ項目"受注金額"の取り得る値の範囲がテーブルによって異なる。</div>
<div class="ap-choice-note">正しい。データ型は命名規約で定められるが，値の範囲は定義されないため，この規約だけでは回避できない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　データ項目"賞与金額"と同じ意味で"ボーナス金額"というデータ項目がある。</div>
<div class="ap-choice-note"><a href="用語/データディクショナリ" class="internal-link" data-href="用語/データディクショナリ">データディクショナリ</a>で異音同義語・同音異義語を防ぐため，命名規約で回避できる。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　データ項目"取引先"が，"取引先コード"か"取引先名"か，判別できない。</div>
<div class="ap-choice-note">末尾に区分語を付与する規約により"取引先"という項目名は存在しえないため，回避できる。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>命名規約では，区分語ごとのデータ型と<a href="用語/データディクショナリ" class="internal-link" data-href="用語/データディクショナリ">データディクショナリ</a>による用語の統一は定められますが，データ型の値の取り得る範囲までは定義されません。そのため，テーブルごとに同じデータ項目名でも値の範囲が異なる事象は，この命名規約だけでは回避できません。正解は「イ」です。</p>
</div>
</div>
