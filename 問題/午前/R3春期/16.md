<p class="ap-question"><strong>問16</strong>　ジョブの多重度が1で，到着順にジョブが実行されるシステムにおいて，表に示す状態のジョブA～Cを処理するとき，ジョブCが到着してから実行が終了するまでのターンアラウンドタイムは何秒か。ここで， OSのオーバーヘッドは考慮しないものとする。<br>（表：ジョブ／到着時刻(秒)／実行時間(秒) — A：0／3、B：2／6、C：3／5。原文は ap-siken 上の図）</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　11</div>
<div class="ap-choice-note">正しい。ジョブCの到着（処理開始から3秒後）から完了（14秒後）までの差が11秒。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　12</div>
<div class="ap-choice-note">ジョブCの完了時刻のみを答えた誤り（14秒ではなく到着からの経過時間を求める）。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　13</div>
<div class="ap-choice-note">待ち時間の加算や完了時刻の取り違えなどで1秒ずれた誤り。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　14</div>
<div class="ap-choice-note">処理開始時点からの完了時刻（14秒）をそのまま答えた誤り。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/ターンアラウンドタイム" class="internal-link" data-href="用語/ターンアラウンドタイム">ターンアラウンドタイム</a>とは、利用者がシステムに処理要求を開始してから、すべての結果を受け取るまでに要する時間のことです。設問の条件に従い、ジョブA～Cが実行される様子を図で表すと次のようになります（元ページでは図で示されています）。</p>
<p>ジョブCの到着は処理開始時点から3秒後、ジョブCの完了は処理開始時点から14秒後なので、<a href="用語/ターンアラウンドタイム" class="internal-link" data-href="用語/ターンアラウンドタイム">ターンアラウンドタイム</a>は「14秒－3秒＝11秒」になります。</p>
<p>したがって正解は「ア」です。</p>
