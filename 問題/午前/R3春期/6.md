<p class="ap-question"><strong>問6</strong>　配列A[1]，A[2]，…，A[n]で，A[1]を根とし，A[i]の左側の子をA[2i]，右側の子をA[2i＋1]とみなすことによって，2分木を表現する。このとき，配列を先頭から順に調べていくことは，2分木の探索のどれに当たるか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　行きがけ順(先行順)深さ優先探索</div>
<div class="ap-choice-note">詳細：<a href="用語/先行順" class="internal-link" data-href="用語/先行順">先行順</a>（<a href="用語/深さ優先探索" class="internal-link" data-href="用語/深さ優先探索">深さ優先探索</a>）。配列の先頭から順にはならない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　帰りがけ順(後行順)深さ優先探索</div>
<div class="ap-choice-note">詳細：<a href="用語/後行順" class="internal-link" data-href="用語/後行順">後行順</a>（<a href="用語/深さ優先探索" class="internal-link" data-href="用語/深さ優先探索">深さ優先探索</a>）。配列の先頭から順にはならない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　通りがけ順(中間順)深さ優先探索</div>
<div class="ap-choice-note">詳細：<a href="用語/中間順" class="internal-link" data-href="用語/中間順">中間順</a>（<a href="用語/深さ優先探索" class="internal-link" data-href="用語/深さ優先探索">深さ優先探索</a>）。配列の先頭から順にはならない。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　幅優先探索</div>
<div class="ap-choice-note">正しい。根から近い順に階層ごとに調べるため，配列を先頭から順に見る操作は<a href="用語/幅優先探索" class="internal-link" data-href="用語/幅優先探索">幅優先探索</a>に当たる。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>仮に配列[1, 2, 3, 4, 5, 6, 7]があったとき、設問のルールにより表される<a href="用語/2分木" class="internal-link" data-href="用語/2分木">2分木</a>は次の構造になります。（元ページでは図で示されています）</p>
<p>この木構造において配列を先頭から順に1，2，…と調べていくと、以下の順序で要素を調べていくことなります。</p>
<p>根から近い順に階層ごとに探索することになるため「<a href="用語/幅優先探索" class="internal-link" data-href="用語/幅優先探索">幅優先探索</a>」が適切となります。よって「エ」が正解です。なお、<a href="用語/深さ優先探索" class="internal-link" data-href="用語/深さ優先探索">深さ優先探索</a>は、根から開始して左の（右の）部分木優先で最も深い葉まで到達してから、一つ前の節まで戻り他方の部分木の探索を行うことを繰り返す方法です。<a href="用語/先行順" class="internal-link" data-href="用語/先行順">先行順</a>、<a href="用語/後行順" class="internal-link" data-href="用語/後行順">後行順</a>、<a href="用語/中間順" class="internal-link" data-href="用語/中間順">中間順</a>は参照するタイミングが違うだけです。</p>
<p>行きがけ順(<a href="用語/先行順" class="internal-link" data-href="用語/先行順">先行順</a>) … 訪問したタイミングで調べる</p>
<p>帰りがけ順(<a href="用語/後行順" class="internal-link" data-href="用語/後行順">後行順</a>) … 移動する子がなくなったタイミングで調べる</p>
<p>通りがけ順(<a href="用語/中間順" class="internal-link" data-href="用語/中間順">中間順</a>) … 左の子に移動できなくなったタイミングで調べる</p>
