<p class="ap-question"><strong>問50</strong>　アジャイル開発などで導入されている"ペアプログラミング"の説明はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　開発工程の初期段階に要求仕様を確認するために，プログラマと利用者がペアとなり，試作した画面や帳票を見て，相談しながらプログラムの開発を行う。</div>
<div class="ap-choice-note">これはプロトタイピングの説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　効率よく開発するために，2人のプログラマがペアとなり，メインプログラムとサブプログラムを分担して開発を行う。</div>
<div class="ap-choice-note">役割分担はドライバとナビゲータであり，メイン／サブプログラムの分担開発ではありません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　短期間で開発するために，2人のプログラマがペアとなり，交互に作業と休憩を繰り返しながら長時間にわたって連続でプログラムの開発を行う。</div>
<div class="ap-choice-note"><a href="用語/ペアプログラミング" class="internal-link" data-href="用語/ペアプログラミング">ペアプログラミング</a>は同時に1つのプログラムを共同実装する手法であり，長時間の交代休憩運用の説明ではありません。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　品質の向上や知識の共有を図るために，2人のプログラマがペアとなり，その場で相談したりレビューしたりしながら，一つのプログラムの開発を行う。</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/ペアプログラミング" class="internal-link" data-href="用語/ペアプログラミング">ペアプログラミング</a></div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/ペアプログラミング" class="internal-link" data-href="用語/ペアプログラミング">ペアプログラミング</a>は、2人1組で1つのプログラムの実装を行い、1人が実際のコードをコンピュータに打ち込み、もう1人はそれをチェックしながら補佐するという役割を随時交代しながら作業を進めていく手法です。プログラムをコンピュータに打ち込む人を「ドライバ」、それを補佐する人を「ナビゲータ」といいます。ペアプログラミングには以下のようなメリットがあるとされています。</p>
<p>1. 2人の共同作業なので個々で行うよりも怠けにくい<br>2. 2人で実装を行う相乗効果でよりよいコードに仕上がることが期待できる<br>3. 他の人からの問い合わせにも空いている1人が対応できるので作業効率が向上する<br>4. 共同作業を行うことでメンバーに対する理解が進みチームワークが向上する</p>
<p>適切な役割分担になっている「エ」が正解です。</p>
