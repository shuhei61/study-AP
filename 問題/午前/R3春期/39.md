<p class="ap-question"><strong>問39</strong>　リスクベース認証の特徴はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　いかなる利用条件でのアクセスの要求においても，ハードウェアトークンとパスワードを併用するなど，常に二つの認証方式を併用することによって，不正アクセスに対する安全性を高める。</div>
<div class="ap-choice-note">二要素認証の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　いかなる利用条件でのアクセスの要求においても認証方法を変更せずに，同一の手順によって普段どおりにシステムにアクセスできるようにし，可用性を高める。</div>
<div class="ap-choice-note"><a href="用語/RADIUS" class="internal-link" data-href="用語/RADIUS">RADIUS</a>認証の説明です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　普段と異なる利用条件でのアクセスと判断した場合には，追加の本人認証をすることによって，不正アクセスに対する安全性を高める。</div>
<div class="ap-choice-note">正しい。<a href="用語/リスクベース認証" class="internal-link" data-href="用語/リスクベース認証">リスクベース認証</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　利用者が認証情報を忘れ，かつ，Webブラウザに保存しているパスワード情報も使用できないリスクを想定して，緊急と判断した場合には，認証情報を入力せずに，利用者は普段どおりにシステムを利用できるようにし，可用性を高める。</div>
<div class="ap-choice-note">災害などの緊急時に認証なしで使えるようにする仕組みとして、公衆無線LAN（00000JAPANなど）の開放機能や"救済パスワード"がありますが、これらは<a href="用語/リスクベース認証" class="internal-link" data-href="用語/リスクベース認証">リスクベース認証</a>ではありません。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/リスクベース認証" class="internal-link" data-href="用語/リスクベース認証">リスクベース認証</a>は、利用者の<a href="用語/ログイン" class="internal-link" data-href="用語/ログイン">ログイン</a>行動や環境を分析し、普段使われているものとは異なる利用環境（<a href="用語/IPアドレス" class="internal-link" data-href="用語/IPアドレス">IPアドレス</a>・ISP・OS・<a href="用語/Webブラウザ" class="internal-link" data-href="用語/Webブラウザ">Webブラウザ</a>など）からの認証要求があった場合に、通常の認証に追加する形で別の認証を求める方式です。この方式は、不正<a href="用語/ログイン" class="internal-link" data-href="用語/ログイン">ログイン</a>のリスクが高いアクセスに対してだけ追加の認証を行うため、一定の利便性を保ちつつ、<a href="用語/不正アクセス" class="internal-link" data-href="用語/不正アクセス">不正アクセス</a>に対する<a href="用語/セキュリティ" class="internal-link" data-href="用語/セキュリティ">セキュリティ</a>を高めることができます。追加認証としては、<a href="用語/ワンタイムパスワード" class="internal-link" data-href="用語/ワンタイムパスワード">ワンタイムパスワード</a>、秘密の質問、SMS認証、<a href="用語/生体認証" class="internal-link" data-href="用語/生体認証">生体認証</a>などが用いられます。したがって適切な記述は「ウ」です。</p>
</div>
