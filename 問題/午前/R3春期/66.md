<p class="ap-question"><strong>問66</strong>　システム開発委託契約の委託報酬におけるレベニューシェア契約の特徴はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　委託側が開発するシステムから得られる収益とは無関係に開発に必要な費用を全て負担する。</div>
<div class="ap-choice-note">これは<a href="用語/実費償還契約" class="internal-link" data-href="用語/実費償還契約">実費償還契約</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　委託側は開発するシステムから得られる収益に関係無く定額で費用を負担する。</div>
<div class="ap-choice-note">これは<a href="用語/定額契約" class="internal-link" data-href="用語/定額契約">定額契約</a>（定額課金）や<a href="用語/サブスクリプションモデル" class="internal-link" data-href="用語/サブスクリプションモデル">サブスクリプションモデル</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　開発するシステムから得られる収益を委託側が受託側にあらかじめ決められた配分率で分配する。</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/レベニューシェア" class="internal-link" data-href="用語/レベニューシェア">レベニューシェア</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　受託側は継続的に固定額の収益が得られる。</div>
<div class="ap-choice-note">収益の多寡に応じて得られる収益は変わります。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/レベニューシェア" class="internal-link" data-href="用語/レベニューシェア">レベニューシェア</a>契約は、レベニュー(収益)をシェア(分け合う)の名称通り、アプリケーションの開発費を無償または通常より安価にする代わりに、そのアプリケーションから得られる収益の何割かを開発受託者に支払うことを約する契約形態です。例えば、「過去問道場のデータを使ったスマホアプリを無償で作ります。ただし、スマホアプリの収益は半々で分けましょう」といった契約形態などです。</p>
<p>委託者側にとっては、システム開発費の削減により事業リスクを低くできるメリットがありますが、システムが成功した場合に得られる収益は少なくなります。受託者側にとっては、システムが長期間にわたり利益を生み出し続けた場合に継続的な収入を得られるメリットがある一方、収益が期待していたほど上がらないと赤字になってしまうリスクもあります。</p>
<p>システムの成功が<a href="用語/SI事業者" class="internal-link" data-href="用語/SI事業者">SI事業者</a>の収益増加に直結するので、従来の「買い切り型」のシステム開発と比較して、開発側の取り組む姿勢が向上するという副次的な作用もあるようです。したがって、「システムから得られる収益のうち所定割合を受託側に分配する」としている「ウ」の記述が<a href="用語/レベニューシェア" class="internal-link" data-href="用語/レベニューシェア">レベニューシェア</a>型契約に該当します。</p>
</div>
