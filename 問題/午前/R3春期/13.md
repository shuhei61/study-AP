<p class="ap-question"><strong>問13</strong>　システムの信頼性設計に関する記述のうち，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　フェールセーフとは，利用者の誤操作によってシステムが異常終了してしまうことのないように，単純なミスを発生させないようにする設計方法である。</div>
<div class="ap-choice-note">これは<a href="用語/フールプルーフ" class="internal-link" data-href="用語/フールプルーフ">フールプルーフ</a>の説明です。<a href="用語/フェールセーフ" class="internal-link" data-href="用語/フェールセーフ">フェールセーフ</a>は、システムの不具合や<a href="用語/故障" class="internal-link" data-href="用語/故障">故障</a>が発生したときに<a href="用語/障害" class="internal-link" data-href="用語/障害">障害</a>の被害を最小限に抑えるために、安全を優先にした制御を行う考え方です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　フェールソフトとは，故障が発生した場合でも機能を縮退させることなく稼動を継続する概念である。</div>
<div class="ap-choice-note">これはフォールトトレランスの説明です。<a href="用語/フェールソフト" class="internal-link" data-href="用語/フェールソフト">フェールソフト</a>は、<a href="用語/障害" class="internal-link" data-href="用語/障害">障害</a>が発生したときに多少の機能制限や性能低下を許容しつつ、システム全体の運転継続に必要な機能を維持しようする設計方針です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　フォールトアボイダンスとは，システム構成要素の個々の品質を高めて故障が発生しないようにする概念である。</div>
<div class="ap-choice-note">正しい。<a href="用語/フォールトアボイダンス" class="internal-link" data-href="用語/フォールトアボイダンス">フォールトアボイダンス</a>は、システムを構成する要素を<a href="用語/稼働率" class="internal-link" data-href="用語/稼働率">稼働率</a>が高いものなど信頼性の高いものに取り換えることで、<a href="用語/故障" class="internal-link" data-href="用語/故障">故障</a>の発生確率自体を低下させ、システム全体の信頼性を向上させようという考え方です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　フォールトトレランスとは，故障が生じてもシステムに重大な影響が出ないように，あらかじめ定められた安全状態にシステムを固定し，全体として安全が維持されるような設計手法である。</div>
<div class="ap-choice-note">これは<a href="用語/フェールセーフ" class="internal-link" data-href="用語/フェールセーフ">フェールセーフ</a>の説明です。フォールトトレランスは、構成機器の二重化などにより<a href="用語/故障" class="internal-link" data-href="用語/故障">故障</a>に備えることで、システムの一部に<a href="用語/障害" class="internal-link" data-href="用語/障害">障害</a>が発生しても全体としては停止することなく稼働し続けることを目指す設計です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>システムの信頼性設計には、<a href="用語/故障" class="internal-link" data-href="用語/故障">故障</a>の発生を抑える考え方、<a href="用語/故障" class="internal-link" data-href="用語/故障">故障</a>が起きても被害を抑える考え方、<a href="用語/障害" class="internal-link" data-href="用語/障害">障害</a>後も稼働を続ける考え方など、目的に応じた複数の概念があります。本問では各選択肢が別の概念の説明と用語名が入れ替わっているため、用語の定義を正しく対応づけることが重要です。</p>
<p><strong><a href="用語/フェールセーフ" class="internal-link" data-href="用語/フェールセーフ">フェールセーフ</a></strong> … システムの不具合や<a href="用語/故障" class="internal-link" data-href="用語/故障">故障</a>が発生したときに<a href="用語/障害" class="internal-link" data-href="用語/障害">障害</a>の被害を最小限に抑えるために、安全を優先にした制御を行う考え方</p>
<p><strong><a href="用語/フェールソフト" class="internal-link" data-href="用語/フェールソフト">フェールソフト</a></strong> … <a href="用語/障害" class="internal-link" data-href="用語/障害">障害</a>が発生したときに多少の機能制限や性能低下を許容しつつ、システム全体の運転継続に必要な機能を維持しようする設計方針</p>
<p><strong><a href="用語/フォールトアボイダンス" class="internal-link" data-href="用語/フォールトアボイダンス">フォールトアボイダンス</a></strong> … システムを構成する要素を<a href="用語/稼働率" class="internal-link" data-href="用語/稼働率">稼働率</a>が高いものなど信頼性の高いものに取り換えることで、<a href="用語/故障" class="internal-link" data-href="用語/故障">故障</a>の発生確率自体を低下させ、システム全体の信頼性を向上させようという考え方（正解）</p>
<p><strong>フォールトトレランス</strong> … 構成機器の二重化などにより<a href="用語/故障" class="internal-link" data-href="用語/故障">故障</a>に備えることで、システムの一部に<a href="用語/障害" class="internal-link" data-href="用語/障害">障害</a>が発生しても全体としては停止することなく稼働し続けることを目指す設計</p>
<p><strong><a href="用語/フールプルーフ" class="internal-link" data-href="用語/フールプルーフ">フールプルーフ</a></strong> … 利用者の誤操作によってシステムが異常終了してしまうことのないように、単純なミスを発生させないようにする設計方法（アの記述内容に相当）</p>
