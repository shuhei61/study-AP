<p class="ap-question"><strong>問19</strong>　ページング方式の仮想記憶において，ページアクセス時に発生する事象をその回数の多い順に並べたものはどれか。ここで，A≧Bは，Aの回数がBの回数以上，A＝Bは，AとBの回数が常に同じであることを表す。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　ページアウト≧ページイン≧ページフォールト</div>
<div class="ap-choice-note"><a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>は必要ページが<a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>に無いときに必ず発生し，そのたびに<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>が伴うため，フォールトが最も少ない順序にはなりません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　ページアウト≧ページフォールト≧ページイン</div>
<div class="ap-choice-note"><a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>と<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>は常に同回数で発生する関係であり，フォールトより<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>の方が少ない順序は成立しません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　ページフォールト＝ページアウト≧ページイン</div>
<div class="ap-choice-note"><a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>に空きがあるときは<a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a>せず<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>だけで済むため，フォールトとアウトが常に同回数とは限りません。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　ページフォールト＝ページイン≧ページアウト</div>
<div class="ap-choice-note">正しい。フォールトのたびに<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>が発生し，空きがあればアウト無しでインできるため，アウトはフォールト以下になり得ます。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>まずはページング方式における各事象の意味を確認します。以下の記述中、<a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>は<a href="用語/主記憶" class="internal-link" data-href="用語/主記憶">主記憶</a>装置、<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>は<a href="用語/補助記憶" class="internal-link" data-href="用語/補助記憶">補助記憶</a>装置（HDDやSSD）を指します。</p>
<p><strong><a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a></strong> … プログラムの実行に必要なページが、<a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>に存在していないときに発生する<a href="用語/割込み" class="internal-link" data-href="用語/割込み">割込み</a></p>
<p><strong><a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a></strong> … <a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>上のページを<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>に移動させる処理</p>
<p><strong><a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a></strong> … <a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>上のページを<a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>に移動させる処理</p>
<p>ページ置換の基本的な流れは以下の通りです。</p>
<ol>
<li><a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>が発生する</li>
<li><a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>に空きがなければ、置換アルゴリズムが<a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>上から<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>に移すページを決定する</li>
<li>②で決定したページを<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>に移動する</li>
<li>プログラムの実行に必要なページを<a href="用語/仮想記憶" class="internal-link" data-href="用語/仮想記憶">仮想記憶</a>から<a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>に移動する</li>
</ol>
<p><a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>は、必要なページが<a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>上に存在しないときに起こるので、必ずそれに伴い<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>が発生します。よって、<a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>＝<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>です。一方、<a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>が発生しても<a href="用語/実記憶" class="internal-link" data-href="用語/実記憶">実記憶</a>上に空きがある場合には、<a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a>を行わずにその空き領域に必要なページを移動すれば済みます（上記②③が実行されない）。すなわち、<a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a>の回数は<a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>の回数よりも少なくなる可能性があります。よって、<a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>≧<a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a>です。以上の2つの関係をまとめると「<a href="用語/ページフォールト" class="internal-link" data-href="用語/ページフォールト">ページフォールト</a>＝<a href="用語/ページイン" class="internal-link" data-href="用語/ページイン">ページイン</a>≧<a href="用語/ページアウト" class="internal-link" data-href="用語/ページアウト">ページアウト</a>」となります。したがって「エ」が適切です。</p>
</div>
