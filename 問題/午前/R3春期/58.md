<p class="ap-question"><strong>問58</strong>　情報セキュリティ管理基準(平成28年)を基に，情報システム環境におけるマルウェア対策の実施状況について監査を実施した。判明したシステム運用担当者の対応状況のうち，監査人が，指摘事項として監査報告書に記載すべきものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　Webページに対して，マルウェア検出のためのスキャンを行っている。</div>
<div class="ap-choice-note">本基準では予防または定常作業としてウェブページのマルウェア検出走査を求めているため，適切な対応状況です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　マルウェア感染によって被害を受けた事態を想定して，事業継続計画を策定している。</div>
<div class="ap-choice-note">本基準ではマルウェアの攻撃から回復するための<a href="用語/事業継続計画" class="internal-link" data-href="用語/事業継続計画">事業継続計画</a>の策定を求めているため，適切な対応状況です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　マルウェア検出のためのスキャンを実施した上で，組織として認可していないソフトウェアを使用している。</div>
<div class="ap-choice-note">正しい。本基準では認可されていないソフトウェアの使用禁止と，その使用の防止・検出のための管理策を求めており，認可外ソフトウェアの使用は<a href="用語/指摘事項" class="internal-link" data-href="用語/指摘事項">指摘事項</a>に該当します。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　マルウェアに付け込まれる可能性のある脆弱性について情報収集を行い，必要に応じて修正コードを適用し，脆弱性の低減を図っている。</div>
<div class="ap-choice-note">本基準ではマルウェアに付け込まれる可能性のある<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>を技術的<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>管理などで低減することを求めているため，適切な対応状況です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/情報セキュリティ管理基準" class="internal-link" data-href="用語/情報セキュリティ管理基準">情報セキュリティ管理基準</a>は、組織において効果的な情報セキュリティマネジメント体制を構築し、適切なコントロール(管理策)を整備・運用するための実践的な規範であると同時に、<a href="用語/情報セキュリティ監査" class="internal-link" data-href="用語/情報セキュリティ監査">情報セキュリティ監査</a>をする際に、<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>人が<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>上の判断の尺度として使うべき基準です。<a href="用語/情報セキュリティ管理基準" class="internal-link" data-href="用語/情報セキュリティ管理基準">情報セキュリティ管理基準</a>(平成28年)では、"12.2 マルウェアからの保護"として全部で14個のコントロール項目を規定しています。</p>
<p>アは、本基準では予防または定常作業として「ウェブページに対するマルウェア検出のための走査」を行うことを求めているため、適切な対応状況と判断できます。</p>
<p>イは、本基準では「マルウェアの攻撃から回復するための適切な<a href="用語/事業継続計画" class="internal-link" data-href="用語/事業継続計画">事業継続計画</a>を策定する」ことを求めているため、適切な対応状況と判断できます。</p>
<p>ウは、本基準では「認可されていないソフトウェアの使用を禁止する正式な方針を確立する」とともに「認可されていないソフトウェアの使用を防止又は検出するための管理策を実施する」ことを求めています。したがって、組織が認可していないソフトウェアを使用している状況は、<a href="用語/指摘事項" class="internal-link" data-href="用語/指摘事項">指摘事項</a>に該当します。</p>
<p>エは、本基準では「マルウェアに付け込まれる可能性のある<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>を、技術的<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>管理などを通じて低減させる」ことを求めているため、適切な対応状況と判断できます。</p>
