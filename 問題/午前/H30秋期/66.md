<p class="ap-question"><strong>問66</strong>　ベンダーX社に対して，表に示すように要件定義フェーズから運用テストフェーズまでを委託したい。X社との契約に当たって，"情報システム・モデル取引・契約書＜第一版＞"に照らし，各フェーズの契約形態を整理した。a～dの契約形態のうち，準委任型が適切であるとされるものはどれか。<br><img class="ap-fig" src="../../../assets/問題/午前/H30秋期/66/66.png" alt="問題図" width="533" height="120" ></p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　a，b</div>
<div class="ap-choice-note">b（システム内部設計）は請負型が適切とされ、準委任型の組合せではない。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　a，d</div>
<div class="ap-choice-note">正しい。要件定義と運用テストは準委任型が適切とされる。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　b，c</div>
<div class="ap-choice-note">b・c（システム内部設計・システム結合）はいずれも請負型が適切とされる。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　b，d</div>
<div class="ap-choice-note">b（システム内部設計）は請負型が適切とされ、準委任型の組合せではない。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/情報システム・モデル取引・契約書" class="internal-link" data-href="用語/情報システム・モデル取引・契約書">情報システム・モデル取引・契約書</a>は、ユーザーとベンダーのあるべき理想的なモデルを提示し、情報システムのライフサイクルプロセスの中で、『ユーザーとベンダーの間でどのようなことを決定し、どのようなことを情報共有すればよいか』についての統一的な指針を目指して策定されたガイドラインです。</p>
<p><a href="用語/準委任契約" class="internal-link" data-href="用語/準委任契約">準委任契約</a>は、仕事の完成を目的とする<a href="用語/請負契約" class="internal-link" data-href="用語/請負契約">請負契約</a>と異なり、委託された仕事の実施自体を目的とする契約形態です。受託者は善良な管理者の注意をもって委任事務を処理する義務（善管注意義務）を負うものの、主に業務分析や要件定義、運用テスト工程などの成果物が特定されていない状況で結ばれます。</p>
<p><a href="用語/情報システム・モデル取引・契約書" class="internal-link" data-href="用語/情報システム・モデル取引・契約書">情報システム・モデル取引・契約書</a>では、フェーズの分類と対応する契約類型を以下のように規定しています。</p>
<p><img class="ap-fig" src="../../../assets/問題/午前/H30秋期/66/66a.png" alt="解説図" width="460" height="410" ></p>
<p>要件定義と運用テストは「準委任型」の契約、システム内部設計とシステム結合は「請負型」の契約が適切とされています。したがって「a，d」の組合せが適切です。</p>
