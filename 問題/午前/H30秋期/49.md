<p class="ap-question"><strong>問49</strong>　プログラムの誤りの一つに，繰返し処理の終了条件として A≧a とすべきところを A＞a とコーディングしたことに起因するものがある。このような誤りを見つけ出すために有効なテストケース設計技法はどれか。ここで，Aは変数，aは定数とする。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　限界値分析</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/限界値分析法" class="internal-link" data-href="用語/限界値分析法">限界値分析法</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　条件網羅</div>
<div class="ap-choice-note">判定条件内の個々の条件の真偽を網羅する技法であり、境界値付近の誤り検出には向かない。詳細：<a href="用語/条件網羅" class="internal-link" data-href="用語/条件網羅">条件網羅</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　同値分割</div>
<div class="ap-choice-note">有効・無効の同値クラスから代表値を選ぶ技法であり、境界のずれを捉えにくい。詳細：<a href="用語/同値分析法" class="internal-link" data-href="用語/同値分析法">同値分析法</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　分岐網羅</div>
<div class="ap-choice-note">判定条件の真偽を少なくとも1回ずつ通す技法（<a href="用語/判定条件網羅" class="internal-link" data-href="用語/判定条件網羅">判定条件網羅</a>）であり、境界値の誤り検出には特化しない。詳細：<a href="用語/判定条件網羅" class="internal-link" data-href="用語/判定条件網羅">判定条件網羅</a></div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>限界値分析は、データ範囲を「有効値」及び「有効値超過」「有効値未満」の3つに分割し、<a href="用語/テストケース" class="internal-link" data-href="用語/テストケース">テストケース</a>に有効範囲の上限値とそれを1つ上回る値、および下限値とそれを1つ下回る値を利用する手法です。</p><p><img class="ap-fig" src="../../../assets/問題/午前/H30秋期/49/49.png" alt="解説図" width="337" height="104" ></p>
<p>繰返し処理の終了条件として A≧a（Aがa以上）とすべきところを A＞a（Aがaより大きい）とした場合、本来終了すべきときに終了しなかったり、処理を繰り返すべきなのに終了してしまったりということが起こります。</p>
<p>この事例のような条件式の記述誤り、またはプログラマの解釈ミスにより、条件判定の境界値付近ではバグが頻発するため、そのバグを効率的に検出するためには<a href="用語/テストケース" class="internal-link" data-href="用語/テストケース">テストケース</a>に境界値を用いる限界値分析が有効です。</p>
<p>したがって「ア」が正解です。</p>
