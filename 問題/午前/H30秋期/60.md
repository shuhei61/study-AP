<p class="ap-question"><strong>問60</strong>　情報システムの可監査性を説明したものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　コントロールの有効性を監査できるように，情報システムが設計・運用されていること</div>
<div class="ap-choice-note">正しい。情報システムが監査可能な状態に設計・運用されていることを指す</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　システム監査人が，監査の目的に合致した有効な手続を行える能力をもっていること</div>
<div class="ap-choice-note"><a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>人の能力・適格性の説明であり、情報システムの設計・運用の性質ではない</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　情報システムから入手した監査証拠の十分性と監査報告書の完成度が保たれていること</div>
<div class="ap-choice-note"><a href="用語/監査証拠" class="internal-link" data-href="用語/監査証拠">監査証拠</a>の十分性や監査報告書の完成度の話であり、情報システム側の可監査性ではない</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　情報システム部門の積極的な協力が得られること</div>
<div class="ap-choice-note">被監査部門の協力体制であり、情報システム自体の設計・運用の性質ではない</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>可監査性(Auditability)とは、ログや処理手順が完備され、業務処理の正当性や<a href="用語/内部統制" class="internal-link" data-href="用語/内部統制">内部統制</a>を効果的に監査又はレビューできるように情報システムが設計・運用されている度合いを指す言葉です。簡単に言えば「監査が実施可能であること」を意味します。<a href="用語/システム監査" class="internal-link" data-href="用語/システム監査">システム監査</a>を円滑に実施するため、情報システムは可監査性を意識して構築、整備されなければなりません。</p>
<p>したがって適切な説明は「ア」です。</p>
</div>
