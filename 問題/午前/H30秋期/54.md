<p class="ap-question"><strong>問54</strong>　ソフトウェアの機能量に着目して開発規模を見積もるファンクションポイント法で，調整前FPを求めるために必要となる情報はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　開発者数</div>
<div class="ap-choice-note">機能量ではなく要員に関する情報であり、調整前FPの算出には用いない</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　画面数</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/ファンクションポイント法" class="internal-link" data-href="用語/ファンクションポイント法">ファンクションポイント法</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　プログラムステップ数</div>
<div class="ap-choice-note"><a href="用語/LOC法" class="internal-link" data-href="用語/LOC法">LOC法</a>（プログラムステップ法）で用いる情報であり、機能量ではない</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　利用者数</div>
<div class="ap-choice-note">機能量とは無関係な利用者の人数であり、調整前FPの算出には用いない</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/ファンクションポイント法" class="internal-link" data-href="用語/ファンクションポイント法">ファンクションポイント法</a>は、システムやプログラムの機能に基づいて全体の開発規模を見積もる手法で、外部入出力や内部ファイルの数と難易度の高さから論理的にファンクションポイントを算出して規模や工数見積りを行います。利用者側に見える入出力画面や帳票を基準に見積りを行うため、依頼者側とのコンセンサス（合意）が取りやすいという特徴があります。</p>
<p>調整前FPは、開発する機能ごとに複雑さの重みを付けたものを合算した値ですので、調整前FPを求めるには機能数がわかっていなければなりません。<a href="用語/ファンクションポイント法" class="internal-link" data-href="用語/ファンクションポイント法">ファンクションポイント法</a>では、入出力データ数やファイル数を機能として扱うので、外部入出力の機能である画面数は調整前FPを求めるために必要な情報になります。</p>
<p>したがって「イ」が正解です。</p>
