<p class="ap-question"><strong>問14</strong>　コンテナ型仮想化の説明として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　アプリケーションの起動に必要なプログラムやライブラリなどをまとめ，ホストOSで動作させるので，独立性を保ちながら複数のアプリケーションを稼働できる。</div>
<div class="ap-choice-note">正しい。コンテナ型は、ホストOS上のプロセスとして作成した独立空間によって<a href="用語/仮想化" class="internal-link" data-href="用語/仮想化">仮想化</a>の仕組みを提供します。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　サーバで仮想化ソフトウェアを動かし，その上で複数のゲストOSを稼働させるので，サーバのOSとは異なるOSも稼働できる。</div>
<div class="ap-choice-note">ハイパーバイザ型の説明です。コンテナ型ではゲストOSを使いません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　サーバで実行されたアプリケーションの画面情報をクライアントに送信し，クライアントからは端末の操作情報がサーバに送信されるので，クライアントにアプリケーションをインストールしなくても利用できる。</div>
<div class="ap-choice-note">これは<a href="用語/VDI" class="internal-link" data-href="用語/VDI">VDI</a>(Virtual Desktop Infrastructure)の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　ホストOSで仮想化ソフトウェアを動かし，その上で複数のゲストOSを稼働させるので，物理サーバへアクセスするにはホストOSを経由する必要がある。</div>
<div class="ap-choice-note">ホストOS型の説明です。コンテナ型ではゲストOSを使いません。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/仮想化" class="internal-link" data-href="用語/仮想化">仮想化</a>のアーキテクチャは、ハイパーバイザ型、ホスト型、コンテナ型に大別されます。</p><p><img class="ap-fig" src="../../../assets/問題/午前/R3秋期/14/14.png" alt="解説図" width="580" height="209" ></p>
<p>コンテナ型は、ホストOS上にコンテナという互いに独立した空間を用意し、その空間上でライブラリやアプリケーションを動かす仕組みです。ゲストOSを起動する必要がないため低リソースで<a href="用語/仮想化" class="internal-link" data-href="用語/仮想化">仮想化</a>環境を構築することができます。しかし、ホストOSとは別のOSの<a href="用語/仮想環境" class="internal-link" data-href="用語/仮想環境">仮想環境</a>を構築することができないという制約もあります。</p>
<p>アは、上記のとおりコンテナ型の説明であり正解です。</p>
<p>イは、ハイパーバイザ型の説明です。コンテナ型ではゲストOSを使いません。</p>
<p>ウは、<a href="用語/VDI" class="internal-link" data-href="用語/VDI">VDI</a>(Virtual Desktop Infrastructure)の説明です。</p>
<p>エは、ホストOS型の説明です。コンテナ型ではゲストOSを使いません。</p>
</div>
