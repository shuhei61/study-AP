#問題 #マネジメント #システム監査 #システム監査

<p class="ap-question"><strong>問58</strong>　システム利用者に対して付与されるアクセス権の管理状況の監査で判明した状況のうち，監査人がシステム監査報告書で報告すべき指摘事項はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　アクセス権を付与された利用者ID・パスワードに関して，システム利用者が遵守すべき事項が規程として定められ，システム利用者に周知されていた。</div>
<div class="ap-choice-note">適切な管理であり<a href="用語/指摘事項" class="internal-link" data-href="用語/指摘事項">指摘事項</a>ではない</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　業務部門長によって，所属するシステム利用者に対するアクセス権の付与状況のレビューが定期的に行われていた。</div>
<div class="ap-choice-note">適切な管理であり<a href="用語/指摘事項" class="internal-link" data-href="用語/指摘事項">指摘事項</a>ではない</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　システム利用者に対するアクセス権の付与・変更・削除に関する管理手続が，規程として定められていた。</div>
<div class="ap-choice-note">適切な管理であり<a href="用語/指摘事項" class="internal-link" data-href="用語/指摘事項">指摘事項</a>ではない</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　退職・異動したシステム利用者に付与されていたアクセス権の削除・変更は，定期人事異動がある年度初めに全てまとめて行われていた。</div>
<div class="ap-choice-note">正しい。不要となった<a href="用語/アクセス権" class="internal-link" data-href="用語/アクセス権">アクセス権</a>はただちに削除・変更すべきであり，年度初めにまとめて行うのは<a href="用語/指摘事項" class="internal-link" data-href="用語/指摘事項">指摘事項</a></div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>異動または退職によって不要となった利用者IDが削除されていないと、元の利用者によって不正に利用されて、重要情報にアクセスされる恐れがあります。このため不要となった利用者IDおよび<a href="用語/アクセス権" class="internal-link" data-href="用語/アクセス権">アクセス権</a>は、ただちに削除・変更しなければなりません。</p>
<p>「エ」のケースでは、<a href="用語/アクセス権" class="internal-link" data-href="用語/アクセス権">アクセス権</a>の削除・変更処理を「年度初めに全てまとめて」行っていたので、この点が<a href="用語/指摘事項" class="internal-link" data-href="用語/指摘事項">指摘事項</a>に該当します。</p>
</div>
