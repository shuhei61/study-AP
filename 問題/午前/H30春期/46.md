<p class="ap-question"><strong>問46</strong>　ソフトウェアライフサイクルプロセスにおいてソフトウェア実装プロセスを構成するプロセスのうち，次のタスクを実施するものはどれか。<br><br>〔タスク〕<br>ソフトウェア品目の外部インタフェース，及びソフトウェアコンポーネント間のインタフェースについて最上位レベルの設計を行う。<br>データベースについて最上位レベルの設計を行う。<br>ソフトウェア結合のために暫定的なテスト要求事項及びスケジュールを定義する。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　ソフトウェア結合プロセス</div>
<div class="ap-choice-note"><a href="用語/ソフトウェアユニット" class="internal-link" data-href="用語/ソフトウェアユニット">ソフトウェアユニット</a>及び構成部品を組み合わせるプロセスであり、最上位レベルの設計は行わない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　ソフトウェア構築プロセス</div>
<div class="ap-choice-note">実行可能な<a href="用語/ソフトウェアユニット" class="internal-link" data-href="用語/ソフトウェアユニット">ソフトウェアユニット</a>を作り出すプロセスであり、設計は行わない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　ソフトウェア詳細設計プロセス</div>
<div class="ap-choice-note"><a href="用語/コーディング" class="internal-link" data-href="用語/コーディング">コーディング</a>可能なレベルまで詳細化する設計であり、「最上位レベルの設計」ではない。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　ソフトウェア方式設計プロセス</div>
<div class="ap-choice-note">正しい。要求事項を実装し検証できるソフトウェアの設計（最上位レベルの構造）を提供する。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>ソフトウェアライフサイクルプロセスを定めた<a href="用語/JIS X 0160" class="internal-link" data-href="用語/JIS X 0160">JIS X 0160</a>によれば、ソフトウェア実装プロセスは「ソフトウェア製品又はソフトウェアサービスとして実現される指定のシステム要素を作り出すことを目的とする」と説明されており、次の6つの下位プロセス※を内包します。</p>
<dl>
<dt>ソフトウェア要求事項分析プロセス</dt>
<dd>システムの<a href="用語/ソフトウェア要素" class="internal-link" data-href="用語/ソフトウェア要素">ソフトウェア要素</a>の要求事項を確立する</dd>
<dt>ソフトウェア方式設計プロセス</dt>
<dd>要求事項を実装し，それに対して検証できるソフトウェアの設計を提供する</dd>
<dt>ソフトウェア詳細設計プロセス</dt>
<dd>要求事項及びソフトウェア方式に対して実装し，検証でき，<a href="用語/コーディング" class="internal-link" data-href="用語/コーディング">コーディング</a>及びテストを可能にするために十分に詳細である設計をソフトウェアのために提供する</dd>
<dt>ソフトウェア構築プロセス</dt>
<dd>ソフトウェア設計を適切に反映した実行可能な<a href="用語/ソフトウェアユニット" class="internal-link" data-href="用語/ソフトウェアユニット">ソフトウェアユニット</a>を作り出す</dd>
<dt>ソフトウェア結合プロセス</dt>
<dd><a href="用語/ソフトウェアユニット" class="internal-link" data-href="用語/ソフトウェアユニット">ソフトウェアユニット</a>及び構成部品を組み合わせ，結合されたソフトウェア品目を作り出す</dd>
<dt>ソフトウェア適格性確認テストプロセス</dt>
<dd>結合されたソフトウェア製品がその定義された要求事項を満たすことを確認する</dd>
</dl>
<p>設問中の「設計を行う」という記述から、正解候補は2つの設計プロセス(方式設計／詳細設計)に絞られます。「ソフトウェア方式設計」では、ソフトウェアに対する要件をどのようにソフトウェアコンポーネントに割り振るか、というソフトウェアの最上位レベルの構造を決定します。一方、「ソフトウェア詳細設計」は、ソフトウェア方式設計で定義された各ソフトウェアコンポーネントを<a href="用語/コーディング" class="internal-link" data-href="用語/コーディング">コーディング</a>可能なレベルまで詳細化します。設問中には「最上位レベルの設計」という記述があるためソフトウェア方式設計プロセスが適切と分かります。</p>
<p>※6つの下位プロセスは共通フレームの開発プロセスのうち「<a href="用語/ソフトウェア要件" class="internal-link" data-href="用語/ソフトウェア要件">ソフトウェア要件</a>定義」から「ソフトウェア適格性確認テスト」に相当します。</p>
</div>
