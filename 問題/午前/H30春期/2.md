<p class="ap-question"><strong>問2</strong>　表は，文字A～Eを符号化したときのビット表記と，それぞれの文字の出現確率を表したものである。1文字当たりの平均ビット数は幾らになるか。<br><img class="ap-fig" src="../../../assets/問題/午前/H30春期/2/02.png" alt="問題図" width="240" height="128" ></p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　1.6</div>
<div class="ap-choice-note">各文字のビット数×出現確率の総和は1.8であり、1.6にはならない。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　1.8</div>
<div class="ap-choice-note">正しい。ビット数×出現確率の総和が1.8ビットになる。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　2.5</div>
<div class="ap-choice-note">各文字のビット数×出現確率の総和は1.8であり、2.5にはならない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　2.8</div>
<div class="ap-choice-note">各文字のビット数×出現確率の総和は1.8であり、2.8にはならない。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>各文字を表すビット数とその出現確率をかけたものを足し合わせて平均ビット数を求めます。<br><br>A → 1ビット×0.5＝0.5ビット<br>B → 2ビット×0.3＝0.6ビット<br>C → 3ビット×0.1＝0.3ビット<br>D → 4ビット×0.05＝0.2ビット<br>E → 4ビット×0.05＝0.2ビット<br><br>すべてを足し合わせると、<br>0.5＋0.6＋0.3＋0.2＋0.2＝1.8ビット<br><br>したがって、平均ビット数は1.8ビットになります。<br><br>このように情報の出現確率が高いデータには短い符号を、低いデータには長い符号を与えることで圧縮を効率よく行う方法を<a href="用語/ハフマン符号" class="internal-link" data-href="用語/ハフマン符号">ハフマン符号</a>といいます。</p>
