<p class="ap-question"><strong>問34</strong>　TCP/IPネットワークにおけるRARPの機能として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　IPパケットが通信先のIPアドレスに到達するかどうかを調べる。</div>
<div class="ap-choice-note">これは<a href="用語/ICMP" class="internal-link" data-href="用語/ICMP">ICMP</a>の説明です</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　MACアドレスからIPアドレスを求める。</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/RARP" class="internal-link" data-href="用語/RARP">RARP</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　ドメイン名とホスト名からIPアドレスを求める。</div>
<div class="ap-choice-note">これは<a href="用語/DNS" class="internal-link" data-href="用語/DNS">DNS</a>の説明です</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　プライベートIPアドレスとグローバルIPアドレスを相互に変換する。</div>
<div class="ap-choice-note">これは<a href="用語/NAT" class="internal-link" data-href="用語/NAT">NAT</a>の説明です</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/RARP" class="internal-link" data-href="用語/RARP">RARP</a>(Reverse Address Resolution Protocol)は、<a href="用語/ARP" class="internal-link" data-href="用語/ARP">ARP</a>が<a href="用語/IPアドレス" class="internal-link" data-href="用語/IPアドレス">IPアドレス</a>から機器のMACアドレスを得るのとは逆に、機器固有のMACアドレスから対応する<a href="用語/IPアドレス" class="internal-link" data-href="用語/IPアドレス">IPアドレス</a>を取得するためのプロトコルです。</p>
</div>
