<p class="ap-question"><strong>問9</strong>　プロセッサの高速化技法の一つとして，同時に実行可能な複数の動作を，コンパイルの段階でまとめて一つの複合命令とし，高速化を図る方式はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　CISC</div>
<div class="ap-choice-note">Complex Instruction Set Computerの略。複雑で多機能な命令セットでコンピュータを構成する設計方法です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　MIMD</div>
<div class="ap-choice-note">詳細：<a href="用語/MIMD" class="internal-link" data-href="用語/MIMD">MIMD</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　RISC</div>
<div class="ap-choice-note">Reduced Instruction Set Computerの略。命令を短く固定長化し、専用の論理回路で高速に実行できるようにした設計方法です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　VLIW</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/VLIW" class="internal-link" data-href="用語/VLIW">VLIW</a></div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/VLIW" class="internal-link" data-href="用語/VLIW">VLIW</a>(Very Long Instruction Word)とは、プログラムのコンパイル段階で、依存関係のない複数の操作をまとめて1つの複合命令にまとめ、同時に並列処理させる手法です。1つのサイクルで多くの演算を行うことができるため、高速なデータ処理が求められる分野で効果を発揮します。「Very Long」の名前のとおり命令がとても長く、最低128ビットといったサイズになります。</p>
<p>通常のパイプライン処理では、<a href="用語/CPU" class="internal-link" data-href="用語/CPU">CPU</a>が実行中に命令の依存関係などを判断して進めますが、<a href="用語/VLIW" class="internal-link" data-href="用語/VLIW">VLIW</a>ではあらかじめ並列実行できる形にしてあるため、<a href="用語/CPU" class="internal-link" data-href="用語/CPU">CPU</a>側のオーバーヘッド減少と高速化が望めます。その一方で、ソースコードからプログラムを生成する<a href="用語/コンパイラ" class="internal-link" data-href="用語/コンパイラ">コンパイラ</a>の設計はその分難しくなります。</p>
<p>したがって「エ」が正解です。</p>
