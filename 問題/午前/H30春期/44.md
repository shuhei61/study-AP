<p class="ap-question"><strong>問44</strong>　パケットフィルタリング型ファイアウォールのフィルタリングルールを用いて，本来必要なサービスに影響を及ぼすことなく防げるものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　外部に公開していないサーバへのアクセス</div>
<div class="ap-choice-note">正しい。公開しないサービスのポートを遮断するルールで、必要サービスに影響なく防げる</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　サーバで動作するソフトウェアの脆弱性を突く攻撃</div>
<div class="ap-choice-note">正当な処理要求も遮断してしまうため、必要サービスに影響なく防げない</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　電子メールに添付されたファイルに含まれるマクロウイルスの侵入</div>
<div class="ap-choice-note"><a href="用語/SMTP" class="internal-link" data-href="用語/SMTP">SMTP</a>を遮断すると必要メールも止まるため、影響なく防げない</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　不特定多数のIoT機器から大量のHTTPリクエストを送り付けるDDoS攻撃</div>
<div class="ap-choice-note"><a href="用語/HTTP" class="internal-link" data-href="用語/HTTP">HTTP</a>を遮断すると外部利用者のWeb利用もできなくなるため不適切</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/パケットフィルタリング" class="internal-link" data-href="用語/パケットフィルタリング">パケットフィルタリング</a>とは、通過する<a href="用語/パケット" class="internal-link" data-href="用語/パケット">パケット</a>の<a href="用語/IPアドレス" class="internal-link" data-href="用語/IPアドレス">IPアドレス</a>(送信元・送信先)や<a href="用語/ポート番号" class="internal-link" data-href="用語/ポート番号">ポート番号</a>、通信の方向などの情報をもとに中継の可否を判断する方式です。ただし、<a href="用語/パケット" class="internal-link" data-href="用語/パケット">パケット</a>のペイロード(データ部分)に関してはチェックを行いません。</p>
<ul>
<li>ア: 正しい。外部に公開していないサービス、若しくは業務に不要なサービスのポートへの通信を遮断するルールを設定することで、必要なサービスに影響を与えることなく<a href="用語/不正アクセス" class="internal-link" data-href="用語/不正アクセス">不正アクセス</a>を防ぐことが可能です。</li>
<li>イ: <a href="用語/IPアドレス" class="internal-link" data-href="用語/IPアドレス">IPアドレス</a>と<a href="用語/ポート番号" class="internal-link" data-href="用語/ポート番号">ポート番号</a>の組みを指定することで、<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>のあるソフトウェアへの攻撃を目的とする通信を遮断できますが、正当な処理要求についても遮断してしまうことになるため不適切です。</li>
<li>ウ: <a href="用語/電子メール" class="internal-link" data-href="用語/電子メール">電子メール</a>を転送する<a href="用語/SMTP" class="internal-link" data-href="用語/SMTP">SMTP</a>(<a href="用語/TCP" class="internal-link" data-href="用語/TCP">TCP</a>/25)に対する攻撃を防ぐことはできますが、それと同時に業務に必要なメール転送もできなくなってしまうため不適切です。</li>
<li>エ: 「ウ」と同様に、<a href="用語/HTTP" class="internal-link" data-href="用語/HTTP">HTTP</a>(<a href="用語/TCP" class="internal-link" data-href="用語/TCP">TCP</a>/80)宛の通信を遮断してしまうと、外部の利用者がWebサービスを利用できなくなってしまうため不適切です。</li>
</ul>
</div>
