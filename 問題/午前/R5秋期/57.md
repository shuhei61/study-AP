#問題 #マネジメント #サービスマネジメント #サービスの運用

<p class="ap-question"><strong>問57</strong>　フルバックアップ方式と差分バックアップ方式とを用いた運用に関する記述のうち，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　障害からの復旧時に差分バックアップのデータだけ処理すればよいので，フルバックアップ方式に比べ，差分バックアップ方式は復旧時間が短い。</div>
<div class="ap-choice-note"><a href="用語/差分バックアップ" class="internal-link" data-href="用語/差分バックアップ">差分バックアップ</a>方式では、<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>の適用後にさらに<a href="用語/差分バックアップ" class="internal-link" data-href="用語/差分バックアップ">差分バックアップ</a>を適用する必要があるため、<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>方式のみの運用より復旧時間は長くなる。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　フルバックアップのデータで復元した後に，差分バックアップのデータを反映させて復旧する。</div>
<div class="ap-choice-note">正しい。<a href="用語/差分バックアップ" class="internal-link" data-href="用語/差分バックアップ">差分バックアップ</a>方式では、<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>を適用した後、<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>以降に更新されたファイルを<a href="用語/差分バックアップ" class="internal-link" data-href="用語/差分バックアップ">差分バックアップ</a>で反映して復旧する。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　フルバックアップ方式と差分バックアップ方式とを併用して運用することはできない。</div>
<div class="ap-choice-note">2つの方式を併用できる。<a href="用語/差分バックアップ" class="internal-link" data-href="用語/差分バックアップ">差分バックアップ</a>のみだと復旧に時間がかかりすぎるため、定期的に<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>を行うのが通常である。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　フルバックアップ方式に比べ，差分バックアップ方式はバックアップに要する時間が長い。</div>
<div class="ap-choice-note"><a href="用語/差分バックアップ" class="internal-link" data-href="用語/差分バックアップ">差分バックアップ</a>方式は、<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>以降に変更のあったファイル（差分）だけをバックアップするため、1回あたりのバックアップ時間は<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>方式より短くなる。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>2つのバックアップ方式は次のようなものです。</p>
<p><strong><a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>方式</strong>　毎回データ全体のバックアップを行う方式。復旧時間は短くなるが、バックアップに要する時間は長い。</p>
<p><strong><a href="用語/差分バックアップ" class="internal-link" data-href="用語/差分バックアップ">差分バックアップ</a>方式</strong>　定期的に<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>を行い、次の<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>までの期間は、<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>以降に変更のあったファイル（差分）だけを記録していく方式。バックアップ時間は短くて済むが、復旧は①<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>の適用→②<a href="用語/差分バックアップ" class="internal-link" data-href="用語/差分バックアップ">差分バックアップ</a>の適用という流れになるので、<a href="用語/フルバックアップ" class="internal-link" data-href="用語/フルバックアップ">フルバックアップ</a>方式と比較して作業時間がかかる。</p>
</div>
</div>
