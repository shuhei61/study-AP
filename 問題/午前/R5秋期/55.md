<p class="ap-question"><strong>問55</strong>　サービスマネジメントシステム(SMS)における是正処置の説明はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　検出された不適合又はほかの望ましくない状況の原因を除去する，又は再発の起こりやすさを低減するための処置</div>
<div class="ap-choice-note">正しい。<a href="用語/是正処置" class="internal-link" data-href="用語/是正処置">是正処置</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　構成品目の変更の展開に伴って，構成情報を更新する処置</div>
<div class="ap-choice-note"><a href="用語/構成管理" class="internal-link" data-href="用語/構成管理">構成管理</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　パフォーマンスを向上するために繰り返し行われる活動であって，SMS及びサービスの適切性，妥当性及び有効性を継続的に改善するための処置</div>
<div class="ap-choice-note"><a href="用語/継続的改善" class="internal-link" data-href="用語/継続的改善">継続的改善</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　問題を"記録・分類"し，"優先度付け"し，"必要ならばエスカレーション"し，"可能ならば解決する"一連の処置</div>
<div class="ap-choice-note"><a href="用語/インシデント管理" class="internal-link" data-href="用語/インシデント管理">インシデント管理</a>の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/サービスマネジメント" class="internal-link" data-href="用語/サービスマネジメント">サービスマネジメント</a>における<a href="用語/是正処置" class="internal-link" data-href="用語/是正処置">是正処置</a>とは、是正（悪い点を改めて正しくする）の言葉どおり、<a href="用語/モニタリング" class="internal-link" data-href="用語/モニタリング">モニタリング</a>や<a href="用語/監査" class="internal-link" data-href="用語/監査">監査</a>において<a href="用語/サービスの要求事項" class="internal-link" data-href="用語/サービスの要求事項">サービスの要求事項</a>への<a href="用語/不適合" class="internal-link" data-href="用語/不適合">不適合</a>が検出された場合に、それらを管理し、修正することで、要求事項を満たす状態にする活動をいいます。問題発生を受けて行うリアクティブなアクションであることが特徴です。</p>
<p>したがって「ア」が正解です。</p>
</div>
