<p class="ap-question"><strong>問49</strong>　アジャイルソフトウェア開発宣言では，"あることがらに価値があることを認めながらも別のことがらにより価値をおく"としている。"別のことがら"に該当するものの組みはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　個人と対話，動くソフトウェア，顧客との協調，変化への対応</div>
<div class="ap-choice-note">正しい。宣言の4つの価値で、左側（プロセスやツール、包括的なドキュメント、契約交渉、計画に従うこと）より右側に価値を置くものの組合せです。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　個人と対話，包括的なドキュメント，顧客との協調，計画に従うこと</div>
<div class="ap-choice-note">包括的なドキュメントと計画に従うことは、宣言で「価値がある」と認めつつも、それより優先しない側（左側）の組合せです。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　プロセスやツール，動くソフトウェア，契約交渉，変化への対応</div>
<div class="ap-choice-note">プロセスやツールと契約交渉は左側の組合せです。動くソフトウェアと変化への対応だけでは「別のことがら」の4つにはなりません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　プロセスやツール，包括的なドキュメント，契約交渉，計画に従うこと</div>
<div class="ap-choice-note">4つとも宣言の左側（あることがら）の組合せであり、「別のことがら」ではありません。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/アジャイルソフトウェア開発宣言" class="internal-link" data-href="用語/アジャイルソフトウェア開発宣言">アジャイルソフトウェア開発宣言</a>は、2001年に米国で開催されたソフトウェア開発者の会議において、17名の開発者によって策定された文書であり、<a href="用語/アジャイル" class="internal-link" data-href="用語/アジャイル">アジャイル</a>開発における基本的な考え方（マインドセット）を示したものです。</p>
<p>本宣言は、ソフトウェア開発における4つの価値と、それに基づく12の原則が提示されています。4つの価値は以下のとおりです。</p>
<ul>
<li>プロセスやツールよりも個人と対話に価値を置く（対面コミュニケーション）</li>
<li>包括的なドキュメントよりも動くソフトウェアに価値を置く（実働検証）</li>
<li>契約交渉よりも顧客との協調に価値を置く（顧客とのWin-Win関係）</li>
<li>計画に従うことよりも変化への対応に価値を置く（変化を味方に）</li>
</ul>
<p>上記をまとめると、<a href="用語/アジャイル" class="internal-link" data-href="用語/アジャイル">アジャイル</a>開発においてより価値を置くものは、①個人と対話、②動くソフトウェア、③顧客との協調、④変化への対応の4つとなります。</p>
<p>したがって「ア」の組合せが正解です。</p>
