<p class="ap-question"><strong>問26</strong>　"売上"表への次の検索処理のうち，B<sup>+</sup>木インデックスよりもハッシュインデックスを設定した方が適切なものはどれか。ここで，インデックスを設定する列を＜＞内に示す。<br><br>　売上 (伝票番号，売上年月日，商品名，利用者ID，店舗番号，売上金額)</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　売上金額が1万円以上の売上を検索する。＜売上金額＞</div>
<div class="ap-choice-note">売上金額が1万円以上の<a href="用語/レコード" class="internal-link" data-href="用語/レコード">レコード</a>を取得するのは条件検索に当たるので、<a href="用語/ハッシュインデックス" class="internal-link" data-href="用語/ハッシュインデックス">ハッシュインデックス</a>には不向きです。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　売上年月日が今月の売上を検索する。＜売上年月日＞</div>
<div class="ap-choice-note">売上年月日が今月の<a href="用語/レコード" class="internal-link" data-href="用語/レコード">レコード</a>を取得するのは範囲検索に当たるので、<a href="用語/ハッシュインデックス" class="internal-link" data-href="用語/ハッシュインデックス">ハッシュインデックス</a>には不向きです。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　商品名が'DB'で始まる売上を検索する。＜商品名＞</div>
<div class="ap-choice-note">商品名が'DB'で始まる<a href="用語/レコード" class="internal-link" data-href="用語/レコード">レコード</a>を取得するのは条件検索に当たるので、<a href="用語/ハッシュインデックス" class="internal-link" data-href="用語/ハッシュインデックス">ハッシュインデックス</a>には不向きです。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　利用者IDが'1001'の売上を検索する。＜利用者ID＞</div>
<div class="ap-choice-note">正しい。キー値に関連付けられた<a href="用語/レコード" class="internal-link" data-href="用語/レコード">レコード</a>を検索するので、<a href="用語/ハッシュインデックス" class="internal-link" data-href="用語/ハッシュインデックス">ハッシュインデックス</a>を有効に活用できる処理です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/インデックス" class="internal-link" data-href="用語/インデックス">インデックス</a>は、<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>に格納されているデータの検索を高速に行うための仕組みです。<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>内のテーブルや列に関する特定のキー値を格納し、その値を元にデータの高速な検索や並替えを可能にします。<a href="用語/インデックス" class="internal-link" data-href="用語/インデックス">インデックス</a>を使用することで、<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>のアクセス効率を高めることができますが、<a href="用語/インデックス" class="internal-link" data-href="用語/インデックス">インデックス</a>の利用に際しては追加のディスクスペースや処理が必要となるので、適切な設計管理が求められます。</p>
<p>B<sup>+</sup>木<a href="用語/インデックス" class="internal-link" data-href="用語/インデックス">インデックス</a>は<a href="用語/木構造" class="internal-link" data-href="用語/木構造">木構造</a>を使ってデータを管理する方式で、RDBMSで最も一般的に使用されています。節ごとにキー値の範囲とその子要素へのポインタを保持しているので、範囲検索をスムーズに行えます。<a href="用語/木構造" class="internal-link" data-href="用語/木構造">木構造</a>の深さが一定なので探索コストが大きく変わらず、<a href="用語/インデックス" class="internal-link" data-href="用語/インデックス">インデックス</a>ノードはソートされた状態なので整列処理も高速です。</p>
<p><a href="用語/ハッシュインデックス" class="internal-link" data-href="用語/ハッシュインデックス">ハッシュインデックス</a>は<a href="用語/ハッシュ関数" class="internal-link" data-href="用語/ハッシュ関数">ハッシュ関数</a>を使用してキー値と<a href="用語/レコード" class="internal-link" data-href="用語/レコード">レコード</a>の格納位置を直接関連付ける方式です。キー値をもとに<a href="用語/レコード" class="internal-link" data-href="用語/レコード">レコード</a>の格納位置を一意に特定できるので、B<sup>+</sup>木<a href="用語/インデックス" class="internal-link" data-href="用語/インデックス">インデックス</a>より高速なアクセスが可能です。その反面、範囲検索やキー値を順番に読み込んで処理を行う用途には使用できません。</p>
<p>アは1万円以上、イは今月の日付範囲、ウは前方一致の条件検索であり、いずれも<a href="用語/ハッシュインデックス" class="internal-link" data-href="用語/ハッシュインデックス">ハッシュインデックス</a>には不向きです。エは利用者IDが'1001'という等値条件で<a href="用語/レコード" class="internal-link" data-href="用語/レコード">レコード</a>を特定する検索なので、<a href="用語/ハッシュインデックス" class="internal-link" data-href="用語/ハッシュインデックス">ハッシュインデックス</a>が有効です。</p>
