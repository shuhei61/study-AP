#問題 #マネジメント #システム監査 #システム監査

<p class="ap-question"><strong>問59</strong>　販売管理システムにおいて，起票された受注伝票の入力が，漏れなく，かつ，重複することなく実施されていることを確かめる監査手続として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　受注データから値引取引データなどの例外取引データを抽出し，承認の記録を確かめる。</div>
<div class="ap-choice-note">例外処理の修正と再処理に関するコントロールであり、受注伝票の入力の漏れ・重複の確認とは関係がない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　受注伝票の入力時に論理チェック及びフォーマットチェックが行われているか，テストデータ法で確かめる。</div>
<div class="ap-choice-note"><a href="用語/テストデータ法" class="internal-link" data-href="用語/テストデータ法">テストデータ法</a>は処理の正当性を確認する手続であり、入力の漏れや重複は確認できない。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　販売管理システムから出力したプルーフリストと受注伝票との照合が行われているか，プルーフリストと受注伝票上の照合印を確かめる。</div>
<div class="ap-choice-note">正しい。プルーフリストと原票の照合により、入力が漏れや重複なく実施されていることを確認できる。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　並行シミュレーション法を用いて，受注伝票を処理するプログラムの論理の正当性を確かめる。</div>
<div class="ap-choice-note">並行シミュレーション法はプログラムの論理の正当性を確かめる手続であり、入力の漏れや重複は確認できない。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>プルーフリストは、入力データを加工せずにそのまま印刷した文書で、マスタデータへの登録時に原票と照合することにより、入力が原票どおりに行われたかどうかを確認するために用いられます。プルーフリストと受注伝票との照合が行われているか、照合の記録があるかを確認すれば、入力が漏れや重複なく実施されていることを確認できます。</p>
<p>アは例外処理に関するコントロール、イは<a href="用語/テストデータ法" class="internal-link" data-href="用語/テストデータ法">テストデータ法</a>、エは並行シミュレーション法であり、いずれも本問の「入力の漏れ・重複」の確認には適さないため、ウが正解です。</p>
</div>
