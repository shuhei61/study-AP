<p class="ap-question"><strong>問52</strong>　システム開発のプロジェクトにおいて，EVMを活用したパフォーマンス管理をしている。開発途中のある時点でEV－PVの値が負であるとき，どのような状況を示しているか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　スケジュール効率が，計画よりも良い。</div>
<div class="ap-choice-note">スケジュール効率（SPI）は EV／PV で求める。EV－PVが負なら SPI は 1 より小さく、スケジュール効率は計画より悪い。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　プロジェクトの完了が，計画よりも遅くなる。</div>
<div class="ap-choice-note">現時点での進捗遅れが、プロジェクト完了にどの程度影響するかまでは判断できない。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　プロジェクトの進捗が，計画よりも遅れている。</div>
<div class="ap-choice-note">正しい。EV－PV の値が負なので、計画よりも進捗に遅れが生じている。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　プロジェクトの進捗が，計画よりも進んでいる。</div>
<div class="ap-choice-note">進捗が計画より進んでいる場合は、EV－PV の値が正になる。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/EVM" class="internal-link" data-href="用語/EVM">EVM</a>（Earned Value Management）は、プロジェクトにおける作業を金銭の価値に置き換えて定量的に実績管理をする<a href="用語/進捗管理" class="internal-link" data-href="用語/進捗管理">進捗管理</a>手法です。PV、EV、AC の 3 つの指標を用いて管理するのが特徴です。</p>
<ul>
<li>PV（Planned Value）… プロジェクト開始当初、現時点までに計画されていた作業に対する予算</li>
<li>EV（Earned Value）… 現時点までに完了した作業に割り当てられていた予算</li>
<li>AC（Actual Cost）… 現時点までに完了した作業に対して実際に投入した総コスト</li>
</ul>
<p>各指標を比較して、EV－AC がマイナス値であれば完了済み作業に対する予算よりも投入コストが多いのでコスト超過、EV－PV がマイナス値であれば完了済み作業に対する予算が当初の予算よりも少ないので進捗遅れ、と判断できます。</p>
</div>
