<p class="ap-question"><strong>問7</strong>　JavaScriptのオブジェクトの表記法などを基にして規定したものであって，"名前と値との組みの集まり"と"値の順序付きリスト"の二つの構造に基づいてオブジェクトを表現する，データ記述の仕様はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　DOM</div>
<div class="ap-choice-note">Document Object Modelの略。<a href="用語/HTML" class="internal-link" data-href="用語/HTML">HTML</a>や<a href="用語/XML" class="internal-link" data-href="用語/XML">XML</a>で記述された各要素をアプリケーションプログラムから取り扱うための<a href="用語/API" class="internal-link" data-href="用語/API">API</a>であり、データ記述の仕様ではない。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　JSON</div>
<div class="ap-choice-note">正しい。JavaScript Object Notationは、名前と値の組と値の順序付きリストの二つの構造でデータを表現する<a href="用語/データ形式" class="internal-link" data-href="用語/データ形式">データ形式</a>です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　SOAP</div>
<div class="ap-choice-note">分散システムやWebサービスで<a href="用語/XML" class="internal-link" data-href="用語/XML">XML</a>データを使ってメッセージをやり取りする<a href="用語/通信プロトコル" class="internal-link" data-href="用語/通信プロトコル">通信プロトコル</a>であり、データ記述の仕様ではない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　XML</div>
<div class="ap-choice-note">eXtensible Markup Languageの略。ユーザーが独自に定義したタグを用いて文書構造を記述する<a href="用語/マークアップ言語" class="internal-link" data-href="用語/マークアップ言語">マークアップ言語</a>であり、JavaScript由来のデータ記述仕様ではない。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/JSON" class="internal-link" data-href="用語/JSON">JSON</a> (JavaScript Object Notation) は、以下の例のように": (コロン)"で連結した名前と値の組を", (コンマ)"で区切って指定する<a href="用語/データ形式" class="internal-link" data-href="用語/データ形式">データ形式</a>です。</p>
<p>{<br>　名前1: 値1,<br>　名前2: 値2,<br>　名前3: [値5, 値6],<br>　名前4: {名前7: 値7, 名前8: 値8}<br>}</p>
<p>値には、単純なスカラ値や真偽値のほか、<a href="用語/配列" class="internal-link" data-href="用語/配列">配列</a>やオブジェクト（連想<a href="用語/配列" class="internal-link" data-href="用語/配列">配列</a>・ハッシュリスト）を指定できるため多次元<a href="用語/配列" class="internal-link" data-href="用語/配列">配列</a>や複雑な<a href="用語/データ構造" class="internal-link" data-href="用語/データ構造">データ構造</a>を表現することができます。元々JavaScriptにおけるオブジェクト定義用の構文という位置付けでしたが、その軽量性と汎用性からRFC8259として<a href="用語/標準化" class="internal-link" data-href="用語/標準化">標準化</a>され、多くのプログラミング言語で利用されるようになりました。</p>
<p>アの<a href="用語/DOM" class="internal-link" data-href="用語/DOM">DOM</a>は<a href="用語/HTML" class="internal-link" data-href="用語/HTML">HTML</a>や<a href="用語/XML" class="internal-link" data-href="用語/XML">XML</a>の各要素をプログラムから取り扱うための<a href="用語/API" class="internal-link" data-href="用語/API">API</a>、ウの<a href="用語/SOAP" class="internal-link" data-href="用語/SOAP">SOAP</a>はWebサービス向けの<a href="用語/通信プロトコル" class="internal-link" data-href="用語/通信プロトコル">通信プロトコル</a>、エの<a href="用語/XML" class="internal-link" data-href="用語/XML">XML</a>はタグで文書構造を記述する<a href="用語/マークアップ言語" class="internal-link" data-href="用語/マークアップ言語">マークアップ言語</a>です。本問が問う「名前と値の組みの集まり」と「値の順序付きリスト」に基づくデータ記述の仕様は<a href="用語/JSON" class="internal-link" data-href="用語/JSON">JSON</a>です。したがって正解は「イ」です。</p>
</div>
