<p class="ap-question"><strong>問27</strong>　RDBMSにおいて，特定の利用者だけに表を更新する権限を与える方法として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　CONNECT文で接続を許可する。</div>
<div class="ap-choice-note">CONNECT文は、データベースへの接続を確立する<a href="用語/SQL" class="internal-link" data-href="用語/SQL">SQL</a>文です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　CREATE ASSERTION文で表明して制限する。</div>
<div class="ap-choice-note">CREATE ASSERTION文は、スキーマ内の複数のテーブルに対して制約を適用する<a href="用語/SQL" class="internal-link" data-href="用語/SQL">SQL</a>文です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　CREATE TABLE文の参照制約で制限する。</div>
<div class="ap-choice-note"><a href="用語/参照制約" class="internal-link" data-href="用語/参照制約">参照制約</a>は、表間の整合性を確保するために設定される制約です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　GRANT文で許可する。</div>
<div class="ap-choice-note">正しい。<a href="用語/GRANT文" class="internal-link" data-href="用語/GRANT文">GRANT文</a>は、特定のユーザーに表などのデータベースオブジェクトに対する権限を付与する<a href="用語/SQL" class="internal-link" data-href="用語/SQL">SQL</a>文です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>正しい。<a href="用語/GRANT文" class="internal-link" data-href="用語/GRANT文">GRANT文</a>は、特定のユーザーに表などのデータベースオブジェクトに対する権限を付与する<a href="用語/SQL" class="internal-link" data-href="用語/SQL">SQL</a>文です。</p>
<p>GRANT オブジェクトの権限 ON オブジェクト名<br>TO { ユーザー名 | ロール名 | PUBLIC }<br>[ WITH GRANT OPTION ]</p>
<p>CONNECT文は、データベースへの接続を確立する<a href="用語/SQL" class="internal-link" data-href="用語/SQL">SQL</a>文です。</p>
<p>CREATE ASSERTION文は、スキーマ内の複数のテーブルに対して制約を適用する<a href="用語/SQL" class="internal-link" data-href="用語/SQL">SQL</a>文です。</p>
<p><a href="用語/参照制約" class="internal-link" data-href="用語/参照制約">参照制約</a>は、表間の整合性を確保するために設定される制約です。</p>
</div>
