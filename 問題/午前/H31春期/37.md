<p class="ap-question"><strong>問37</strong>　リスクベース認証の特徴はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　いかなる環境からの認証の要求においても認証方法を変更せずに，同一の手順によって普段どおりにシステムが利用できる。</div>
<div class="ap-choice-note">これは<a href="用語/RADIUS" class="internal-link" data-href="用語/RADIUS">RADIUS</a>認証の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　ハードウェアトークンとパスワードを併用させるなど，認証要求元の環境によらず常に二つの認証方式を併用することによって，安全性を高める。</div>
<div class="ap-choice-note">これは二要素認証の説明です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　普段と異なる環境からのアクセスと判断した場合，追加の本人認証をすることによって，不正アクセスに対抗し安全性を高める。</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/リスクベース認証" class="internal-link" data-href="用語/リスクベース認証">リスクベース認証</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　利用者が認証情報を忘れ，かつ，Webブラウザに保存しているパスワード情報も使用できない場合でも，救済することによって，利用者は普段どおりにシステムを利用できる。</div>
<div class="ap-choice-note">これはパスワードリマインダの説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/リスクベース認証" class="internal-link" data-href="用語/リスクベース認証">リスクベース認証</a>は、利用者の<a href="用語/ログイン" class="internal-link" data-href="用語/ログイン">ログイン</a>行動や環境を分析し、普段使われているものとは異なる利用環境（<a href="用語/IPアドレス" class="internal-link" data-href="用語/IPアドレス">IPアドレス</a>・ISP・OS・<a href="用語/Webブラウザ" class="internal-link" data-href="用語/Webブラウザ">Webブラウザ</a>など）からの認証要求があった場合に、通常の認証に追加する形で別の認証を求める方式です。</p>
<p>この方式は、不正<a href="用語/ログイン" class="internal-link" data-href="用語/ログイン">ログイン</a>のリスクが高いアクセスに対してだけ追加の認証を行うため、一定の利便性を保ちつつ、<a href="用語/不正アクセス" class="internal-link" data-href="用語/不正アクセス">不正アクセス</a>に対する<a href="用語/セキュリティ" class="internal-link" data-href="用語/セキュリティ">セキュリティ</a>を高めることができます。追加認証としては、<a href="用語/ワンタイムパスワード" class="internal-link" data-href="用語/ワンタイムパスワード">ワンタイムパスワード</a>、秘密の質問、SMS認証、<a href="用語/生体認証" class="internal-link" data-href="用語/生体認証">生体認証</a>などが用いられます。</p>
<p>したがって適切な記述は「ウ」です。</p>
