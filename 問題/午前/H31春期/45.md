<p class="ap-question"><strong>問45</strong>　WAFの説明はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　Webアプリケーションへの攻撃を検知し，阻止する。</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/WAF" class="internal-link" data-href="用語/WAF">WAF</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　Webブラウザの通信内容を改ざんする攻撃をPC内で監視し，検出する。</div>
<div class="ap-choice-note">これは<a href="用語/SSL/TLS" class="internal-link" data-href="用語/SSL/TLS">SSL/TLS</a>の機能です。<a href="用語/WAF" class="internal-link" data-href="用語/WAF">WAF</a>は<a href="用語/Webサーバ" class="internal-link" data-href="用語/Webサーバ">Webサーバ</a>の前段に設置されます。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　サーバのOSへの不正なログインを監視する。</div>
<div class="ap-choice-note">これはHIDS(ホスト型<a href="用語/IDS" class="internal-link" data-href="用語/IDS">IDS</a>)の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　ファイルへのマルウェア感染を監視し，検出する。</div>
<div class="ap-choice-note">これはウイルス対策ソフトの説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/WAF" class="internal-link" data-href="用語/WAF">WAF</a>(Web Application Firewall)は、通過する<a href="用語/パケット" class="internal-link" data-href="用語/パケット">パケット</a>の<a href="用語/IPアドレス" class="internal-link" data-href="用語/IPアドレス">IPアドレス</a>や<a href="用語/ポート番号" class="internal-link" data-href="用語/ポート番号">ポート番号</a>だけでなくペイロード部(データ部分)をチェックすることで、Webアプリケーションに対する攻撃を検知し、遮断することが可能な<a href="用語/ファイアウォール" class="internal-link" data-href="用語/ファイアウォール">ファイアウォール</a>です。チェックされる内容には「<a href="用語/URL" class="internal-link" data-href="用語/URL">URL</a><a href="用語/パラメータ" class="internal-link" data-href="用語/パラメータ">パラメータ</a>」や「クッキーの内容」などの<a href="用語/HTTP" class="internal-link" data-href="用語/HTTP">HTTP</a><a href="用語/ヘッダー" class="internal-link" data-href="用語/ヘッダー">ヘッダー</a>情報や、「POSTデータの内容」などのメッセージボディ部などがあります。</p>
<p>本来、<a href="用語/Webシステム" class="internal-link" data-href="用語/Webシステム">Webシステム</a>への攻撃はWebアプリケーション側で対処すべき問題ですが、<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>のないWebアプリケーションを作成するためには専門的な知識や技術が必要であるため、全てのWebアプリケーションのセキュリティ対策を万全にすることは難しいのが現実です。<a href="用語/WAF" class="internal-link" data-href="用語/WAF">WAF</a>はこのようなセキュリティ対策の不十分さを補完し、Webアプリケーションの堅牢性を高める役割をもちます。</p>
