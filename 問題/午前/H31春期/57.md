<p class="ap-question"><strong>問57</strong>　システム開発委託先(受託者)から委託元(委託者)に納品される成果物に対するユーザー受入テストの適切性を確かめるためのシステム監査の要点はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　委託者が作成したユーザー受入テスト計画書に従って，受託者が成果物に対してユーザー受入テストを実施していること</div>
<div class="ap-choice-note">受託側が受入テストを実施しているため不適切です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　受託者が成果物と一緒にユーザー受入テスト計画書を納品していること</div>
<div class="ap-choice-note">受入<a href="用語/テスト計画" class="internal-link" data-href="用語/テスト計画">テスト計画</a>は委託元が作成します。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　受託者から納品された成果物に対して，委託者が要件定義に基づきユーザー受入テストを実施していること</div>
<div class="ap-choice-note">正しい。委託元が作成した計画に基づき委託元が実施していることが重要です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　受託者から納品された成果物に対して，監査人がユーザー受入テスト計画を策定していること</div>
<div class="ap-choice-note">監査人が受入<a href="用語/テスト計画" class="internal-link" data-href="用語/テスト計画">テスト計画</a>を策定することはありません。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>委託先から<a href="用語/納品" class="internal-link" data-href="用語/納品">納品</a>された成果物に対する受入テストは、「委託元が作成した<a href="用語/テスト計画" class="internal-link" data-href="用語/テスト計画">テスト計画</a>に基づいて委託元が実施している」ことが重要です。受入テストに受託側が関与すると、受入テストの適切性・公平性が損なわれ、成果物の不具合を(意図的に)見逃してしまうリスクがあるからです。</p>
</div>
</div>
