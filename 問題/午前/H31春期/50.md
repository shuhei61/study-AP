<p class="ap-question"><strong>問50</strong>　ソフトウェアライフサイクルプロセスのうちの，システム要求事項分析プロセスにおける要求事項評価の基準はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　システム要求事項のテスト網羅性</div>
<div class="ap-choice-note">「システム結合プロセス」及び「適格性確認テスト」における評価基準として記載されています。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　システム要求事項への追跡可能性</div>
<div class="ap-choice-note">「システム方式の確立」におけるシステム方式の評価の基準です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　取得ニーズとの一貫性</div>
<div class="ap-choice-note">正しい。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　使用されたテスト方法及び作業標準の適切性</div>
<div class="ap-choice-note">「システム結合プロセス」における結合したシステムの評価基準として記載されています。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>システム要求事項分析プロセスとは、<a href="用語/JIS X 0160" class="internal-link" data-href="用語/JIS X 0160">JIS X 0160</a>(ソフトウェアライフサイクルプロセス)で定義されているプロセス名で、共通フレーム2013における<a href="用語/システム要件" class="internal-link" data-href="用語/システム要件">システム要件</a>定義プロセスに対応します。</p>
<p><a href="用語/JIS X 0160" class="internal-link" data-href="用語/JIS X 0160">JIS X 0160</a>では、システム要求事項は、次の基準を考慮して評価するとしています(共通フレーム2013でも同様です)。</p>
<ul>
<li>取得ニーズの<a href="用語/追跡可能性" class="internal-link" data-href="用語/追跡可能性">追跡可能性</a></li>
<li>取得ニーズとの<a href="用語/一貫性" class="internal-link" data-href="用語/一貫性">一貫性</a></li>
<li><a href="用語/テスト可能性" class="internal-link" data-href="用語/テスト可能性">テスト可能性</a></li>
<li>システム方式設計の<a href="用語/実現可能性" class="internal-link" data-href="用語/実現可能性">実現可能性</a></li>
<li><a href="用語/運用及び保守の実現可能性" class="internal-link" data-href="用語/運用及び保守の実現可能性">運用及び保守の実現可能性</a></li>
</ul>
<p><a href="用語/システム要件" class="internal-link" data-href="用語/システム要件">システム要件</a>定義は、要件定義プロセスで定義された利害関係者要件を基に、システムで実現する機能を明らかにする工程ですので、まとめられた<a href="用語/システム要件" class="internal-link" data-href="用語/システム要件">システム要件</a>を評価する際には、利害関係者要件を実現できるのかどうか、取得ニーズとの<a href="用語/一貫性" class="internal-link" data-href="用語/一貫性">一貫性</a>・整合性があるか、などがチェックポイントになります。したがって「ウ」が適切です。</p>
