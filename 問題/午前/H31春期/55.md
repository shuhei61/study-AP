# [平成31年春期 午前 問55](https://www.ap-siken.com/kakomon/31_haru/q55.html)

#問題 #マネジメント #サービスマネジメント #サービスマネジメントプロセス

<details class="ap-reveal">

<p class="ap-question"><strong>問55</strong>　ITIL 2011 editionによれば，ITサービス継続性管理における復旧オプションのうち，段階的復旧(コールド・スタンバイ)はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　遠隔地のデータセンターに機器を設置できる場所を確保しておき，災害時にはコンピュータ機器を設置し，ソフトウェアのセットアップやデータの復元を行って，ITサービスを復旧させる。</div>
<div class="ap-choice-note">正しい。段階的復旧(<a href="用語/コールドスタンバイ" class="internal-link" data-href="用語/コールドスタンバイ">コールドスタンバイ</a>)の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　遠隔地のデータセンターに本稼働システムと同一構成の機器を準備するとともに，本稼働システムのデータをバックアップしておき，災害時にはその遠隔地のデータセンターに切り替えてITサービスを復旧させる。</div>
<div class="ap-choice-note">高速復旧(<a href="用語/ホットスタンバイ" class="internal-link" data-href="用語/ホットスタンバイ">ホットスタンバイ</a>)の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　災害時のIT資源の相互利用について他の組織と合意しておき，災害時には他の組織のIT資源を利用してITサービスを復旧させる。</div>
<div class="ap-choice-note">相互協定の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　同時に被災する可能性がほとんどない複数のデータセンターにシステムを分散して稼働させることによって，一部のデータセンターが被災しても顧客に対するサービスの深刻な停止を引き起こすことなく，ITサービスを即時に復旧させる。</div>
<div class="ap-choice-note">即時的復旧(<a href="用語/ホットスタンバイ" class="internal-link" data-href="用語/ホットスタンバイ">ホットスタンバイ</a>)の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/ITIL" class="internal-link" data-href="用語/ITIL">ITIL</a> 2011 editionでは、ITサービスの中断に対処するための戦略として以下の6つの復旧オプションを挙げています。</p>
<dl>
<dt>段階的復旧（<a href="用語/コールドスタンバイ" class="internal-link" data-href="用語/コールドスタンバイ">コールドスタンバイ</a>）</dt>
<dd>コンピュータシステムを設置できる施設だけを確保しておき、障害発生時にはそこに機材などを搬入して<a href="用語/バックアップサイト" class="internal-link" data-href="用語/バックアップサイト">バックアップサイト</a>として機能させる方式。障害発生後には、システムの導入とデータの<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>を行って業務を引き継ぐ</dd>
<dt>中間的復旧（<a href="用語/ウォームスタンバイ" class="internal-link" data-href="用語/ウォームスタンバイ">ウォームスタンバイ</a>）</dt>
<dd><a href="用語/バックアップサイト" class="internal-link" data-href="用語/バックアップサイト">バックアップサイト</a>に本システムと同じ機器を全部(あるいは部分的に)設置しておく方式。障害発生後には、追加の機器やデータおよびプログラム媒体を搬入してから予備系システムを立ち上げて処理を引き継ぐ</dd>
<dt>高速復旧（<a href="用語/ホットスタンバイ" class="internal-link" data-href="用語/ホットスタンバイ">ホットスタンバイ</a>）</dt>
<dd><a href="用語/バックアップサイト" class="internal-link" data-href="用語/バックアップサイト">バックアップサイト</a>に本システムと同じ機器及び<a href="用語/バックアップ" class="internal-link" data-href="用語/バックアップ">バックアップ</a>データを用意しておく方式。非常時にはデータの<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>を行ってから処理を引き継ぐ。本システムを同じものを稼働させておき、定期的に<a href="用語/同期" class="internal-link" data-href="用語/同期">同期</a>をとる方式もこれに含まれる</dd>
<dt>即時的復旧（<a href="用語/ホットスタンバイ" class="internal-link" data-href="用語/ホットスタンバイ">ホットスタンバイ</a>）</dt>
<dd>日常から<a href="用語/バックアップサイト" class="internal-link" data-href="用語/バックアップサイト">バックアップサイト</a>で本システムと同じものを稼働させておき、非常事態が起きたときに切れ目なく業務を引き継ぐ方式。常に本システムのデータとの<a href="用語/同期" class="internal-link" data-href="用語/同期">同期</a>が行われており、障害発生時に直ぐにその施設でシステムを運用できる体制になっている</dd>
<dt>相互協定</dt>
<dd>他の事業者と協定を結び、非常時には他の事業者のリソースを使用してITサービスを<a href="用語/復旧" class="internal-link" data-href="用語/復旧">復旧</a>させる方法</dd>
<dt>手作業のワークアラウンド</dt>
<dd>ITサービスが再開するまでの数日間、緊急的な措置として業務を手作業で遂行する方法。通常は他の復旧オプションと併用される</dd>
</dl>
<p>したがって「ア」が正解です。</p>
