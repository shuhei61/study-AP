<p class="ap-question"><strong>問46</strong>　ソフトウェアの分析・設計技法の特徴のうち，データ中心分析・設計技法の特徴として，最も適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　機能を詳細化する過程で，モジュールの独立性が高くなるようにプログラムを分割していく。</div>
<div class="ap-choice-note">これはオブジェクト指向アプローチの説明です</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　システムの開発後の仕様変更は，データ構造や手続の局所的な変更で対応可能なので，比較的容易に実現できる。</div>
<div class="ap-choice-note">データを中心に設計するため、<a href="用語/データ構造" class="internal-link" data-href="用語/データ構造">データ構造</a>の変更を伴う仕様変更は難しくなります。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ウ　対象業務領域のモデル化に当たって，情報資源であるデータの構造に着目する。</div>
<div class="ap-choice-note">正しい。詳細：<a href="用語/データ中心アプローチ" class="internal-link" data-href="用語/データ中心アプローチ">データ中心アプローチ</a></div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　プログラムが最も効率よくアクセスできるようにデータ構造を設計する。</div>
<div class="ap-choice-note">情報の局所参照性を考えた設計手法です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/データ中心アプローチ" class="internal-link" data-href="用語/データ中心アプローチ">データ中心アプローチ</a>(<a href="用語/DOA" class="internal-link" data-href="用語/DOA">DOA</a>：Data Oriented Approach)は、プログラムとデータは独立したものと考え、データを中心に据えてシステム／ソフトウェアの設計を行う手法です。<a href="用語/E-R図" class="internal-link" data-href="用語/E-R図">E-R図</a>などで対象世界をモデル化し、それに基づいて<a href="用語/データ構造" class="internal-link" data-href="用語/データ構造">データ構造</a>などを決めて、プログラムはその共通のデータを操作するものとして設計します。</p>
<p>業務プロセスやビジネスロジックは時間や周りの環境により変化することが多いですが、<a href="用語/データ構造" class="internal-link" data-href="用語/データ構造">データ構造</a>はそれと比べて安定しています。<a href="用語/データ構造" class="internal-link" data-href="用語/データ構造">データ構造</a>を基盤としてシステム／ソフトウェアを設計することで、プロセス中心アプローチと比較して、柔軟性があり長期的な安定性を確保することができるのが強みです。</p>
</div>
