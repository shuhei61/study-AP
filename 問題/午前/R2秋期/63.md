<p class="ap-question"><strong>問63</strong>　SOAの説明はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　会計，人事，製造，購買，在庫管理，販売などの企業の業務プロセスを一元管理することによって，業務の効率化や経営資源の全体最適を図る手法</div>
<div class="ap-choice-note"><a href="用語/ERP" class="internal-link" data-href="用語/ERP">ERP</a>（Enterprise Resource Planning）の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　企業の業務プロセス，システム化要求などのニーズと，ソフトウェアパッケージの機能性がどれだけ適合し，どれだけかい離しているかを分析する手法</div>
<div class="ap-choice-note">フィット＆ギャップ分析の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　業務プロセスの問題点を洗い出して，目標設定，実行，チェック，修正行動のマネジメントサイクルを適用し，継続的な改善を図る手法</div>
<div class="ap-choice-note"><a href="用語/PDCA" class="internal-link" data-href="用語/PDCA">PDCA</a>サイクルの説明です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　利用者の視点から業務システムの機能を幾つかの独立した部品に分けることによって，業務プロセスとの対応付けや他のソフトウェアとの連携を容易にする手法</div>
<div class="ap-choice-note">正しい。<a href="用語/SOA" class="internal-link" data-href="用語/SOA">SOA</a>の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/SOA" class="internal-link" data-href="用語/SOA">SOA</a>（Service Oriented Architecture：<a href="用語/サービス" class="internal-link" data-href="用語/サービス">サービス</a>指向<a href="用語/アーキテクチャ" class="internal-link" data-href="用語/アーキテクチャ">アーキテクチャ</a>）は，業務を構成する機能や役割を<a href="用語/サービス" class="internal-link" data-href="用語/サービス">サービス</a>という独立した単位として分割し，それらの<a href="用語/サービス" class="internal-link" data-href="用語/サービス">サービス</a>を組み合わせ相互に連携させることで一つのシステムを構築する考え方です。</p>
<p>オンラインショッピングのWebサイトを例にすると，商品検索，ショッピングカート，決済，ユーザー管理などの機能などを，それぞれ独立した<a href="用語/サービス" class="internal-link" data-href="用語/サービス">サービス</a>として実装することができます。こうして分けることで，各<a href="用語/サービス" class="internal-link" data-href="用語/サービス">サービス</a>を個別に開発・変更・拡張できるようになり，開発効率や<a href="用語/保守性" class="internal-link" data-href="用語/保守性">保守性</a>が向上します。また，各<a href="用語/サービス" class="internal-link" data-href="用語/サービス">サービス</a>は独立した部品であるため，<a href="用語/再利用" class="internal-link" data-href="用語/再利用">再利用</a>や機能の入替え，システムの再構築がしやすい，特定の言語やプラットフォームに依存しないというメリットがあります。</p>
<p>したがって「エ」が正解です。</p>
</div>
