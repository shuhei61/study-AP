<p class="ap-question"><strong>問30</strong>　トランザクションのACID特性のうち，耐久性(durability)に関する記述として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　正常に終了したトランザクションの更新結果は，障害が発生してもデータベースから消失しないこと</div>
<div class="ap-choice-note">正しい。耐久性の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　データベースの内容が矛盾のない状態であること</div>
<div class="ap-choice-note">これは<a href="用語/一貫性" class="internal-link" data-href="用語/一貫性">一貫性</a>（Consistency）の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　トランザクションの処理が全て実行されるか，全く実行されないかのいずれかで終了すること</div>
<div class="ap-choice-note">これは原子性（Atomicity）の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　複数のトランザクションを同時に実行した場合と，順番に実行した場合の処理結果が一致すること</div>
<div class="ap-choice-note">これは独立性（Isolation）の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/トランザクション" class="internal-link" data-href="用語/トランザクション">トランザクション</a>の原子性を含む<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>の<a href="用語/トランザクション" class="internal-link" data-href="用語/トランザクション">トランザクション</a>処理を行う上で必要不可欠とされる4つの性質(Atomicity・Consistency・Isolation・Durability)はまとめて<a href="用語/ACID特性" class="internal-link" data-href="用語/ACID特性">ACID特性</a>と呼ばれます。</p>
<p>Atomicity（原子性）<br><a href="用語/トランザクション" class="internal-link" data-href="用語/トランザクション">トランザクション</a>内の処理がすべて実行されるか、または全く実行されないことを保証する性質。</p>
<p>Consistency（<a href="用語/一貫性" class="internal-link" data-href="用語/一貫性">一貫性</a>）<br><a href="用語/トランザクション" class="internal-link" data-href="用語/トランザクション">トランザクション</a>によりデータの矛盾が生じないこと。常に<a href="用語/データベース" class="internal-link" data-href="用語/データベース">データベース</a>の整合性が保たれていることを保証する性質。</p>
<p>Isolation（独立性）<br>複数の<a href="用語/トランザクション" class="internal-link" data-href="用語/トランザクション">トランザクション</a>を同時に実行した場合と、順番に実行した場合の結果が等しくなることを保証する性質。一般に<a href="用語/ロック" class="internal-link" data-href="用語/ロック">ロック</a>などをかけることで直列可能性を保証する。<a href="用語/隔離" class="internal-link" data-href="用語/隔離">隔離</a>性と呼ばれる場合もある。</p>
<p>Durability（耐久性）<br>一旦正常終了した<a href="用語/トランザクション" class="internal-link" data-href="用語/トランザクション">トランザクション</a>の結果は、以後システムに<a href="用語/障害" class="internal-link" data-href="用語/障害">障害</a>が発生しても失われないことを保証する性質。永続性と呼ばれる場合もある。</p>
<p>各記述を<a href="用語/ACID特性" class="internal-link" data-href="用語/ACID特性">ACID特性</a>に当てはめると、「ア」が耐久性の説明です。</p>
</div>
