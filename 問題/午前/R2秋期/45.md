<p class="ap-question"><strong>問45</strong>　電子メールをスマートフォンで受信する際のメールサーバとスマートフォンとの間の通信をメール本文を含めて暗号化するプロトコルはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　APOP</div>
<div class="ap-choice-note">Authenticated POPの略。認証時のパスワード送信をMD5ハッシュ化して安全性を高めたプロトコル。暗号化されるのはパスワードだけなので、メール本文は平文のまま通信経路上を流れる。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　IMAPS</div>
<div class="ap-choice-note">正しい。IMAPSは認証やメール本文の受信など、<a href="用語/IMAP" class="internal-link" data-href="用語/IMAP">IMAP</a>通信の全てをTLSによって暗号化するプロトコル。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　POP3</div>
<div class="ap-choice-note">これは<a href="用語/POP3" class="internal-link" data-href="用語/POP3">POP3</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　SMTP Submission</div>
<div class="ap-choice-note">ユーザー(メールソフト)からのメール送信を専門に受け付ける仕組み。メール受信用途ではない。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>IMAPS(<a href="用語/IMAP" class="internal-link" data-href="用語/IMAP">IMAP</a> over <a href="用語/SSL/TLS" class="internal-link" data-href="用語/SSL/TLS">SSL/TLS</a>)は、メール受信プロトコルである<a href="用語/IMAP" class="internal-link" data-href="用語/IMAP">IMAP</a>(Internet Message Access Protocol)にTLSを組み合わせ、TLSによって暗号化された通信コネクション上でメール受信を行うプロトコルです。バージョン番号を付けてIMAP4Sと表記されることもあります。</p>
<p>既存のプロトコル名の後ろに"S"が付いているものは、TLSにより暗号化されたものであることが多いです。<a href="用語/HTTPS" class="internal-link" data-href="用語/HTTPS">HTTPS</a>もその1つです。<a href="用語/POP3" class="internal-link" data-href="用語/POP3">POP3</a>にもTLS版であるPOP3Sというプロトコルがあります。</p>
<p>したがって「イ」が適切です。</p>
</div>
