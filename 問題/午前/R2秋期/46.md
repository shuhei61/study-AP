<p class="ap-question"><strong>問46</strong>　UMLのアクティビティ図の特徴はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　多くの並行処理を含むシステムの，オブジェクトの振る舞いが記述できる。</div>
<div class="ap-choice-note">正しい。<a href="用語/アクティビティ図" class="internal-link" data-href="用語/アクティビティ図">アクティビティ図</a>の特徴です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　オブジェクト群がどのようにコラボレーションを行うか記述できる。</div>
<div class="ap-choice-note"><a href="用語/コミュニケーション図" class="internal-link" data-href="用語/コミュニケーション図">コミュニケーション図</a>の特徴です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　クラスの仕様と，クラスの間の静的な関係が記述できる。</div>
<div class="ap-choice-note"><a href="用語/クラス図" class="internal-link" data-href="用語/クラス図">クラス図</a>の特徴です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　システムのコンポーネント間の物理的な関係が記述できる。</div>
<div class="ap-choice-note"><a href="用語/コンポーネント図" class="internal-link" data-href="用語/コンポーネント図">コンポーネント図</a>の特徴です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/アクティビティ図" class="internal-link" data-href="用語/アクティビティ図">アクティビティ図</a>は、ビジネスプロセスの流れやプログラムの制御フローのような一連の手続きを可視化できる図です。<a href="用語/フローチャート" class="internal-link" data-href="用語/フローチャート">フローチャート</a>の<a href="用語/UML" class="internal-link" data-href="用語/UML">UML</a>版と言えるでしょう。<a href="用語/フローチャート" class="internal-link" data-href="用語/フローチャート">フローチャート</a>と似た表記法で処理の流れを記述できるほか、処理の分岐やマージ、並行処理のフォークやジョイン、タイマー制御や例外処理なども表現できるようになっています。</p><p><img class="ap-fig" src="../../../assets/問題/午前/R2秋期/46/46.png" alt="解説図" width="489" height="381" ></p>
<p>したがって「ア」が適切な記述です。</p>
