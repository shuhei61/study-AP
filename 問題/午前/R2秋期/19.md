<p class="ap-question"><strong>問19</strong>　OSS(Open Source Software)における，ディストリビュータの役割はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　OSSやアプリケーションソフトウェアを組み合わせて，パッケージにして提供する。</div>
<div class="ap-choice-note">正しい。ディストリビュータの役割です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　OSSを開発し，活動状況をWebで公開する。</div>
<div class="ap-choice-note">コミッタの役割です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　OSSを稼働用のコンピュータにインストールし，動作確認を行う。</div>
<div class="ap-choice-note">コントリビュータの役割です。テスタやレビューアが含まれます。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　OSSを含むソフトウェアを利用したシステムの提案を行う。</div>
<div class="ap-choice-note"><a href="用語/OSS" class="internal-link" data-href="用語/OSS">OSS</a>ベンダーやシステムインテグレータの役割です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>ディストリビュータ(Distributor)は、<a href="用語/OSS" class="internal-link" data-href="用語/OSS">OSS</a>として個々に開発されている(狭義の)OS、<a href="用語/ライブラリ" class="internal-link" data-href="用語/ライブラリ">ライブラリ</a>やコマンドなどのユーティリティ、<a href="用語/GUI" class="internal-link" data-href="用語/GUI">GUI</a>、サーバソフトウェア、アプリケーションソフトウェアなどを組み合わせた実用的な<a href="用語/パッケージ" class="internal-link" data-href="用語/パッケージ">パッケージ</a>を提供する者や組織です。提供される<a href="用語/パッケージ" class="internal-link" data-href="用語/パッケージ">パッケージ</a>を「ディストリビューション」と言い、<a href="用語/Linux" class="internal-link" data-href="用語/Linux">Linux</a>だとDebian系やRedHat系が有名です。ディストリビューションには基本的な機能が一式揃っているので、利用者はディストリビューションをインストールするだけで、複雑な環境設定等をせずに様々な機能を使えるようになります。</p>
</div>
