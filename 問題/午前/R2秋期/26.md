<p class="ap-question"><strong>問26</strong>　表に対するSQLのGRANT文の説明として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　パスワードを設定してデータベースへの接続を制限する。</div>
<div class="ap-choice-note">これはALTER USER文の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　ビューを作成して，ビューの基となる表のアクセスできる行や列を制限する。</div>
<div class="ap-choice-note">これはCREATE VIEW文の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　表のデータを暗号化して，第三者がアクセスしてもデータの内容が分からないようにする。</div>
<div class="ap-choice-note">これはCREATE TABLE文またはALTER TABLE文の説明です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　表の利用者に対し，表への問合せ，更新，追加，削除などの操作権限を付与する。</div>
<div class="ap-choice-note">正しい。<a href="用語/GRANT文" class="internal-link" data-href="用語/GRANT文">GRANT文</a>の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/GRANT文" class="internal-link" data-href="用語/GRANT文">GRANT文</a>は、特定のユーザーに表などのオブジェクトに関する権限を付与する<a href="用語/SQL" class="internal-link" data-href="用語/SQL">SQL</a>文です。したがって正解は「エ」になります。</p>
<p>GRANT オブジェクトの権限 ON オブジェクト名<br>TO { ユーザー名 | ロール名 | PUBLIC }<br>[ WITH GRANT OPTION ]</p>
<p>反対に付与されている権限を取り消すには、REVOKE文を使用します。</p>
<p>REVOKE オブジェクトの権限 ON オブジェクト名<br>FROM { ユーザー名 | ロール名 | PUBLIC }</p>
</div>
