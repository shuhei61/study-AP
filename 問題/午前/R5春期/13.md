<p class="ap-question"><strong>問13</strong>　スケールインの説明として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　想定されるCPU使用率に対して，サーバの能力が過剰なとき，CPUの能力を減らすこと</div>
<div class="ap-choice-note">スケールダウンの説明。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　想定されるシステムの処理量に対して，サーバの台数が過剰なとき，サーバの台数を減らすこと</div>
<div class="ap-choice-note">正しい。スケールインの説明。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　想定されるシステムの処理量に対して，サーバの台数が不足するとき，サーバの台数を増やすこと</div>
<div class="ap-choice-note"><a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>の説明。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　想定されるメモリ使用率に対して，サーバの能力が不足するとき，メモリの容量を増やすこと</div>
<div class="ap-choice-note"><a href="用語/スケールアップ" class="internal-link" data-href="用語/スケールアップ">スケールアップ</a>の説明。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>システムの能力を増強する方法として<a href="用語/スケールアウト" class="internal-link" data-href="用語/スケールアウト">スケールアウト</a>と<a href="用語/スケールアップ" class="internal-link" data-href="用語/スケールアップ">スケールアップ</a>、逆に縮小する方法としてスケールインとスケールダウンがあります。</p><p><img class="ap-fig" src="../../../assets/問題/午前/R5春期/13/13.png" alt="解説図" width="489" height="101" ></p>
<p>スケールインは、システムに余剰能力があるときに、物理サーバの台数を減らすことでシステムの能力を縮小することです。したがって「イ」が正解です。</p>
</div>
