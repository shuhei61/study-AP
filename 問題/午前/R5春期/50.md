<p class="ap-question"><strong>問50</strong>　サーバプロビジョニングツールを使用する目的として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　サーバ上のサービスが動作しているかどうかを，他のシステムからリモートで監視する。</div>
<div class="ap-choice-note">サーバ監視ツールの目的です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　サーバにインストールされているソフトウェアを一元的に管理する。</div>
<div class="ap-choice-note"><a href="用語/構成管理" class="internal-link" data-href="用語/構成管理">構成管理</a>ツールの目的です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　サーバを監視して，システムやアプリケーションのパフォーマンスを管理する。</div>
<div class="ap-choice-note">パフォーマンス監視ツールの目的です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　システム構成をあらかじめ記述しておくことによって，サーバを自動的に構成する。</div>
<div class="ap-choice-note">正しい。サーバ<a href="用語/プロビジョニング" class="internal-link" data-href="用語/プロビジョニング">プロビジョニング</a>ツールの目的です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>サーバの<a href="用語/プロビジョニング" class="internal-link" data-href="用語/プロビジョニング">プロビジョニング</a>とは、サーバを使用可能な状態にするために行う、ハードウェアのセットアップやOSやアプリケーションのインストール、設定の変更、<a href="用語/ミドルウェア" class="internal-link" data-href="用語/ミドルウェア">ミドルウェア</a>、ネットワーク、ストレージとの接続、ユーザーアカウントの作成などの一連の準備作業です。</p>
<p>サーバ<a href="用語/プロビジョニング" class="internal-link" data-href="用語/プロビジョニング">プロビジョニング</a>ツールは、設定ファイルに記述した内容をもとに、<a href="用語/プロビジョニング" class="internal-link" data-href="用語/プロビジョニング">プロビジョニング</a>作業のうちソフト的な作業を人に代わって自動的に行うものです。サーバ<a href="用語/プロビジョニング" class="internal-link" data-href="用語/プロビジョニング">プロビジョニング</a>ツールを使うことにより、手動でサーバの初期設定をする時間や費用の削減や、人的ミスを回避できるといったメリットがあります。大量のサーバを使用するシステムなどではツールを使って自動化しているのが普通です。</p>
