<p class="ap-question"><strong>問28</strong>　データベースシステムの操作の説明のうち，べき等(idempotent)な操作の説明はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">ア　同一の操作を複数回実行した結果と，一回しか実行しなかった結果が同一になる操作</div>
<div class="ap-choice-note">正しい。べき等な操作の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　トランザクション内の全ての処理が成功したか，何も実行されなかったかのいずれかの結果にしかならない操作</div>
<div class="ap-choice-note">これは<a href="用語/ACID特性" class="internal-link" data-href="用語/ACID特性">ACID特性</a>のうち原子性（Atomicity）の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　一つのノードへのレコードの挿入処理を，他のノードでも実行する操作</div>
<div class="ap-choice-note">これは<a href="用語/レプリケーション" class="internal-link" data-href="用語/レプリケーション">レプリケーション</a>の説明です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　複数のトランザクションを同時に実行した結果と，順番に実行した結果が同一になる操作</div>
<div class="ap-choice-note">これは直列化可能性の説明です。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>べき等(Idempotent)とは、コンピュータサイエンスの世界で使われる言葉で、ある操作を1回実行しても複数回実行しても結果が同じになる性質です。主要なデータ操作<a href="用語/SQL" class="internal-link" data-href="用語/SQL">SQL</a>文のうち、SELECTと障害回復時のUNDO・REDOはべき等、INSERT・UPDATE・DELETEはべき等ではありません。</p>
<p>したがって「ア」の説明が適切です。</p>
</div>
