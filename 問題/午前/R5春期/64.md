<p class="ap-question"><strong>問64</strong>　システム要件定義プロセスにおいて，トレーサビリティが確保されていることを説明した記述として，適切なものはどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　移行マニュアルや運用マニュアルなどの文書化が完了しており，システム上でどのように業務を実施するのかを利用者が確認できる。</div>
<div class="ap-choice-note">移行・運用マニュアルの整備は利用者向けの手順説明であり、利害関係者の要求と成果物の相互関係を追跡できる<a href="用語/トレーサビリティ" class="internal-link" data-href="用語/トレーサビリティ">トレーサビリティ</a>の説明ではない。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　所定の内外作基準に基づいて外製する部分が決定され，調達先が選定され，契約が締結されており，調達先を容易に変更することはできない。</div>
<div class="ap-choice-note">外製・調達・契約の確定は調達計画の話であり、要求の根拠と成果物の関係を文書化して追跡する<a href="用語/トレーサビリティ" class="internal-link" data-href="用語/トレーサビリティ">トレーサビリティ</a>とは別の内容である。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　モジュールの相互依存関係が確定されており，以降の開発プロセスにおいて個別モジュールの仕様を変更することはできない。</div>
<div class="ap-choice-note">モジュール間の依存関係の固定は設計・実装の制約の話であり、利害関係者の要求から成果物までの<a href="用語/追跡可能性" class="internal-link" data-href="用語/追跡可能性">追跡可能性</a>を示す記述ではない。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　利害関係者の要求の根拠と成果物の相互関係が文書化されており，開発の途中で生じる仕様変更をシステムに求められる品質に立ち返って検証できる。</div>
<div class="ap-choice-note">正しい。要求の根拠と成果物の関係が記録され、仕様変更時に<a href="用語/品質要件" class="internal-link" data-href="用語/品質要件">品質要件</a>へ立ち返って検証できる状態が<a href="用語/トレーサビリティ" class="internal-link" data-href="用語/トレーサビリティ">トレーサビリティ</a>の確保を示す。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/トレーサビリティ" class="internal-link" data-href="用語/トレーサビリティ">トレーサビリティ</a>(Traceability)は、製品やサービスのライフサイクルにおいて、現在の状態に至るまでのプロセスや変更の履歴を追跡できる能力のことです。履歴を整えておくことで、その物品に関心を持つ人が過去の履歴を追跡したり、物品に問題がある場合に購入した顧客を特定したりといったことが可能となります。</p>
<p>現在の開発に至るまでの過去の意思決定が過不足なく記録され、必要に応じて参照することができる「エ」が正解となります。</p>
</div>
