<p class="ap-question"><strong>問40</strong>　ソフトウェアの既知の脆弱性を一意に識別するために用いる情報はどれか。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　CCE(Common Configuration Enumeration)</div>
<div class="ap-choice-note">これはCCE(共通セキュリティ設定)の説明です。コンピュータのセキュリティ設定項目ごとに付けられた識別子であり、<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>の識別子ではありません。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">イ　CVE(Common Vulnerabilities and Exposures)</div>
<div class="ap-choice-note">正しい。<a href="用語/CVE" class="internal-link" data-href="用語/CVE">CVE</a>は、<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>ごとに付けられている識別子です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　CVSS(Common Vulnerability Scoring System)</div>
<div class="ap-choice-note">これは<a href="用語/CVSS" class="internal-link" data-href="用語/CVSS">CVSS</a>(共通<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>評価システム)の説明です。情報システムの<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>の深刻度を同一の基準のもとで定量的に評価する共通的な手法であり、<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>を一意に識別する識別子ではありません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">エ　CWE(Common Weakness Enumeration)</div>
<div class="ap-choice-note">これは<a href="用語/CWE" class="internal-link" data-href="用語/CWE">CWE</a>(共通<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>タイプ)の説明です。ソフトウェアの<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>の種類を識別するための共通基準であり、個々の既知の<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>を一意に識別する識別子ではありません。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/CVE" class="internal-link" data-href="用語/CVE">CVE</a>(共通<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>識別子)は、一般に知られている個々の<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>ごとに採番された識別子です。世界各国の製品開発企業、セキュリティ関連企業、<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>検査ツールや<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>対策情報提供サービスにおいて、<a href="用語/脆弱性" class="internal-link" data-href="用語/脆弱性">脆弱性</a>を識別するために使用されています。採番の形式は <a href="用語/CVE" class="internal-link" data-href="用語/CVE">CVE</a>-[西暦年号]-[4桁以上の数字] です。</p>
</div>
