<p class="ap-question"><strong>問66</strong>　組込み機器の開発を行うために，ベンダーに見積りを依頼する際に必要なものとして，適切なものはどれか。ここで，システム開発の手順は共通フレーム2013に沿うものとする。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　納品書</div>
<div class="ap-choice-note"><a href="用語/納品" class="internal-link" data-href="用語/納品">納品</a>書は、完成した製品やシステムを顧客に<a href="用語/納品" class="internal-link" data-href="用語/納品">納品</a>する際に提出される文書です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　評価仕様書</div>
<div class="ap-choice-note">評価仕様書は、テストの判断基準を定めた文書です。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　見積書</div>
<div class="ap-choice-note">見積書は、顧客からの見積要求に応じて、ベンダーから顧客に対して提出される文書です。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　要件定義書</div>
<div class="ap-choice-note">正しい。要件定義書は、開発する製品やシステムに求められる機能、性能、制約条件などをまとめた文書です。要件が明確に決まっていなければ、ベンダー側は何を作成すべきか正確に把握できず、見積もりをすることができません。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p>ベンダーに見積りを依頼するには、開発対象の機能・性能・制約などが記載された要件定義書が必要です。要件が定まっていないと、ベンダーは作成すべき範囲を把握できず見積もりができません。<a href="用語/納品" class="internal-link" data-href="用語/納品">納品</a>書は<a href="用語/納品" class="internal-link" data-href="用語/納品">納品</a>時、見積書はベンダーから顧客へ提出される文書、評価仕様書はテストの判断基準を定めた文書です。</p>
</div>
</div>
