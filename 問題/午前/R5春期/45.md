<p class="ap-question"><strong>問45</strong>　次に示すような組織の業務環境において，特定のIPセグメントのIPアドレスを幹部のPCに動的に割り当て，一部のサーバへのアクセスをそのIPセグメントからだけ許可することによって，幹部のPCだけが当該サーバにアクセスできるようにしたい。利用するセキュリティ技術として，適切なものはどれか。<br>〔組織の業務環境〕業務ではサーバにアクセスする。サーバは，組織の内部ネットワークからだけアクセスできる。幹部及び一般従業員は同一フロアで業務を行っており，日によって席が異なるフリーアドレス制を取っている。各席には有線LANポートが設置されており，PCを接続して組織の内部ネットワークに接続する。ネットワークスイッチ1台に全てのPCとサーバが接続される。</p>
<ul class="ap-choices">
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ア　IDS</div>
<div class="ap-choice-note">これは<a href="用語/侵入検知" class="internal-link" data-href="用語/侵入検知">侵入検知</a>システムの説明です。本問の要件であるユーザー単位の<a href="用語/VLAN" class="internal-link" data-href="用語/VLAN">VLAN</a>割当てやIPセグメントによる<a href="用語/アクセス制御" class="internal-link" data-href="用語/アクセス制御">アクセス制御</a>には該当しません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">イ　IPマスカレード</div>
<div class="ap-choice-note">これはIPマスカレードの説明です。プライベートアドレスの複数端末を1つのグローバルIPでインターネット接続する技術であり、内部ネットワーク内の幹部向けセグメント分離には該当しません。</div>
</li>
<li class="ap-choice-item ap-wrong">
<div class="ap-choice-text">ウ　スタティックVLAN</div>
<div class="ap-choice-note">ポートベースで<a href="用語/VLAN" class="internal-link" data-href="用語/VLAN">VLAN</a>を固定する方式です。フリーアドレス制では利用ポートが日々変わるため、幹部のPCだけを特定セグメントに分離できません。</div>
</li>
<li class="ap-choice-item ap-correct">
<div class="ap-choice-text">エ　認証VLAN</div>
<div class="ap-choice-note">正しい。接続ポートでユーザーを識別できない環境でも、認証によりユーザーごとに<a href="用語/VLAN" class="internal-link" data-href="用語/VLAN">VLAN</a>を割り当て、幹部向けIPセグメントへ振り分けられます。</div>
</li>
</ul>
<div class="ap-explanation">
<h4>解説</h4>
<p><a href="用語/認証VLAN" class="internal-link" data-href="用語/認証VLAN">認証VLAN</a>は、<a href="用語/VLAN" class="internal-link" data-href="用語/VLAN">VLAN</a>の方式の1つで、ネットワークの接続前に<a href="用語/MAC" class="internal-link" data-href="用語/MAC">MAC</a>アドレス認証、IDとパスワード認証、IEEE 802.1xなどでユーザーを特定し、ユーザーごとに所属すべき<a href="用語/VLAN" class="internal-link" data-href="用語/VLAN">VLAN</a>に振り分けることで端末のグルーピングを行います。これにより接続ポート単位やアドレス単位というグルーピングができない場合でも、<a href="用語/VLAN" class="internal-link" data-href="用語/VLAN">VLAN</a>に対応させることができます。</p>
</div>
